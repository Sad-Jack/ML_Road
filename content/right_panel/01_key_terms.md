# Ключевые термины и краткие объяснения

## 0. Карта направления

### AI — Artificial Intelligence / Искусственный интеллект

Общее название для систем, которые пытаются выполнять задачи, похожие на человеческое мышление.

ML — это одна из частей AI.

### Machine Learning / Машинное обучение

Подход, при котором система не получает все правила вручную, а учится находить закономерности на данных.

Проще говоря: мы показываем примеры, а модель учится делать выводы сама.

### Deep Learning / Глубокое обучение

Это часть ML, где используют большие нейронные сети.

Обычно применяется там, где задачи сложные: картинки, речь, тексты.

### Data Science

Направление про работу с данными: изучение, анализ, поиск закономерностей и построение моделей.

ML — это только одна часть Data Science.

### Model / Модель

Это математическая система, которая учится на данных и потом делает предсказание.

Например: “купит клиент товар или нет”.

### Prediction / Предсказание

Ответ модели на новых данных.

Например: вероятность того, что клиент уйдёт.

### Training / Обучение

Этап, когда модель смотрит на примеры и настраивает себя.

Именно тут она “учится”.

### Inference

Этап, когда уже обученная модель применяется в реальной жизни.

То есть обучение закончилось, и теперь модель просто выдает ответы.

---

## 1. База про данные

### Dataset / Датасет

Это набор данных, на котором мы работаем.

Обычно это таблица, где строки — объекты, а столбцы — их свойства.

### Row / Строка

Одна запись в таблице.

Например, один клиент, один заказ или один товар.

### Column / Колонка

Один признак или одно поле в таблице.

Например: возраст, город, цена.

### Feature / Признак

Это информация, которую мы даём модели на вход.

Именно по признакам модель пытается понять ответ.

### Target / Целевая переменная

Это то, что мы хотим предсказать.

Например: уйдёт клиент или нет, какая будет цена, купит ли человек товар.

### X

Так часто называют все входные признаки модели.

Проще говоря: всё, на что модель смотрит перед ответом.

### y

Так часто называют target — правильный ответ, который модель должна научиться предсказывать.

Например: `0` или `1`, если это задача “да/нет”.

### Label

Почти то же самое, что target, особенно в задачах классификации.

Это правильный класс или ответ для примера.

### Missing values / Пропуски

Это пустые значения в данных.

Они мешают обучению и часто требуют отдельной обработки.

### Numerical feature / Числовой признак

Признак, который выражен числом.

Например: возраст, зарплата, количество покупок.

### Categorical feature / Категориальный признак

Признак, который описывает категорию, а не число.

Например: город, пол, тип клиента.

---

## 2. Как устроена задача в ML

### Supervised Learning / Обучение с учителем

У модели есть примеры с готовыми правильными ответами.

Она учится по ним предсказывать ответ для новых данных.

### Unsupervised Learning / Обучение без учителя

У модели нет готовых правильных ответов.

Она сама ищет структуру, группы или закономерности в данных.

### Classification / Классификация

Задача, где модель выбирает один из вариантов ответа.

Например: “спам / не спам”, “уйдёт / не уйдёт”.

### Regression / Регрессия

Задача, где модель предсказывает число.

Например: цену квартиры или сумму заказа.

### Clustering / Кластеризация

Задача, где похожие объекты автоматически собираются в группы.

Например: сегменты клиентов по поведению.

### Binary classification

Классификация, где всего два варианта ответа.

Например: “да / нет”.

### Multiclass classification

Классификация, где вариантов ответа больше двух.

Например: определить категорию товара из нескольких классов.

### Baseline

Это очень простое стартовое решение, с которым сравнивают всё остальное.

Нужно, чтобы понять: новая модель реально лучше или просто сложнее.

---

## 3. Разделение данных

### Train set / Обучающая выборка

Часть данных, на которой модель учится.

Это её “учебник”.

### Validation set / Валидационная выборка

Часть данных, на которой мы проверяем модель в процессе работы.

Она помогает сравнивать варианты и настраивать модель.

### Test set / Тестовая выборка

Часть данных для финальной честной проверки.

На ней смотрят, как модель работает на новых данных.

### Split / Разбиение данных

Это разделение датасета на train, validation и test.

Нужно, чтобы обучение и проверка были честными.

### Generalization / Обобщающая способность

Способность модели хорошо работать не только на знакомых данных, но и на новых.

Это одна из главных целей ML.

---

## 4. Как модель учится

### Algorithm / Алгоритм

Это способ, по которому строится модель.

Например: линейная регрессия, дерево решений, нейросеть.

### Parameter / Параметр

Это внутренние настройки модели, которые она подбирает во время обучения.

Именно через них модель “запоминает” найденные закономерности.

### Epoch / Эпоха

Один полный проход модели по всем обучающим данным.

Часто модель обучается много эпох подряд.

### Loss function / Функция потерь

Это число, которое показывает, насколько модель ошибается во время обучения.

Чем меньше loss, тем лучше модель учится на текущем этапе.

### Optimization / Оптимизация

Процесс, в котором модель пытается уменьшить ошибку.

То есть постепенно становится точнее.

### Gradient descent / Градиентный спуск

Один из основных способов уменьшать ошибку модели.

Если очень просто: это метод, который помогает идти в сторону лучшего решения маленькими шагами.

### Learning rate / Скорость обучения

Показывает, насколько большими шагами модель меняет свои настройки.

Слишком большой шаг может мешать обучению, слишком маленький — делать его очень медленным.

---

## 5. Как понять, хороша ли модель

### Metric / Метрика

Это число, по которому мы оцениваем качество модели.

Разные задачи требуют разных метрик.

### Accuracy

Доля правильных ответов из всех ответов.

Полезная метрика, но иногда может обманывать, если классы несбалансированы.

### Precision

Показывает, как часто модель права, когда говорит “да”.

Важна там, где ложные срабатывания дорогие.

### Recall

Показывает, сколько настоящих “да” модель смогла найти.

Важна там, где опасно что-то пропустить.

### F1-score

Это баланс между precision и recall.

Удобно, когда важны оба показателя сразу.

### MAE — Mean Absolute Error

Средняя абсолютная ошибка в числовых задачах.

Показывает, на сколько в среднем модель ошибается.

### MSE — Mean Squared Error

Средняя ошибка, где большие промахи наказываются сильнее.

Часто используется в задачах регрессии.

### RMSE

Это корень из MSE.

Его удобно читать, потому что он ближе к обычному масштабу ошибки.

## 6. Что может пойти не так

### Overfitting / Переобучение

Модель слишком хорошо запомнила обучающие данные, но плохо работает на новых.

То есть она “выучила примеры”, а не поняла общий принцип.

### Underfitting / Недообучение

Модель слишком простая или слишком слабо обучена.

Она плохо работает даже на обучающих данных.

### Data leakage / Утечка данных

Ситуация, когда в признаки случайно попадает информация из будущего или почти готовый ответ.

Из-за этого модель кажется умной, хотя на деле проверка нечестная.

### Noise / Шум

Случайные или бесполезные данные, которые не помогают решать задачу.

Они могут мешать модели учиться.

### Outlier / Выброс

Очень необычное значение, сильно отличающееся от остальных.

Иногда это ошибка, а иногда редкий, но реальный случай.

### Imbalanced data / Дисбаланс классов

Ситуация, когда одного класса очень много, а другого очень мало.

Из-за этого модель может научиться игнорировать редкий, но важный класс.

## 7. Подготовка данных

### Preprocessing / Предобработка

Подготовка данных перед обучением.

Сюда входит очистка, заполнение пропусков, преобразование типов и другие шаги.

### Encoding / Кодирование категорий

Преобразование текстовых категорий в числовой вид, понятный модели.

Например, чтобы модель могла работать с колонкой “город”.

### One-hot encoding

Способ кодирования категорий, когда каждая категория превращается в отдельную колонку.

Например: Москва, Берлин, Париж — каждая как отдельный признак.

### Scaling / Масштабирование

Приведение числовых признаков к более удобному масштабу.

Это помогает некоторым моделям учиться стабильнее.

### Normalization / Нормализация

Один из способов привести значения к общему диапазону.

Часто используется, когда признаки слишком разные по масштабу.

### Standardization / Стандартизация

Преобразование данных так, чтобы у признака были удобные средние и разброс.

Это тоже помогает ряду моделей работать лучше.

### Feature engineering

Создание или улучшение признаков на основе исходных данных.

Часто хороший feature engineering даёт сильный прирост качества.

### Feature importance

Оценка того, насколько сильно признак влияет на решение модели.

Помогает понять, что для модели действительно важно.

## 8. Более глубокий слой

### Neural network / Нейронная сеть

Модель, вдохновлённая идеей работы нейронов.

Хорошо подходит для сложных задач: тексты, картинки, звук.

### Embedding

Способ представить объект в виде набора чисел так, чтобы похожие объекты были ближе друг к другу.

Часто используется для слов, текстов, товаров, пользователей.

### Transformer

Современная архитектура нейросетей, которая особенно хорошо работает с текстом.

Именно на ней построено много LLM.

### LLM — Large Language Model

Большая языковая модель, обученная на огромном количестве текста.

Она умеет понимать запросы и генерировать осмысленный текст.

### Fine-tuning

Дообучение уже готовой модели на своей задаче или своих данных.

Это дешевле и быстрее, чем обучать всё с нуля.

### Prompt

Текстовая инструкция, которую пользователь даёт языковой модели.

От формулировки prompt часто сильно зависит качество ответа.

---

## 9. Продовый слой

### Deployment

Этап, когда модель выкладывают в рабочую систему.

После этого её могут использовать реальные пользователи или сервисы.

### Monitoring

Наблюдение за моделью после запуска.

Нужно, чтобы замечать ошибки, просадку качества и странное поведение.

### Data drift

Ситуация, когда новые данные стали отличаться от тех, на которых модель училась.

Из-за этого качество модели может падать.

### Model drift

Ситуация, когда сама модель со временем начинает работать хуже.

Обычно это видно уже после запуска в реальной среде.

### Retraining

Повторное обучение модели на новых данных.

Нужно, чтобы она не устаревала.

### A/B test

Проверка двух вариантов решения на реальных пользователях.

Помогает понять, какой вариант действительно лучше на практике.
