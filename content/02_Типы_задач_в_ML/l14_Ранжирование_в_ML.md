# 14. Ранжирование в ML

## Зачем это нужно

Когда ты вводишь запрос в поисковик, тебе важно не просто получить релевантные страницы — тебе важно, чтобы самые нужные были первыми. Задача ранжирования — это про порядок, а не про абсолютные значения. Именно это отличает её от обычной регрессии или классификации.

## Простая интуиция

Представь жюри на конкурсе. Их задача не поставить каждому участнику оценку от 1 до 10, а выстроить финальный список: кто первый, кто второй, кто третий. Даже если два участника почти одинаковые — кто-то должен быть выше. Важен относительный порядок, а не абсолютная оценка.

То же самое с поиском: ты ищешь «ноутбук для работы», система находит сотни подходящих страниц. Её задача — поставить самую нужную первой. Пользователь редко смотрит дальше третьего результата.

## Что это такое

Ранжирование (ranking) — ML-задача, где нужно упорядочить объекты по релевантности для данного запроса или пользователя.

Три главных понятия:
- **Запрос (query)** — то, для чего ранжируем: поисковая фраза, профиль пользователя, открытая страница товара
- **Объекты (items)** — то, что ранжируем: страницы, товары, посты, фильмы
- **Релевантность** — насколько объект подходит для данного запроса. Это не фиксированное свойство объекта: один и тот же товар может быть очень релевантен одному запросу и совсем не нужен другому

Ключевое отличие от регрессии: модели важен не точный числовой score, а правильный **порядок**. Небольшая ошибка в абсолютном значении не страшна, если итоговый список расставлен верно.

## Как это работает

Модель получает запрос и список объектов. Для каждого объекта она вычисляет **score релевантности** — число, показывающее насколько объект подходит для этого запроса. Затем объекты сортируются по score — это и есть финальный список.

Пример — запрос «ноутбук для работы»:

| Результат | Score | Позиция |
|---|---:|:---:|
| Обзор MacBook Pro для офиса | 0.92 | 1 |
| Топ-10 ноутбуков для работы | 0.87 | 2 |
| Игровой ноутбук Asus ROG | 0.61 | 3 |
| Ремонт ноутбуков, Москва | 0.34 | 4 |
| История компании Lenovo | 0.11 | 5 |

Система не говорит «релевантен / не релевантен» — она говорит _насколько_, и сортирует по этому числу.

**Почему позиция критична**

Первый результат получает в разы больше кликов, чем второй. Второй — больше, чем третий. Пятый результат почти никто не смотрит. Поэтому «поставить нужное на 3-е место вместо 1-го» — это реальная ошибка, которую модель должна учиться не допускать.

**Как модель учится**

Сигнал — поведение пользователей. Если при запросе «ноутбук для работы» пользователи чаще кликают на второй результат, а не на первый — значит второй на самом деле релевантнее. Модель учится на этих сигналах и корректирует порядок.

**Два этапа в реальных системах**

Поисковать среди миллионов объектов сразу — дорого. Поэтому реальные системы работают в два шага:
1. **Быстрый отбор** — из миллионов грубо отбираются несколько сотен кандидатов
2. **Точное ранжирование** — среди кандидатов модель расставляет точный порядок

## Практика

1. В чём принципиальная разница между ранжированием и регрессией? Почему нельзя просто предсказывать «насколько товар хороший» числом и сортировать по нему?
2. Посмотри на таблицу выше. Модель поставила игровой ноутбук на 3-е место при запросе «ноутбук для работы». Это ошибка или нормально? Аргументируй.
3. Один и тот же товар «беговые кроссовки Nike» может быть релевантен одному запросу и нерелевантен другому. Придумай два таких запроса.
4. Почему позиция в списке так важна? Что меняется для бизнеса, если нужный товар стоит на 6-м месте вместо 1-го?
5. Мини-кейс: маркетплейс показывает результаты поиска «зимние ботинки», но ставит на первое место самые дорогие модели — у них выше маржа. Почему это плохая стратегия и как это скажется на пользователях?

## Краткий итог

- В ранжировании ошибка на первом месте стоит дороже всего: пользователь смотрит на топ-5, остальное игнорирует
- Модель вычисляет score для каждого объекта → сортировка по score = финальный список
- Релевантность зависит от запроса: один и тот же объект нужен для одного запроса и не нужен для другого
- Позиция критична: первый результат получает в разы больше внимания, чем пятый
- Accuracy не работает — она не учитывает порядок; нужны метрики, чувствительные к позиции
- В реальных системах: быстрый отбор кандидатов → точное ранжирование
- Применяется в поиске, лентах, маркетплейсах
- В реальных системах ранжирование многоэтапное: отбор кандидатов + точное ранжирование

## Ключевые термины урока

- **Ранжирование** — задача упорядочивания объектов по релевантности для конкретного запроса.
- **Запрос (query)** — входной сигнал, определяющий, что именно нужно отранжировать.
- **Релевантность** — степень соответствия объекта запросу.
- **Score** — числовая оценка релевантности, по которой сортируется список.
- **Двухэтапный поиск** — быстрый отбор кандидатов + точное ранжирование.
