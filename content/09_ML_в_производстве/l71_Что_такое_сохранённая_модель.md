# 71. Что такое сохранённая модель

## Зачем это нужно

Обучение модели занимает время и ресурсы. После обучения нужно сохранить результат, чтобы использовать модель без повторного обучения. Понимание того, что именно сохраняется и как загружается, обязательно для работы с production ML.

## Простая интуиция

Представьте повара, который записал все свои рецепты в книгу. Открыв книгу, любой повар может приготовить те же блюда без переобучения. Сохранённая модель — это «книга рецептов»: все параметры, которые модель «выучила», записаны и могут быть загружены позже.

## Что это такое

- **Сериализация** — процесс сохранения объекта Python (модели) в файл
- **pickle** — стандартная библиотека Python для сериализации; простой, но не самый безопасный
- **joblib** — рекомендуется для sklearn: эффективно сохраняет numpy массивы
- **ONNX** — открытый формат для обмена моделями между фреймворками (sklearn → ONNX → C++/Java)
- Что сохраняется: параметры модели (веса, пороги), гиперпараметры, preprocessing параметры

## Как это работает

```python
import joblib

# Сохранить обученный pipeline (scaler + модель вместе)
joblib.dump(pipeline, "model_v1.pkl")

# Загрузить и сразу использовать — без повторного .fit()
loaded = joblib.load("model_v1.pkl")
predictions = loaded.predict(new_data)

# Версионируйте файлы и фиксируйте версию sklearn:
# model_v1.pkl, model_v2.pkl — храните историю
```

## Простой пример

ML-команда обучает модель предсказания оттока клиентов: Pipeline из imputer, scaler и Random Forest. Обучение на 50 000 клиентов занимает 4 минуты, происходит каждое воскресенье. Результат сохраняется как `churn_model_2024_w42.pkl` (2.3 МБ). Inference-сервер загружает файл при старте (0.8 секунды), затем обрабатывает запросы. Когда sklearn обновили до 1.4, старый файл вызвал предупреждение при загрузке — пришлось переобучить и пересохранить. Теперь в файле `requirements.txt` фиксирована `scikit-learn==1.4.2`.

## Краткий итог

- Используйте joblib.dump() для сохранения sklearn моделей
- Сохраняйте Pipeline (включает scaler + модель) как единый объект
- Версионируйте модели и фиксируйте версию sklearn
- Проверяйте корректность загруженной модели на нескольких примерах

## Ключевые термины урока

- **Сериализация** — сохранение обученной модели на диск для последующей загрузки.
- **joblib** — рекомендуемая библиотека для сохранения sklearn-моделей.
- **pickle** — стандартный Python-модуль для сериализации объектов.
- **ONNX** — универсальный формат моделей для переноса между фреймворками.
- **Версионирование моделей** — фиксация версии модели и библиотек для воспроизводимости.
