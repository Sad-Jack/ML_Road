# 03. Типы данных в Pandas

## Зачем это нужно

Не все колонки можно сразу отдать модели. Строки, пропуски, даты — это всё требует обработки. Понять типы данных заранее — значит избежать ошибок при обучении.

## Простая интуиция

Pandas хранит данные с указанием типа: это число, строка или пропуск. Модель чаще всего ожидает числа. Строка `"Berlin"` — не число. Пустое значение `NaN` — тоже не число. Поэтому до обучения нужно проверить, что лежит в каждой колонке.

## Что это такое

Основные типы в pandas:
- `int64`, `float64` — числа (модели работают напрямую)
- `object` — строки или смешанные данные (нужна обработка)
- `bool` — True/False
- `NaN` / `None` — пропуски (многие модели не умеют их обрабатывать)

Важно: число `25` и строка `"25"` — разные вещи. Pandas может хранить их по-разному.

## Как это работает

```python
import pandas as pd

df = pd.DataFrame({
    "age": [25, 31, None],
    "city": ["Berlin", "Hamburg", "Berlin"],
    "salary": [70000, 90000, 120000]
})

df.dtypes        # типы колонок
df.info()        # общая информация + сколько непустых значений
df.isna().sum()  # сколько пропусков в каждой колонке
```

## Простой пример

Запускаем `df.dtypes` и видим: `age` — float64 (из-за None), `city` — object, `salary` — int64. Отсюда сразу видно: `city` нужно кодировать, `age` нужно обработать пропуск.

## Краткий итог

- Числа (`int`, `float`) — модели работают с ними напрямую.
- Строки (`object`) — требуют кодирования (уроки 37–40).
- Игнорировать пропуски нельзя: большинство алгоритмов sklearn не примут NaN и упадут с ошибкой.
- Три команды диагностики за пять секунд покажут, что нужно исправить перед обучением.
- Если таблица выглядит нормально, это ещё не значит, что данные готовы для модели.

## Ключевые термины урока

- **dtype** — тип данных колонки (`int64`, `float64`, `object`); влияет на дальнейшую обработку.
- **NaN** — обозначение пропущенного значения в pandas.
- **df.dtypes** — вывод типов данных всех колонок DataFrame.
- **df.info()** — сводная информация: типы, число non-null значений, память.
- **df.isnull().sum()** — подсчёт числа пропусков в каждой колонке.
