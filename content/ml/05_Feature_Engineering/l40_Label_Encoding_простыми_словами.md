# 39. Label Encoding простыми словами

## Зачем это нужно

Мы уже знаем (урок 37), что категориальные признаки нужно кодировать. **Label Encoding** — самый простой способ: каждой категории присваивается число. Простой, но с важным риском ложного порядка.

---

## Простая интуиция

Label Encoding просто заменяет каждую категорию на число.

Было:

```text
city = Berlin
city = Hamburg
city = Munich
```

Стало:

```text
Berlin  -> 0
Hamburg -> 1
Munich  -> 2
```

То есть каждая категория получает свой числовой код.

---

## Что это такое

**Label Encoding** — это способ кодирования категорий, при котором каждому уникальному значению присваивается отдельное число.

Пример:

| city    | city_encoded |
| ------- | -----------: |
| Berlin  |            0 |
| Hamburg |            1 |
| Munich  |            2 |
| Berlin  |            0 |

Модель уже получает не строки, а числа.

Но здесь есть важный нюанс: эти числа не всегда имеют смысл как порядок.

---

## Как это работает

### 1. Берём категориальный признак

Например:

```text
city = Berlin / Hamburg / Munich
```

---

### 2. Находим все уникальные категории

```text
Berlin
Hamburg
Munich
```

---

### 3. Каждой категории даём номер

```text
Berlin  -> 0
Hamburg -> 1
Munich  -> 2
```

---

### 4. Заменяем строки на числа

Было:

| city    |
| ------- |
| Berlin  |
| Munich  |
| Hamburg |
| Berlin  |

Стало:

| city_encoded |
| -----------: |
|            0 |
|            2 |
|            1 |
|            0 |

---

### Важный момент

Label Encoding может создать ложный порядок.

Например:

```text
Berlin = 0
Hamburg = 1
Munich = 2
```

Модель может воспринять это так, будто:

```text
Munich > Hamburg > Berlin
```

Но у городов нет такого смысла.
Город с кодом `2` не "больше" города с кодом `1`.

Поэтому Label Encoding не всегда подходит для обычных неупорядоченных категорий.

---

### Когда Label Encoding подходит

Label Encoding можно использовать, когда категории имеют естественный порядок.

Например:

```text
low < medium < high
```

Или:

```text
junior < middle < senior
```

Тогда числа отражают реальный смысл:

| level  | encoded |
| ------ | ------: |
| junior |       0 |
| middle |       1 |
| senior |       2 |

Здесь порядок важен и понятен.

---

### Label Encoding vs One-Hot Encoding

| Способ           | Что делает                                     | Когда подходит                                                         |
| ---------------- | ---------------------------------------------- | ---------------------------------------------------------------------- |
| Label Encoding   | Заменяет категорию одним числом                | Когда есть порядок или модель умеет нормально работать с такими кодами |
| One-Hot Encoding | Создаёт отдельную колонку для каждой категории | Когда порядка между категориями нет                                    |

Пример для `city`:

### Label Encoding

| city    | encoded |
| ------- | ------: |
| Berlin  |       0 |
| Hamburg |       1 |
| Munich  |       2 |

Может появиться ложный порядок.

### One-Hot Encoding

| city_Berlin | city_Hamburg | city_Munich |
| ----------: | -----------: | ----------: |
|           1 |            0 |           0 |
|           0 |            1 |           0 |
|           0 |            0 |           1 |

Порядок не создаётся.

---

## Простой пример

```python
from sklearn.preprocessing import LabelEncoder

cities = ["Berlin", "Hamburg", "Munich", "Berlin"]

encoder = LabelEncoder()
cities_encoded = encoder.fit_transform(cities)
print(cities_encoded)
```

Результат:

```text
[0 1 2 0]
```

`LabelEncoder` нашёл уникальные категории и заменил их числами.

---

### Важное уточнение

В `sklearn` `LabelEncoder` чаще используют для кодирования `y`, то есть target.

Например:

```text
cat -> 0
dog -> 1
bird -> 2
```

Для признаков `X` чаще используют другие инструменты, например:

* `OrdinalEncoder`;
* `OneHotEncoder`.

Смысл похожий, но важно не путать, где именно применяется кодирование.

## Краткий итог

* Label Encoding заменяет категории числами.
* Каждая уникальная категория получает свой код.
* Это простой способ превратить строки в числовой вид.
* Главный риск — ложный порядок между категориями.
* Для категорий с естественным порядком такой подход может быть уместен.
* Для категорий без порядка часто безопаснее использовать One-Hot Encoding.

## Ключевые термины урока

- **Label Encoding** — один числовой признак вместо N бинарных колонок. Экономит память, но создаёт риск ложного порядка — приемлем только при наличии реального порядка или для деревьев.
- **Числовой код** — порядок присвоения кодов случаен (алфавитный по умолчанию). Berlin=0, Hamburg=1, Munich=2 — это не «рейтинг городов», хотя линейная модель может именно так это воспринять.
- **Ложный порядок** — главная ловушка. Линейные модели и нейросети интерпретируют числа как расстояния, поэтому код «2» воспринимается как «в два раза больше, чем код 1». Для деревьев это менее критично: они разбивают по порогу, а не по расстоянию.
- **LabelEncoder (sklearn)** — предназначен для кодирования `y` (target), а не признаков. Для признаков используйте `OrdinalEncoder` — он принимает 2D-массив и поддерживает явный порядок.
