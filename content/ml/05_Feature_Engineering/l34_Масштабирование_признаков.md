# 33. Масштабирование признаков

## Зачем это нужно

Некоторые модели реагируют на масштаб чисел: если `age = 25` и `salary = 150000`, модель может уделять больше внимания зарплате просто из-за большей числовой величины, а не потому что она важнее. Масштабирование устраняет эту проблему.

## Простая интуиция

Представь двух учителей: один оценивает по шкале 1–5, другой — по шкале 1–100. Если сложить их оценки без нормализации, второй будет доминировать только за счёт шкалы. Масштабирование приводит признаки к одному диапазону, чтобы модель оценивала их по реальной важности.

## Что это такое

**Масштабирование (scaling)** — приведение числовых признаков к примерно одинаковому диапазону. Смысл данных не меняется, меняется только шкала.

Когда нужно: линейные модели (`LinearRegression`, `LogisticRegression`), `k-NN`, `SVM`.

Когда обычно не нужно: деревья (`DecisionTree`, `RandomForest`, `GradientBoosting`) — они разбивают данные по порогам и не зависят от масштаба.

## Как это работает

```python
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # учим на train
X_test_scaled = scaler.transform(X_test)        # применяем к test
```

Важно: `fit` только на train. К test — только `transform`. Если сделать `fit` на всём датасете до split — информация из test попадёт в train (утечка данных).

## Простой пример

`age` колеблется от 18 до 70, `salary` — от 30 000 до 300 000. После StandardScaler оба признака будут примерно в диапазоне [-3, 3], и модель будет оценивать их по реальной информативности.

## Краткий итог

- Масштабирование нужно для линейных моделей, k-NN, SVM.
- Для деревьев и бустинга обычно не нужно.
- Scaler учится только на train: это гарантирует, что статистика test не повлияет на обучение.
- Применять fit к test — конкретная форма data leakage: статистика test просачивается в обучение через scaler.
- Масштабирование до split — ошибка: информация из test попадает в train.

## Ключевые термины урока

- **Масштабирование (scaling)** — нужно для линейных моделей, k-NN, SVM; для деревьев и бустинга обычно бесполезно, так как они разбивают данные по порогам, а не по расстояниям.
- **StandardScaler** — формула: `z = (x - mean) / std`. Результат: среднее=0, std=1. `mean` и `std` берутся **только из train** — не из всего датасета.
- **fit_transform** — вызывается только на train: вычисляет `mean`/`std` и сразу применяет преобразование. Применять `fit_transform` к test — data leakage.
- **transform** — применяет **уже вычисленные** `mean`/`std` к test. Никакой новой статистики не считает.
