# 34. Как выбирать признаки для X

## Зачем это нужно

Не каждая колонка в таблице должна попасть в X. Неправильный выбор признаков — одна из главных причин плохой модели: лишний мусор мешает учиться, а утечка данных делает результаты нечестными.

## Простая интуиция

В таблице обычно есть четыре типа колонок: полезные признаки, target, технический мусор и утечка данных. Задача — оставить только первый тип.

## Что это такое

**Полезные признаки** — описывают объект и реально связаны с тем, что мы предсказываем. Например: `age`, `city`, `orders_count`.

**Target** — то, что предсказываем. Нельзя включать в X.

**Технический мусор** — `id`, `row_number`, `uuid`. Не описывают объект по смыслу — модель может найти ложные связи с target.

**Утечка данных** — признак, который становится известен только после события, которое мы предсказываем. Например, `purchase_date` для задачи «купит ли клиент».

## Как это работает

Главный вопрос перед добавлением колонки в X:

> Была бы эта информация известна в момент предсказания?

Если нет — не включаем. Дополнительная проверка: эта колонка реально описывает объект по смыслу?

```python
X = df[["age", "city"]]   # полезные признаки
y = df["bought"]          # target
# client_id и purchase_date — не берём
```

## Простой пример

Задача: предсказать, купит ли клиент. Таблица содержит: `client_id`, `age`, `city`, `orders_count`, `purchase_date`, `bought`.

В X берём: `age`, `city`, `orders_count`. Не берём: `client_id` (мусор), `purchase_date` (утечка — дата появляется только после покупки), `bought` (это target).

## Практика

1. Какой главный вопрос нужно задать перед тем, как добавить колонку в X?
2. В таблице есть колонки: `client_id`, `age`, `city`, `purchase_date`, `bought`. Задача — предсказать, купит ли клиент. Какие колонки войдут в X, а какие нет и почему?
3. Коллега добавил признак `delivery_confirmed = 1/0` в модель для предсказания доставки. Почему это может быть утечкой?
4. В уроке 6 мы узнали, что колонка с типом `object` требует обработки. Как знание типов помогает принимать решение о том, включать ли колонку в X?
5. У вас DataFrame с `age`, `city`, `orders_count`, `cancellation_date`, `churn`. Задача — предсказать churn. Опишите, какие колонки войдут в X, что станет y, и что нельзя использовать.

## Краткий итог

- В X идут только полезные признаки, известные в момент предсказания.
- Случайно включить target в X — самая грубая форма leakage: модель «знает ответ» до предсказания.
- `id`, `uuid` — технический мусор, обычно не нужен.
- Утечка: признак появляется после события → нельзя использовать.
- Главный вопрос: «Была бы эта информация известна в момент предсказания?»

## Ключевые термины урока

- **Полезные признаки** — те, что описывают объект и известны в момент предсказания.
- **Target leakage (утечка данных)** — признак, содержащий информацию о целевой переменной, недоступную при реальном предсказании.
- **Технический мусор** — колонки `id`, `uuid`, `row_number` — обычно не несут полезной информации для модели.
- **Data Leakage** — ситуация, когда признак содержит данные, которых не должно быть в момент предсказания.
