# 39. One-Hot Encoding простыми словами

## Зачем это нужно

Из урока «Кодирование категориальных признаков» мы знаем, что категориальные признаки нужно кодировать. Если просто заменить города числами `0/1/2`, модель может решить, что один город «в два раза больше» другого — это неверно. **One-Hot Encoding** — один из основных способов: подходит для категорий без естественного порядка и не создаёт ложной шкалы.

---

## Простая интуиция

Представьте, что на входе в театр у каждого зрителя спрашивают место: ложа, партер или балкон. Вместо того чтобы записывать слово, вам дают три кнопки — и вы нажимаете ровно одну. Одна горит, две погашены. Именно так работает One-Hot Encoding: одна «лампочка» загорается для нужной категории, остальные остаются нулями.

## Что это такое

**One-Hot Encoding** — это способ кодирования категорий, где каждая категория превращается в отдельную колонку.

Если объект относится к категории — ставим `1`.
Если не относится — ставим `0`.

Было:

| city    |
| ------- |
| Berlin  |
| Hamburg |
| Munich  |
| Berlin  |

Стало:

| city_Berlin | city_Hamburg | city_Munich |
| ----------: | -----------: | ----------: |
|           1 |            0 |           0 |
|           0 |            1 |           0 |
|           0 |            0 |           1 |
|           1 |            0 |           0 |

## Простой пример

Есть данные:

| user_id | device  | bought |
| ------: | ------- | -----: |
|       1 | mobile  |      1 |
|       2 | desktop |      0 |
|       3 | mobile  |      1 |
|       4 | tablet  |      0 |

`device` — категориальный признак.
`bought` — target.

После One-Hot Encoding:

| user_id | device_mobile | device_desktop | device_tablet | bought |
| ------: | ------------: | -------------: | ------------: | -----: |
|       1 |             1 |              0 |             0 |      1 |
|       2 |             0 |              1 |             0 |      0 |
|       3 |             1 |              0 |             0 |      1 |
|       4 |             0 |              0 |             1 |      0 |

Теперь модель получает числа и может использовать `device` как признак.

Обратите внимание: колонка `device_tablet` технически лишняя. Если `device_mobile=0` и `device_desktop=0`, то объект — точно tablet. Одну колонку всегда можно вычислить из остальных. Для линейных моделей это создаёт мультиколлинеарность — поэтому sklearn по умолчанию предлагает параметр `drop='first'`, убирающий первую колонку. Для деревьев и бустинга это не важно.

---

### Когда One-Hot Encoding подходит

One-Hot Encoding хорошо подходит, если:

* категорий не слишком много;
* между категориями нет естественного порядка;
* модель не умеет работать со строками напрямую;
* нужно сохранить смысл "это просто разные варианты".

Примеры хороших признаков:

| Признак        | Категории                 |
| -------------- | ------------------------- |
| device         | mobile / desktop / tablet |
| payment_method | card / cash / transfer    |
| color          | red / blue / green        |
| city           | Berlin / Hamburg / Munich |

---

### Когда One-Hot Encoding может быть неудобен

Если категорий очень много, One-Hot Encoding создаст слишком много новых колонок.

Например:

```text
city = 10 000 разных городов
```

После кодирования может появиться почти `10 000` новых признаков.

Это может:

* раздуть таблицу;
* замедлить обучение;
* увеличить расход памяти;
* сделать данные сложнее для обработки.

В таких случаях могут понадобиться другие способы кодирования.

---

### Важный момент про новые категории

Модель может столкнуться с категорией, которой не было на обучении.

Например, на train были:

```text
mobile
desktop
```

А в новых данных пришло:

```text
tablet
```

Если это не обработать заранее, могут возникнуть проблемы.

Обычно для таких случаев используют настройки encoder'а или отдельную категорию:

```text
unknown
```

Смысл: если категория неизвестна, не ломаем обработку, а относим её к "другим".

## Краткий итог

- One-Hot Encoding — для категорий без порядка: не создаёт ложной шкалы.
- При высокой кардинальности (тысячи значений) даёт curse of dimensionality — нужен другой метод.
- Для линейных моделей используйте `drop='first'`, чтобы убрать мультиколлинеарность. Для деревьев — не нужно.
- `OneHotEncoder` из sklearn надёжнее `pd.get_dummies` в production: запоминает категории на fit и не ломается на новых значениях.

## Ключевые термины урока

- **One-Hot Encoding** — единственный 1 в строке, остальное — 0. Ключевое свойство: никакая категория не «больше» другой — модель не может вывести ложный порядок.
- **Бинарные колонки** — в отличие от числовых признаков, несут только сигнал «принадлежит / не принадлежит». Модель обрабатывает их как независимые признаки.
- **OneHotEncoder (sklearn)** — в отличие от `get_dummies`, **умеет работать в Pipeline**: запоминает категории на `fit` и корректно обрабатывает новые на `transform` через параметр `handle_unknown`.
- **get_dummies (pandas)** — удобен для быстрого EDA, но не сохраняет состояние: при новых категориях в production структура колонок изменится и модель сломается.
