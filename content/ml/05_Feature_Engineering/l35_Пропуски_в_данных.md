# 35. Пропуски в данных

## Зачем это нужно

В реальных данных часто чего-то не хватает:

* человек не указал возраст;
* у товара нет рейтинга;
* у квартиры не заполнен этаж;
* у пользователя нет истории покупок;
* часть данных потерялась при сборе.

Для модели это проблема: многие алгоритмы не умеют работать с пустыми значениями напрямую.
Поэтому перед обучением нужно решить, что делать с пропусками.

---

## Простая интуиция

Пропуск — это место, где значение должно быть, но его нет.

Например:

```text
age = ?
city = Berlin
orders_count = 4
```

Модель не может нормально использовать `?`.
Нужно либо:

* убрать этот объект;
* заменить пропуск каким-то значением;
* отдельно показать модели, что значение было пропущено.

---

## Что это такое

**Пропуск в данных** — это отсутствующее значение в таблице.

В Python и Pandas такие значения часто отображаются как:

```text
NaN
None
null
```

Пример:

| age | city    | bought |
| --: | ------- | -----: |
|  25 | Berlin  |      1 |
| NaN | Hamburg |      0 |
|  31 | NaN     |      1 |

Здесь пропущены:

* `age` во второй строке;
* `city` в третьей строке.

---

## Как это работает

Пропуски могут появиться по разным причинам:

| Причина                  | Пример                                  |
| ------------------------ | --------------------------------------- |
| Человек не заполнил поле | не указал возраст                       |
| Данных ещё нет           | пользователь не сделал ни одной покупки |
| Ошибка сбора данных      | значение не записалось                  |
| Поле не подходит объекту | у нового товара ещё нет рейтинга        |

Важно понимать причину, потому что от неё зависит способ обработки.

---

### Как работать с пропусками

#### 1. Удалить строки с пропусками

Самый простой вариант — удалить объекты, где чего-то не хватает.

```text
если строка содержит NaN -> удалить строку
```

Это подходит, если:

* пропусков мало;
* удаление не испортит датасет;
* строки с пропусками не слишком важны.

Минус: можно потерять полезные данные.

---

### 2. Заполнить числовые пропуски

Если пропуск в числовой колонке, его можно заменить:

* средним значением;
* медианой;
* нулём;
* специальным значением.

Пример:

| age |
| --: |
|  25 |
| NaN |
|  31 |

Можно заменить `NaN` средним возрастом:

| age |
| --: |
|  25 |
|  28 |
|  31 |

Часто для чисел используют **медиану**, потому что она меньше зависит от выбросов.

---

### 3. Заполнить категориальные пропуски

Если пропуск в категории, можно заменить его:

* самым частым значением;
* отдельной категорией `unknown`.

Пример:

| city    |
| ------- |
| Berlin  |
| NaN     |
| Hamburg |

После обработки:

| city    |
| ------- |
| Berlin  |
| unknown |
| Hamburg |

Так модель видит, что город не был указан.

---

### 4. Добавить признак "был ли пропуск"

Иногда сам факт пропуска несёт полезный сигнал.

Например, если пользователь не указал доход, это может быть связано с его поведением.

Можно добавить отдельную колонку:

| income | income_was_missing |
| -----: | -----------------: |
|  50000 |                  0 |
|      0 |                  1 |
|  70000 |                  0 |

`income_was_missing = 1` показывает, что исходное значение было пропущено.

---

### Важный порядок обработки

Если мы заполняем пропуски средним, медианой или самым частым значением, это преобразование нужно считать **только на train**.

Правильно:

```text
train -> посчитали значение для заполнения
validation/test -> применили это же значение
```

Нельзя сначала заполнить пропуски на всём датасете, а потом делать split.
Иначе информация из validation/test может попасть в train.

---

## Простой пример

Есть колонка `orders_count`:

| orders_count |
| -----------: |
|            4 |
|          NaN |
|            2 |
|           10 |

Варианты обработки:

### Вариант 1 — заполнить нулём

Если пропуск означает, что заказов не было:

```text
NaN -> 0
```

### Вариант 2 — заполнить медианой

Если значение просто неизвестно:

```text
NaN -> median
```

### Вариант 3 — добавить флаг пропуска

Если сам факт отсутствия данных может быть важен:

```text
orders_count_was_missing = 1
```

Нет одного правильного способа для всех случаев.
Нужно смотреть на смысл данных.

## Краткий итог

* Пропуски — обычная часть реальных данных.
* Их нужно обработать до обучения модели.
* Строки с пропусками можно удалять, но не всегда.
* Числовые пропуски часто заполняют медианой или средним.
* Категориальные пропуски можно заменить на `unknown`.
* Иногда полезно добавить отдельный признак, который показывает факт пропуска.
* Значения для заполнения нужно считать на train, а не на всём датасете.

## Ключевые термины урока

- **Пропуск (NaN)** — отсутствующее значение в данных.
- **Импутация** — заполнение пропусков числом (медианой, средним, константой).
- **SimpleImputer** — sklearn-класс для заполнения пропусков.
- **Правило train-only** — imputer (как и scaler) обучается только на train, применяется к test.
