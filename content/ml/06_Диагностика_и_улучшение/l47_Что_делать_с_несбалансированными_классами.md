# 47. Что делать с несбалансированными классами

## Зачем это нужно

Зная, что дисбаланс есть, нужно уметь с ним бороться. Существует несколько практичных техник, которые работают в разных ситуациях. Правильный выбор метода может превратить бесполезную модель в рабочую без сбора новых данных.

## Простая интуиция

Представьте, что у вас 1000 фото собак и 10 фото кошек для обучения классификатора. Вы можете либо скопировать фото кошек несколько раз (oversampling), либо случайно выбросить часть фото собак (undersampling), либо сказать алгоритму: «ошибка на кошке в 100 раз дороже ошибки на собаке» (class_weight).

## Что это такое

- **Oversampling** — увеличение minority class: дублирование примеров или генерация синтетических (SMOTE)
- **Undersampling** — уменьшение majority class: случайное удаление примеров
- **class_weight** — параметр в sklearn, увеличивающий штраф за ошибки на редком классе
- **SMOTE** (Synthetic Minority Over-sampling Technique) — создаёт новые синтетические примеры путём интерполяции
- **Threshold tuning** — изменение порога классификации (по умолчанию 0.5) для баланса precision/recall

## Как это работает

```python
from sklearn.linear_model import LogisticRegression

# Метод 1: class_weight="balanced" — первое что пробуем, ничего не меняет в данных
model = LogisticRegression(class_weight="balanced")
model.fit(X_train, y_train)

# Метод 2: порог классификации (по умолчанию 0.5)
probs = model.predict_proba(X_test)[:, 1]
y_pred = (probs >= 0.3).astype(int)  # снижаем порог → больше recall, меньше precision

# Метод 3: SMOTE — генерация синтетических примеров (pip install imbalanced-learn)
# from imblearn.over_sampling import SMOTE
# X_res, y_res = SMOTE().fit_resample(X_train, y_train)
```

## Простой пример

Медицинская система диагностики рака кожи: 9 500 снимков здоровой кожи и 500 снимков с меланомой. Первый шаг — добавить `class_weight="balanced"`: recall для меланомы вырастает с 12% до 61% без изменения данных. Затем снижают порог классификации с 0.5 до 0.3 — recall достигает 78%, но precision падает до 45%. В медицине это приемлемо: лучше ложная тревога, чем пропущенный рак. Если этого мало — применяют SMOTE для генерации 1 000 синтетических снимков меланомы.

## Краткий итог

- Первый шаг: попробуйте class_weight="balanced" — просто и эффективно
- SMOTE создаёт синтетические примеры и работает лучше при очень малом minority class
- Undersampling быстр, но теряет данные
- Порог классификации позволяет гибко управлять балансом precision/recall

## Ключевые термины урока

- **Oversampling** — увеличение доли minority class за счёт дублирования или генерации примеров.
- **Undersampling** — уменьшение доли majority class за счёт удаления примеров.
- **class_weight="balanced"** — автоматический учёт дисбаланса в функции потерь модели.
- **SMOTE** — генерация синтетических примеров minority class по интерполяции.
- **Threshold tuning** — подбор порога вероятности для улучшения recall или precision.
