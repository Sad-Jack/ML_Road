# 44. Почему модель ошибается

## Зачем это нужно

Понимание причин ошибок модели — первый шаг к её улучшению. Без диагностики разработчик действует вслепую: меняет архитектуру, добавляет данные, подбирает параметры — и не знает, поможет ли это. Умение определить источник проблемы экономит недели работы.

## Простая интуиция

Представьте повара, который учится готовить. Он может пересолить блюдо (неправильные данные), готовить только одно блюдо, игнорируя всё остальное (переобучение), или использовать не те ингредиенты вовсе (неправильные признаки). Каждая проблема требует своего решения.

## Что это такое

Модели ошибаются по нескольким типичным причинам:

- **Underfitting (недообучение)** — модель слишком простая, не улавливает закономерности даже в тренировочных данных
- **Overfitting (переобучение)** — модель запомнила обучающую выборку, но не обобщает на новые данные
- **Неправильные признаки** — в данных нет информации, нужной для предсказания (или нужные признаки отсутствуют)
- **Плохие данные** — шум, ошибки разметки, выбросы, несбалансированные классы
- **Утечка данных (data leakage)** — в тренировке случайно использована информация из будущего или из теста

## Как это работает

```python
# Паттерн диагностики: сравниваем метрику на train и test
model.fit(X_train, y_train)
train_score = model.score(X_train, y_train)
test_score  = model.score(X_test, y_test)

# Недообучение:  train=0.61, test=0.59  → оба низкие, модель слишком проста
# Переобучение:  train=0.99, test=0.71  → train >> test, модель запомнила
# Норма:         train=0.88, test=0.85  → близкие значения, хорошая модель
```

## Простой пример

Компания обучает модель предсказания кредитного дефолта на 10 000 клиентах. Простая логистическая регрессия: train accuracy 62%, test accuracy 61% — оба показателя низкие, модель не улавливает паттерны. Глубокое дерево без ограничений: train accuracy 99%, test accuracy 68% — явное переобучение. Дерево с max_depth=5: train 85%, test 83% — разумный баланс. Диагноз поставлен за 3 минуты, без перебора алгоритмов вслепую.

## Краткий итог

- Модели ошибаются по нескольким причинам: недообучение, переобучение, плохие признаки, плохие данные
- Сравнение метрик на train и test — главный диагностический инструмент
- Переобучение: train >> test; Недообучение: оба низкие
- Диагностика должна предшествовать любым улучшениям модели

## Ключевые термины урока

- **Underfitting (недообучение)** — модель слишком проста: плохие метрики и на train, и на test.
- **Overfitting (переобучение)** — модель слишком сложна: train отличный, test плохой.
- **Неправильные признаки** — когда X не содержит нужной информации для предсказания.
- **Диагностический разрыв** — разница train/test метрик как главный индикатор типа проблемы.
