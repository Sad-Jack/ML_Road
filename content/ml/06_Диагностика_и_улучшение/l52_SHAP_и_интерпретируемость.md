# 50.5. SHAP и интерпретируемость

## Зачем это нужно

Модель выдала отказ в кредите. Клиент спрашивает: «Почему?». Compliance требует объяснение по GDPR. Менеджер хочет убедиться, что модель не дискриминирует по полу.

Accuracy 91% — это хорошо. Но если ты не можешь объяснить, почему модель приняла конкретное решение, ты не можешь ей доверять, не можешь её дебаггить и не можешь использовать в регулируемых отраслях.

## Простая интуиция

Представь, что модель — это судья. Feature importance говорит: «Этот судья в целом больше всего обращает внимание на возраст и доход подозреваемого». Полезно, но это статистика по всем делам.

SHAP говорит: «Вот конкретное дело Иванова. Его возраст добавил +15 к приговору, доход снизил на −8, отсутствие судимости снизило на −20». Это объяснение конкретного решения, а не среднего.

## Что это такое

**Интерпретируемость** — способность объяснить, почему модель приняла то или иное решение.

**Feature importance** — глобальная метрика: какие признаки в среднем влияют на предсказания модели. Встроена в деревья и градиентный бустинг.

**SHAP (SHapley Additive exPlanations)** — метод, который для каждого отдельного предсказания показывает вклад каждого признака: насколько он сдвинул результат вверх или вниз относительно среднего предсказания модели.

Идея пришла из **теории кооперативных игр**: каждый признак — «игрок» в команде. SHAP вычисляет справедливый вклад каждого игрока, перебирая все возможные «коалиции» (подмножества признаков) — это математически обоснованный вклад, а не эвристика.

SHAP работает с любой моделью — от линейной регрессии до XGBoost и нейросети.

## Как это работает

### SHAP-значение

Для каждого объекта и каждого признака SHAP вычисляет: «Насколько изменилось бы предсказание, если бы у этого объекта не было этого признака?»

Результат — число со знаком:
- **положительное** — признак увеличил предсказание (сдвинул к «да», «выше», «1»)
- **отрицательное** — признак уменьшил предсказание (сдвинул к «нет», «ниже», «0»)

Сумма всех SHAP-значений для одного объекта равна разнице между его предсказанием и средним предсказанием по всему датасету.

### Два уровня объяснений

**Локальное объяснение** — почему модель приняла конкретное решение для конкретного объекта. Пример для заявки на кредит:

```
Базовое предсказание (среднее): 0.42

age = 52         → −0.15  (возраст снижает шанс)
income = 180k    → +0.28  (высокий доход повышает)
debt_ratio = 0.6 → −0.19  (высокий долг снижает)
credit_hist = 5y → +0.11  (хорошая история повышает)

Итоговое предсказание: 0.47
```

**Глобальное объяснение** — на большой выборке SHAP показывает, какие признаки в целом важны для модели (аналог feature importance, но честнее).

### Что ещё можно обнаружить

SHAP помогает поймать неожиданные паттерны:
- модель использует признак «почтовый индекс» как прокси для этнического происхождения
- модель опирается на признак, который в production недоступен
- данные утекают через признак, который создан из target

## Простой пример

Банк обучил модель оценки кредитного риска. Accuracy на тесте — 88%. Compliance-офицер спрашивает: «Не используете ли вы пол клиента как признак?»

Напрямую — нет, колонка `gender` удалена. Но SHAP-анализ показал: признак `job_type = "maternity_leave"` имеет большой отрицательный вклад. Это де-факто гендерный признак — непрямая дискриминация. Без SHAP это не заметили бы.

После удаления этого признака accuracy упала до 86%, но модель перестала дискриминировать. Compliance принял.

## Практика

1. В чём разница между feature importance и SHAP? Почему feature importance не объясняет конкретное предсказание?
2. SHAP-значение признака `age` для конкретного клиента равно −0.12. Что это означает? В какую сторону признак сдвинул предсказание?
3. Модель отказала в кредите. SHAP показывает: `income` → +0.18, `debt_ratio` → −0.31, `employment_years` → −0.05. Какой признак сильнее всего повлиял на отказ? Как объяснить это клиенту?
4. Почему интерпретируемость важна не только для «объяснения клиенту», но и для отладки самой модели?
5. Мини-кейс: модель предсказывает отток сотрудников. Глобальный SHAP показывает, что признак `employee_id` входит в топ-3 важных. Это подозрительно. Что это может означать и как исправить?

## Краткий итог

Интерпретируемость — это не опция, а требование для моделей, влияющих на людей. Feature importance отвечает на вопрос «что важно в среднем», SHAP отвечает на вопрос «почему именно это решение для этого объекта». SHAP помогает не только объяснять, но и находить баги: утечки данных, косвенную дискриминацию, использование недоступных признаков.

## Ключевые термины урока

- **Интерпретируемость** — требование для моделей в медицине, финансах, HR: без объяснений нельзя пройти compliance, обнаружить дискриминацию или найти баг типа «модель использует недоступный признак».
- **Feature importance** — глобальная статистика по всему датасету; не скажет, почему отказано конкретному клиенту. В отличие от SHAP, не имеет знака и не учитывает контекст объекта.
- **SHAP** — сумма всех SHAP-значений объекта = его предсказание минус среднее по датасету; математически обоснован теорией игр Шепли. Работает с любой моделью (не только деревьями).
- **Локальное объяснение** — SHAP-значения для одного объекта; используется для объяснения клиенту и для отладки конкретных предсказаний.
- **Глобальное объяснение** — агрегирование SHAP по всей выборке; честнее feature importance, потому что учитывает направление и взаимодействия признаков.
