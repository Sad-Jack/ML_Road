# 50. Grid Search и Random Search

## Зачем это нужно

Ручной подбор гиперпараметров — это угадывание. Grid Search и Random Search делают этот процесс систематическим и воспроизводимым. Без автоматического поиска практик тратит часы на то, что алгоритм делает за минуты.

## Простая интуиция

Grid Search — это как попробовать каждое блюдо в меню ресторана, чтобы найти любимое. Вы гарантированно найдёте лучшее, но потратите много времени. Random Search — выбрать 10 случайных блюд. Вы с высокой вероятностью найдёте хорошее блюдо намного быстрее.

## Что это такое

- **GridSearchCV** — перебирает все комбинации заданных гиперпараметров, оценивает каждую через кросс-валидацию
- **RandomizedSearchCV** — перебирает случайное подмножество комбинаций из заданного распределения
- **CV** (Cross-Validation) — каждая комбинация оценивается на нескольких фолдах для надёжности
- **param_grid** — словарь с гиперпараметрами и их значениями для перебора
- **best_params_** — лучшая найденная комбинация
- **best_score_** — лучший средний score по кросс-валидации

## Как это работает

**Grid Search** перебирает декартово произведение всех заданных значений. Например, 3 значения `n_estimators` × 3 значения `max_depth` = 9 комбинаций. Каждая оценивается через кросс-валидацию. Итог: `best_params_` — лучшая комбинация, `best_score_` — средний score по фолдам.

**Random Search** не перебирает все комбинации — случайно сэмплирует заданное количество (например, 20). При большом пространстве параметров это даёт почти такой же результат за долю времени.

Оба метода оценивают каждую комбинацию через кросс-валидацию, а не на одном разбиении.

## Простой пример

Команда настраивает Random Forest для предсказания оттока клиентов (8 000 клиентов, 20 признаков). Grid Search с `n_estimators` ∈ {50, 100, 200} и `max_depth` ∈ {3, 5, None}: 9 комбинаций × 5 фолдов = 45 обучений, занимает 4 минуты. Лучшие параметры: `n_estimators=200`, `max_depth=5`, CV score 87.3%. Random Search с 6 параметрами и 4 значениями каждого (1 296 комбинаций) занял бы часы с Grid Search, но Random Search с n_iter=30 находит почти такой же результат за 12 минут.

## Краткий итог

- GridSearch неприменим при большом пространстве параметров: экспоненциальный рост делает полный перебор нереальным
- RandomizedSearch даёт почти такое же качество за долю времени: на практике 100 случайных точек покрывают пространство не хуже
- Оба используют кросс-валидацию для честной оценки каждой комбинации
- После поиска всегда оценивайте финальную модель на отдельной тестовой выборке

## Ключевые термины урока

- **GridSearchCV** — перебирает декартово произведение значений; при 3 параметрах по 4 значения = 64 комбинации × cv_folds обучений. При 5+ параметрах становится нереальным.
- **RandomizedSearchCV** — сэмплирует n_iter случайных комбинаций из распределений; часто 20–50 итераций дают результат, близкий к полному перебору, в разы быстрее.
- **best_params_** — оптимальные гиперпараметры по CV, но финальное качество всегда проверять на отдельном test, который не участвовал в поиске.
- **best_estimator_** — уже обученная модель с best_params_ на всём train; использовать для предсказания, не переобучать заново.
- **CV в поиске** — каждая комбинация оценивается на k фолдах, итоговый score — среднее; это защищает от случайно удачного разбиения.
