# 48. Гиперпараметры модели

## Зачем это нужно

Гиперпараметры — это рычаги управления моделью, которые задаются до обучения. Правильный выбор может превратить слабую модель в сильную. Неправильный — даже мощный алгоритм сделает бесполезным. Понимание того, что такое гиперпараметры и как они влияют на модель, обязательно для любого ML-практика.

## Простая интуиция

Представьте, что модель — это кофемашина. Параметры — это количество молотого кофе, которое она определяет сама в процессе приготовления. Гиперпараметры — это температура воды, давление, время заваривания: вы устанавливаете их вручную до того, как машина начинает работу.

## Что это такое

- **Параметры модели** — то, что модель учит из данных (веса нейросети, коэффициенты линейной регрессии)
- **Гиперпараметры** — то, что задаёте вы до обучения (нельзя выучить из данных напрямую)
- Примеры гиперпараметров:
  -  — максимальная глубина дерева решений
  -  — количество деревьев в Random Forest
  -  — размер шага в градиентном спуске
  -  — сила регуляризации в SVM и логистической регрессии
  -  — количество соседей в kNN

## Как это работает

```python
from sklearn.ensemble import RandomForestClassifier

# Гиперпараметры задаются ДО обучения — не выучиваются из данных
model = RandomForestClassifier(
    n_estimators=100,    # количество деревьев
    max_depth=5,         # глубина каждого дерева (управляет bias-variance)
    min_samples_leaf=2,  # минимум объектов в листе (регуляризация)
    random_state=42
)
model.fit(X_train, y_train)

# Параметры модели (веса деревьев) — обновляются автоматически при .fit()
# Гиперпараметры — задаёте вы и не меняются при обучении
```

## Простой пример

Датасет из 5 000 отзывов на товары для предсказания тональности (позитив/негатив). Дерево решений с `max_depth=2` не улавливает нюансы текста — train accuracy 63%, явное недообучение. Дерево с `max_depth=None` запоминает каждое слово — train 99%, test 71%, переобучение. После перебора значений `max_depth` от 1 до 20 через кросс-валидацию оптимум находится при `max_depth=8`: train 88%, test 86%. Именно гиперпараметр `max_depth` управлял этим балансом.

## Краткий итог

- Гиперпараметры задаются до обучения и не обновляются из данных
- Они управляют сложностью модели и балансом bias-variance
- Подбор вручную неэффективен — нужен систематический поиск
- Оценивать качество гиперпараметров нужно через кросс-валидацию

## Ключевые термины урока

- **Параметры модели** — обновляются автоматически при `fit()` (веса, коэффициенты); их не трогают вручную. В отличие от гиперпараметров, не задаются до обучения.
- **Гиперпараметры** — задаются до `fit()` и фиксируются на всё обучение; подбираются через CV, не через метрику на test. Подбор на test — это data leakage для гиперпараметров.
- **Поиск гиперпараметров** — Grid Search исчерпывающий, Random Search стохастический; оба оценивают каждую комбинацию через кросс-валидацию, а не на одном разбиении.
- **max_depth как пример** — малое значение → высокий bias (underfitting), большое → высокая variance (overfitting); оптимум находится через CV, а не интуитивно.
