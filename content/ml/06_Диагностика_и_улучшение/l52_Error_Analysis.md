# 52. Error Analysis

## Зачем это нужно

Метрики говорят вам, насколько хороша модель. Error Analysis говорит, почему она ошибается и что делать дальше. Без анализа ошибок разработчик не знает, куда направить усилия: собирать данные, менять признаки или улучшать алгоритм.

## Простая интуиция

Представьте учителя, проверяющего контрольные. Он мог бы просто поставить оценки и двигаться дальше. Но хороший учитель смотрит на ошибки: «10 студентов не поняли задачи на дроби, 5 — на проценты». Зная паттерн ошибок, он знает, что объяснить заново. Так же работает error analysis в ML.

## Что это такое

- **Error Analysis** — ручной или автоматический просмотр неправильных предсказаний для поиска паттернов
- **False Positives (FP)** — модель предсказала 1, а было 0 (ложная тревога)
- **False Negatives (FN)** — модель предсказала 0, а было 1 (пропущенный случай)
- **Confusion Matrix** — матрица ошибок, показывающая все 4 типа предсказаний
- **Паттерны ошибок** — категории или характеристики объектов, на которых модель систематически ошибается

## Как это работает

Процесс Error Analysis состоит из нескольких шагов:

1. Собрать все неправильные предсказания — где предсказание модели не совпало с реальным значением
2. Разделить ошибки по типам: False Positives (ложная тревога) и False Negatives (пропущенный случай)
3. Изучить объекты, на которых модель ошиблась — найти общий признак
4. Сформулировать гипотезу: нужны новые данные, новые признаки или другой алгоритм

## Простой пример

Классификатор спама на 2 000 писем: общая accuracy 94%, кажется хорошим результатом. Error Analysis показывает: из 120 ошибок — 90 FN (спам помечен как нормальное письмо). Просмотр этих 90 писем выявляет паттерн: все они с темой «Вы выиграли...» на испанском языке. Модель обучалась только на английских письмах. Решение: добавить 200 испаноязычных спам-примеров. После дообучения FN снизились с 90 до 11.

## Практика

1. Спам-фильтр для корпоративной почты: accuracy = 94%, команда довольна. Error Analysis показывает: из 200 ошибок 160 — FN (реальный спам, пропущенный в inbox). Коллега предлагает: «Нужно больше данных для переобучения». Ты просматриваешь конкретные FN-примеры и видишь: почти все они от отправителей из редких доменов, которых почти нет в train. Что ты предложишь вместо «больше данных» — и как именно Error Analysis привёл тебя к более точному плану, чем просто смотреть на метрику?
2. Вы проанализировали 150 ошибок классификатора изображений: 100 из них — изображения при плохом освещении, остальные 50 — случайный набор разных сцен. Что говорит каждая группа и какие разные действия они требуют?
3. Вы обнаружили, что модель кредитного скоринга систематически ошибается на клиентах старше 70 лет. Что это говорит о данных и модели и каков следующий шаг?
4. Из урока «Метрики классификации» мы знаем, что recall = TP / (TP + FN). В задаче медицинской диагностики, где цена FN несопоставимо выше цены FP, как Error Analysis помогает расставить приоритеты между типами ошибок — и что именно нужно минимизировать?
5. Из урока «Overfitting и Underfitting» мы знаем признаки переобучения. Если Error Analysis показывает, что модель ошибается систематически на одной подгруппе (например, пациентах старше 70), как отличить: это переобучение на остальных данных или нехватка информации именно об этой подгруппе?

## Краткий итог

- Error Analysis — самый прямой путь к следующему улучшению: ошибки подсказывают, чего модель не умеет
- Разделяйте ошибки по типам: False Positives и False Negatives имеют разную цену
- Систематические ошибки на подгруппах — сигнал о нужных новых признаках
- Случайные ошибки без паттернов — возможно, вы достигли предела данных

## Ключевые термины урока

- **Error Analysis** — анализируй не агрегированную метрику, а конкретные ошибки; цель — найти общий признак у ошибочных объектов и превратить его в улучшение (признак, данные, правило).
- **False Positives (FP)** — модель кричит «тревога» там, где её нет; высокая цена FP → нужно повышать precision (порог вверх). В спаме: нормальное письмо попало в спам.
- **False Negatives (FN)** — модель молчит там, где должна сработать; высокая цена FN → нужно повышать recall (порог вниз). В медицине: больной пропущен как здоровый.
- **Паттерны ошибок** — если ошибки систематические (один язык, одна подгруппа, один диапазон значений) — это сигнал нехватки данных или признаков. Случайные ошибки без паттерна — вы достигли предела данных.
