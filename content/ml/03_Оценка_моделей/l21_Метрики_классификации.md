# 21. Метрики классификации

## Зачем это нужно

После обучения модели нужно понять: насколько она хороша и где именно ошибается. Accuracy — одна цифра на все случаи жизни — часто даёт ложное ощущение качества. Для разных задач нужны разные метрики, потому что цена ошибок везде разная.

## Простая интуиция

Представьте суд. Можно ошибиться двумя способами: осудить невиновного или оправдать виновного. Accuracy скажет лишь «в 90% дел приговор правильный». Но судье важно знать: сколько невиновных осуждено (это неприемлемо) и сколько виновных отпущено (это опасно). Это две разные метрики, и обе нужны.

## Что это такое

Из урока «Confusion Matrix» мы знаем четыре типа исходов: TP, TN, FP, FN. На их основе считаются все основные метрики классификации.

**Accuracy** = (TP + TN) / (TP + TN + FP + FN)
Доля всех правильных ответов. Обманчива при несбалансированных классах.

**Precision** = TP / (TP + FP)
Из всех предсказанных «1» — сколько реально «1»? Отвечает на вопрос: «Насколько модели можно доверять, когда она говорит "да"?»

**Recall** = TP / (TP + FN)
Из всех реальных «1» — сколько нашли? Отвечает на вопрос: «Сколько реальных случаев модель не пропустила?»

**F1** = 2 × (Precision × Recall) / (Precision + Recall)
Среднее гармоническое precision и recall. Если одна из двух очень низкая — F1 тоже будет низкой, арифметическое среднее этого бы не показало.

## Как это работает

Выбор метрики определяется ценой каждого типа ошибки:

**Когда важен Recall** — когда опасно пропустить положительный случай. Примеры: диагностика болезни (FN = пропущенный диагноз), обнаружение мошенничества (FN = прошедший fraud). Лучше лишний раз сработать тревогу, чем пропустить.

**Когда важен Precision** — когда дорого стоит ложная тревога. Примеры: спам-фильтр (FP = потерянное важное письмо), рекомендации товаров (FP = нерелевантный результат раздражает). Лучше реже реагировать, но точнее.

**Когда использовать F1** — когда нет явного перекоса в сторону precision или recall, или нужна одна цифра для сравнения нескольких моделей.

**Когда Accuracy не лжёт** — только при сбалансированных классах, где оба класса примерно одинаково представлены в данных.

## Простой пример

Модель проверяет рентгеновские снимки: класс «1» — есть патология, класс «0» — здоров. На val-выборке из 100 пациентов (20 с патологией, 80 здоровых):

| | Предсказал «здоров» | Предсказал «патология» |
|---|---|---|
| **Реально здоров** | 75 (TN) | 5 (FP) |
| **Реально патология** | 8 (FN) | 12 (TP) |

Считаем:
- Accuracy = (75 + 12) / 100 = **0.87** — выглядит неплохо
- Precision = 12 / (12 + 5) = **0.71** — из найденных патологий 71% реальные
- Recall = 12 / (12 + 8) = **0.60** — модель пропустила 40% реальных патологий
- F1 = 2 × (0.71 × 0.60) / (0.71 + 0.60) = **0.65** — оба слабых места отражены в одной цифре

Recall 0.60 — это катастрофа для медицины: каждый второй больной уходит домой без диагноза. Accuracy при этом смотрится хорошо — 87%. Вот почему метрика выбирается под задачу.

## Практика

1. Модель выдаёт Precision = 0.95 и Recall = 0.20. Что это говорит о поведении модели — она слишком «осторожная» или слишком «агрессивная»? Что теряет бизнес в каждом из этих случаев?
2. Вы строите модель для диагностики рака. Что важнее: высокий Precision или высокий Recall? Объясните, что означает каждый тип ошибки в этом контексте.
3. Почему F1 считается через среднее гармоническое, а не арифметическое? Что произойдёт с F1, если Precision = 0.99, а Recall = 0.01?
4. Из урока «Confusion Matrix» мы знаем, что FP и FN — принципиально разные ошибки. Как это знание меняет выбор между двумя моделями для спам-фильтра: модель А (Precision=0.98, Recall=0.60) и модель Б (Precision=0.75, Recall=0.95)? Обоснуйте выбор через цену каждого типа ошибки.

## Краткий итог

- Accuracy обманывает при несбалансированных классах — модель «всегда говорю 0» даёт 95% при нулевой пользе.
- Precision важна, когда дорого ошибиться с «да»: спам-фильтры, рекомендации.
- Recall важен, когда опасно пропустить «да»: болезни, мошенничество, угрозы безопасности.
- F1 — компромисс: одна цифра для сравнения моделей, когда оба направления важны.
- Выбор метрики — это выбор того, какой тип ошибки обходится дороже.

## Ключевые термины урока

- **Accuracy** — обманчива при дисбалансе: 95% точность при классах 95/5 означает, что модель просто говорит «нет» всем, и при этом «права» в 95% случаев.
- **Precision** — «не кричи волк попусту»: из всех предсказанных «1» — сколько реально «1». Низкая Precision = много ложных тревог.
- **Recall** — «не пропусти волка»: из всех реальных «1» — сколько нашли. Низкий Recall = много пропусков.
- **F1** — среднее гармоническое, а не арифметическое: если Precision = 1.0, а Recall = 0.0, то F1 = 0, а не 0.5. Одно слабое место не скрыть за сильным.
