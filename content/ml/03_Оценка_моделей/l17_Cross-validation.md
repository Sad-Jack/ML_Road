# 17. Cross-validation

## Зачем это нужно

Один прогон на валидационной выборке даёт одну оценку качества модели — и она может быть случайно удачной или неудачной в зависимости от того, как данные разбились. Cross-validation решает эту проблему: модель оценивается несколько раз на разных частях данных, и итоговая оценка надёжнее. Особенно важно, когда данных мало.

## Простая интуиция

Представь, что ты хочешь оценить нового сотрудника. Если дать ему один тест в один день — результат может зависеть от случайных факторов. Если провести пять тестов в разные дни на разных задачах и усреднить — оценка будет честнее. Именно так работает k-fold cross-validation.

## Что это такое

Cross-validation (перекрёстная проверка) — метод оценки модели, при котором данные разбиваются на k частей (фолдов), модель обучается k раз, каждый раз используя k-1 фолдов для обучения и 1 фолд для валидации. Финальная оценка — среднее по k запускам.

Ключевые понятия:
- **k-fold CV** — базовый вариант, обычно k=5 или k=10
- **Stratified k-fold** — для классификации: каждый фолд сохраняет пропорцию классов
- **Leave-One-Out (LOO)** — k равно числу объектов, очень медленно, используется редко
- **Преимущество** — каждый объект используется и для обучения, и для валидации

## Как это работает

```python
from sklearn.model_selection import cross_val_score

# model и данные X, y — уже подготовлены
scores = cross_val_score(model, X, y, cv=5, scoring='accuracy')
# → [0.82, 0.84, 0.86, 0.83, 0.85]

print(scores.mean())  # 0.84 — итоговая оценка
print(scores.std())   # 0.015 — разброс: чем меньше, тем стабильнее
```

Большой `scores.std()` говорит о нестабильности модели или о том, что данных слишком мало.

## Простой пример

Команда строит модель оттока клиентов для телеком-оператора. Датасет — 2 000 клиентов. Если сделать один split 80/20, валидационная выборка — 400 клиентов, и оценка случайно зависит от того, какие именно 400 попали в неё. С 5-fold CV модель оценивается 5 раз, каждый раз на 400 разных клиентах. Среднее accuracy по пяти фолдам — 84.2%, std — 1.8%. Это говорит: модель стабильна и оценка надёжна.

## Краткий итог

- K-fold CV снижает зависимость оценки от случайного разбиения: чем меньше данных — тем важнее его применять
- Итоговая оценка — среднее по фолдам, она надёжнее одного split
- Stratified k-fold обязателен при дисбалансе: без него фолды могут содержать неравные доли классов и давать нестабильную оценку
- CV заменяет val split, но не заменяет тестовую выборку
- Высокий std по фолдам = нестабильная модель или мало данных
- Особенно ценен при малом объёме данных, где каждый пример важен

## Ключевые термины урока

- **Cross-validation (CV)** — оценка модели на нескольких разных разбиениях данных для надёжности.
- **k-fold CV** — данные делятся на k частей; модель обучается и оценивается k раз.
- **Stratified k-fold** — k-fold с сохранением пропорций классов в каждом фолде.
- **Фолд** — одна часть данных при разбиении на k блоков.
