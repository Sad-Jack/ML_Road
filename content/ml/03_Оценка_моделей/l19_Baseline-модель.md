# 19. Baseline-модель

## Зачем это нужно

Прежде чем оценивать модель, нужен ориентир: **что считается «достаточно хорошим» результатом** для конкретной задачи? Без этого ориентира непонятно — оценка модели 0.85 это победа или провал? Этот ориентир называется baseline.

## Простая интуиция

Представьте синоптика, который каждый день говорит одно и то же: «Завтра будет как сегодня». В Москве в ноябре он будет прав в 70% случаев — просто потому что погода редко меняется резко. Если ваша модель прогнозирует погоду с той же точностью 70% — она ничего не сделала. Это и есть baseline: самое простое решение, которое нужно обогнать.

## Что это такое

**Baseline** — предсказание, которое строится без ML, используя только простую статистику по целевой переменной из обучающей выборки.

Как строится:

- **Классификация**: найти **мажоритарный класс** — тот, что встречается в обучающих данных чаще всего — и предсказывать его для каждого нового объекта, не глядя ни на один признак.
- **Регрессия**: вычислить среднее значение целевой переменной на обучающей выборке и предсказывать это число для каждого нового объекта.

Результат один и тот же для всех объектов — baseline полностью игнорирует признаки.

## Как это работает

В реальном проекте baseline строится в первый же день, ещё до любых экспериментов с моделями:

1. Смотрите на распределение целевой переменной в обучающей выборке.
2. Считаете результат baseline на val — это нижняя планка.
3. Любая модель, которую вы обучите дальше, сравнивается с этой планкой.

Важен не абсолютный результат модели, а **разрыв между моделью и baseline**:

- **Разрыв большой** (baseline 60%, модель 85%) — модель хорошо уловила закономерности в данных.
- **Разрыв маленький** (baseline 94%, модель 95%) — 95% выглядит красиво, но реальный прирост всего 1 процентный пункт. Модель почти ничему не научилась. Скорее всего, данные плохие или задача сформулирована неверно.

## Простой пример

Задача: предсказать, уйдёт клиент из банка или нет. В обучающей выборке:
- «остался» — 950 из 1000 (95%)
- «ушёл» — 50 из 1000 (5%)

**Baseline**: мажоритарный класс — «остался». Предсказываем «остался» для всех объектов, не глядя ни на один признак. Точность на val = 95%.

Вы обучаете модель и получаете точность 95.2%. Без baseline кажется, что 95.2% — отличный результат. С baseline понятно: реальный прирост всего 0.2 процентного пункта, модель почти не отличается от правила «всегда говори остался».

## Практика

1. Команда получила точность 92% на задаче классификации и считает модель отличной. Какой первый вопрос нужно задать, прежде чем согласиться с этой оценкой?
2. В датасете 700 объектов «не купил» и 300 «купил». Коллега обучил модель и доволен: accuracy = 69%. Посчитай baseline для этих данных и объясни, стоит ли продолжать проект с такой моделью.
3. Baseline для регрессии предсказывает одно и то же число — среднее по обучающей выборке — для всех объектов. Он явно ошибается для большинства из них. Почему такой baseline всё равно полезен?
4. В уроке «Train / Validation / Test» мы говорили, что val нужен для честной оценки модели. Почему baseline нужно считать на val, а не на train — что изменится, если посчитать baseline на тех же данных, где учится модель?
5. В уроке «Cross-validation» мы узнали, что одно разбиение ненадёжно. Как это применимо к baseline: стоит ли пересчитывать baseline при каждом фолде k-fold CV или достаточно посчитать его один раз на всём train?

## Краткий итог

- Baseline строится в первый день проекта — до любых экспериментов с моделями.
- Для классификации: мажоритарный класс. Для регрессии: среднее по обучающей выборке.
- Baseline использует только целевую переменную из обучающих данных — ни один признак не задействован.
- Важен не абсолютный результат, а разрыв: 95.2% при baseline 95% — почти ничего; 75% при baseline 50% — модель реально работает.
- Высокая точность без сравнения с baseline не говорит ничего о реальной ценности модели.

## Ключевые термины урока

- **Baseline** — нижняя планка качества. Важно: иногда правильный baseline — не «всегда предсказывать среднее», а бизнес-правило («одобрять всех» / «брать топ-10 по выручке»). ML должен превзойти именно этот baseline, а не только статистический.
- **Мажоритарный класс** — ловушка: при дисбалансе 99:1 мажоритарный baseline даёт accuracy 99% — выглядит отлично, но бесполезен. Именно поэтому accuracy — плохая метрика для несбалансированных данных, и нужны precision/recall.
- **Тривиальный baseline для регрессии** — всегда предсказывает среднее по обучающей выборке; его ошибка на val — минимальная планка, которую должна превзойти любая модель.
- **Разрыв над baseline** — реальная ценность модели. 95.2% при baseline 95% почти ничего не стоит; 75% при baseline 50% — модель реально работает.
