# 58. Isolation Forest

## Зачем это нужно

Isolation Forest — один из самых эффективных и популярных алгоритмов обнаружения аномалий. Он работает быстро, масштабируется на большие данные и хорошо справляется с многомерными данными. Понимание его принципа помогает правильно настраивать и интерпретировать результаты.

## Простая интуиция

Представьте игру: вы произвольно разрезаете прямоугольник вертикальными и горизонтальными линиями. Чтобы изолировать одиночную точку-аномалию вдали от основного облака, нужно мало разрезов. Чтобы изолировать точку внутри плотного облака — много разрезов. Аномалии «изолируются» быстрее — в этом суть алгоритма.

## Что это такое

- **Isolation Forest** — ансамбль случайных деревьев, каждое из которых случайно делит данные
- **Путь изоляции** — количество разрезов, нужных чтобы «поймать» объект в одиночестве
- Аномалии имеют **короткий путь изоляции**: их легко отделить от остальных
- **Anomaly score** — нормализованная длина пути; ближе к 1 = аномалия, к 0.5 = нормальный
- **contamination** — гиперпараметр, задающий ожидаемую долю аномалий

## Как это работает

```python
from sklearn.ensemble import IsolationForest

iso_forest = IsolationForest(
    n_estimators=100,    # деревьев в ансамбле
    contamination=0.05,  # ожидаемая доля аномалий
    random_state=42
)
iso_forest.fit(X_train)

# -1 = аномалия, 1 = нормальный объект
labels = iso_forest.predict(X)
print(f"Аномалий: {(labels == -1).sum()}")

# Численный скор аномальности (чем выше — тем аномальнее)
scores = -iso_forest.score_samples(X)
```

## Простой пример

Производственная линия фиксирует 15 датчиков каждую секунду: температура, давление, вибрация и др. За неделю собрано 600 000 записей нормальной работы оборудования. Isolation Forest обучается на этих данных (без меток аномалий). При появлении нетипичной комбинации показателей — аномальный score резко растёт. За следующую неделю алгоритм выдал 340 предупреждений; 28 из них совпали с реальными поломками, произошедшими через 10–120 минут после сигнала.

## Краткий итог

- Isolation Forest не требует разметки аномалий: он ищет объекты, которые легко «отрезать» от остальных
- Правильно оценить contamination важно: завысишь — много ложных тревог, занизишь — пропустишь реальные аномалии
- Хорошо масштабируется на большие данные и высокую размерность
- Результат (-1/1) или аномальный score можно использовать как признак в downstream задачах

## Ключевые термины урока

- **Isolation Forest** — алгоритм обнаружения аномалий через случайную изоляцию точек.
- **Путь изоляции** — число разрезов, нужных для изоляции точки; у аномалий он короче.
- **Anomaly score** — оценка аномальности от -1 (аномалия) до +0.5 (норма).
- **contamination** — доля аномалий в данных; задаётся вручную.
