# 54. Снижение размерности

## Зачем это нужно

Реальные датасеты часто имеют сотни и тысячи признаков. Это проблема: многие признаки коррелируют, модели переобучаются, визуализация невозможна. Снижение размерности помогает сжать данные, убрать шум и понять их структуру.

## Простая интуиция

Представьте, что вы фотографируете 3D-объект. Фотография — это 2D-проекция 3D-объекта. Вы теряете одно измерение, но сохраняете главную информацию о форме. Снижение размерности делает то же самое с данными: проецирует их из многомерного пространства в пространство меньшей размерности.

## Что это такое

- **Проклятие размерности** — в высоких измерениях данные становятся разреженными, расстояния теряют смысл, модели нуждаются в экспоненциально больше данных
- **Линейные методы** — PCA, SVD: ищут линейные комбинации признаков
- **Нелинейные методы** — t-SNE, UMAP: сохраняют нелинейную структуру данных
- **Автоэнкодеры** — нейросети, сжимающие и восстанавливающие данные
- Цели снижения размерности: визуализация, ускорение обучения, борьба с переобучением, сжатие

## Как это работает

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
import numpy as np

# Нормализация обязательна перед снижением размерности!
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)  # fit только на train

pca = PCA()
pca.fit(X_scaled)

# Накопленная объяснённая дисперсия — сколько компонент нужно?
cumsum = np.cumsum(pca.explained_variance_ratio_)
n_95 = np.searchsorted(cumsum, 0.95) + 1
print(f"Для 95% дисперсии: {n_95} компонент из {X.shape[1]}")
```

## Простой пример

Датасет из 800 пациентов с результатами 200 медицинских анализов. Половина признаков — разные измерения одних и тех же показателей крови (высокая корреляция). PCA показывает: первые 15 компонент объясняют 95% дисперсии. Снижение с 200 до 15 признаков ускоряет обучение модели в 10 раз, при этом качество предсказания падает с 87.3% до 86.8% — незначительная потеря. Плюс визуализация первых 2 компонент показывает явное разделение здоровых пациентов и больных.

## Краткий итог

- Снижение размерности борется с проклятием размерности и помогает визуализировать данные
- PCA можно безопасно включить в pipeline как preprocessing; t-SNE и UMAP — нельзя: их трансформацию нельзя корректно применить к новым данным
- Использовать t-SNE/UMAP как preprocessing — распространённая ошибка: они не создают переиспользуемую трансформацию
- Нормализация перед снижением размерности обязательна

## Ключевые термины урока

- **Проклятие размерности** — при p >> n все точки становятся «одинаково далеко» друг от друга; расстояния теряют смысл. k-NN и SVM деградируют первыми; деревья устойчивее, но тоже страдают от шумовых признаков.
- **PCA (Principal Component Analysis)** — единственный из трёх (PCA/t-SNE/UMAP), который создаёт переиспользуемую трансформацию: pca.fit(train) → pca.transform(test). Поэтому только PCA можно включить в Pipeline.
- **t-SNE / UMAP** — нелинейные, только для визуализации в 2D/3D. Не имеют стабильного transform() для новых данных. Распространённая ошибка: использовать их как preprocessing — модель, обученная на t-SNE координатах, несовместима с prod-данными.
- **Нормализация обязательна** — PCA, t-SNE и UMAP чувствительны к масштабу; признак с диапазоном 1–1000 доминирует над признаком 0–1. StandardScaler перед любым снижением размерности.
