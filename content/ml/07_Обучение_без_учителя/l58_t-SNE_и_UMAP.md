# 56. t-SNE и UMAP

## Зачем это нужно

PCA линеен и не всегда хорошо разделяет сложные структуры данных. t-SNE и UMAP — нелинейные методы, созданные специально для визуализации высокоразмерных данных. Они помогают увидеть кластеры, паттерны и аномалии, которые PCA скрывает.

## Простая интуиция

Представьте карту метро. Реальные расстояния между станциями не важны — важно, какие станции находятся рядом по линиям. t-SNE делает то же самое: сохраняет «кто рядом с кем» в исходном пространстве, пренебрегая глобальными расстояниями. UMAP делает это быстрее и лучше сохраняет глобальную структуру.

## Что это такое

- **t-SNE** (t-distributed Stochastic Neighbor Embedding) — минимизирует расстояния между похожими точками в 2D
- **UMAP** (Uniform Manifold Approximation and Projection) — более быстрая и масштабируемая альтернатива t-SNE
- **perplexity** — параметр t-SNE, контролирующий «количество соседей» (обычно 5-50)
- **n_neighbors** — параметр UMAP, аналогичный perplexity
- Оба метода **недетерминированы** и не создают переиспользуемой трансформации для новых данных

## Как это работает

```python
from sklearn.manifold import TSNE
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# t-SNE: нелинейное сжатие до 2D для визуализации
tsne = TSNE(n_components=2, perplexity=30, random_state=42)
X_2d = tsne.fit_transform(X_scaled)
print(f"{X.shape} → {X_2d.shape}")

# UMAP: быстрее и лучше масштабируется (pip install umap-learn)
# X_2d = umap.UMAP(n_components=2, random_state=42).fit_transform(X_scaled)

# Внимание: оба метода не создают переиспользуемой трансформации для новых данных!
```

## Простой пример

Датасет из 5 000 изображений рукописных цифр (0–9), каждое — 784 пикселя. PCA до 2 компонент — цифры перемешаны, визуально не разделить. t-SNE (perplexity=30) за 3 минуты формирует 10 чётких кластеров на плоскости — каждая цифра своя группа. До запуска любого классификатора это сразу показывает: данные разделимы, задача реалистична. UMAP даёт похожий результат за 40 секунд вместо 3 минут.

## Краткий итог

- t-SNE и UMAP раскрывают скрытые кластеры, которые линейный PCA не покажет: используй для понимания данных перед моделированием
- Используйте для EDA, не для предобработки
- UMAP быстрее t-SNE и лучше масштабируется
- Всегда фиксируйте random_state для воспроизводимости

## Ключевые термины урока

- **t-SNE** — нелинейный метод снижения размерности для визуализации высокоразмерных данных.
- **UMAP** — более быстрая альтернатива t-SNE с лучшей масштабируемостью.
- **perplexity** — ключевой гиперпараметр t-SNE, влияющий на размер «соседства» точки.
- **n_neighbors** — ключевой гиперпараметр UMAP, задающий число ближайших соседей.
- **EDA (разведочный анализ)** — основное применение t-SNE и UMAP.
