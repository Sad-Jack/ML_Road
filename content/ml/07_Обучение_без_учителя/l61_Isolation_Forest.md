# 61. Isolation Forest

## Зачем это нужно

У вас миллион транзакций и ни одной метки «мошенничество». Нужно найти подозрительные — без учителя. Статистические методы пропускают аномалии, которые нормальны по каждому признаку отдельно, но аномальны в сочетании. Isolation Forest решает это: он не учится предсказывать класс, а ищет объекты, которые «легко отрезать» от остальных случайными разрезами. Именно это делает его одним из стандартных инструментов обнаружения аномалий.

## Простая интуиция

Представьте игру: вы произвольно разрезаете прямоугольник вертикальными и горизонтальными линиями. Чтобы изолировать одиночную точку-аномалию вдали от основного облака, нужно мало разрезов. Чтобы изолировать точку внутри плотного облака — много разрезов. Аномалии «изолируются» быстрее — в этом суть алгоритма.

## Что это такое

- **Isolation Forest** — ансамбль случайных деревьев, каждое из которых случайно делит данные
- **Путь изоляции** — количество разрезов, нужных чтобы «поймать» объект в одиночестве
- Аномалии имеют **короткий путь изоляции**: их легко отделить от остальных
- **predict()** — бинарная метка: `-1` = аномалия, `+1` = нормальный объект
- **Anomaly score** — непрерывная оценка через `score_samples()`: значения ближе к 0.5 = норма, более отрицательные = аномалия. В коде используется `-score_samples()`, чтобы перевернуть знак: выше = аномальнее
- **contamination** — гиперпараметр, задающий ожидаемую долю аномалий

## Как это работает

Isolation Forest строит ансамбль случайных деревьев изоляции:

1. Из данных случайно выбирается подвыборка (обычно 256 объектов)
2. Дерево строится случайными разрезами: выбирается произвольный признак и случайное значение разреза в его диапазоне
3. Дерево делит точки до тех пор, пока каждая точка не окажется в одиночестве
4. Для каждой точки фиксируется глубина, на которой она была изолирована — «путь изоляции»
5. Процесс повторяется для n_estimators деревьев; средний путь изоляции по всем деревьям — аномальный скор

Аномалии изолируются быстро (короткий путь), нормальные точки — медленно (длинный путь). Порог между нормой и аномалией определяется параметром contamination.

Помимо бинарного результата (−1 / +1), алгоритм выдаёт непрерывный скор: чем выше скор, тем аномальнее объект — это позволяет ранжировать аномалии по степени подозрительности.

## Простой пример

Производственная линия фиксирует 15 датчиков каждую секунду: температура, давление, вибрация и др. За неделю собрано 600 000 записей нормальной работы оборудования. Isolation Forest обучается на этих данных (без меток аномалий). При появлении нетипичной комбинации показателей — аномальный score резко растёт. За следующую неделю алгоритм выдал 340 предупреждений; 28 из них совпали с реальными поломками, произошедшими через 10–120 минут после сигнала.

## Практика

1. Isolation Forest с contamination=0.05 помечает 500 объектов из 10 000. Ты снизил до 0.01 — помечается 100. Коллега предлагает автоматически выбирать contamination по критерию «наименьшее число тревог». Объясни, почему этот критерий неверен, — и как правильно валидировать выбор contamination, если размеченных аномалий нет.
2. Вы хотите не просто детектировать аномалии, но и ранжировать их по «тяжести». Как использовать Isolation Forest для ранжирования, а не только для бинарной классификации? Какой метод и что возвращает?
3. Вы анализируете логи сервера: 1 миллион строк, подозреваете менее 1% атак. Как настроить Isolation Forest и как оценить его работу в отсутствие размеченных данных?
4. Random Forest и Isolation Forest оба строят ансамбль случайных деревьев. В чём принципиальная разница: как они строят деревья, что за «цель» у каждого дерева и как итоговый вывод получается из ансамбля?
5. В уроке «Поиск аномалий» мы говорили, что Z-score и IQR пропускают аномалии, которые нормальны по каждому признаку отдельно. Почему Isolation Forest обнаруживает такие аномалии — что в его механике позволяет учитывать сочетание признаков, а не каждый по отдельности?

## Краткий итог

- Isolation Forest не требует разметки аномалий: он ищет объекты, которые легко «отрезать» от остальных
- Правильно оценить contamination важно: завысишь — много ложных тревог, занизишь — пропустишь реальные аномалии
- Хорошо масштабируется на большие данные и высокую размерность
- Результат (-1/1) или аномальный score можно использовать как признак в downstream задачах

## Ключевые термины урока

- **Isolation Forest** — аномалию легче изолировать: нужно меньше случайных разрезов. Ключевая идея: random forest без таргета, где «лёгкость изоляции» = аномальность. Не требует меток.
- **Путь изоляции** — среднее число разрезов для изоляции точки по n_estimators деревьям. Нормальные точки требуют длинного пути (log n), аномалии — короткого (2–5 разрезов).
- **predict() vs score_samples()** — predict() возвращает бинарный результат: -1 (аномалия) или 1 (норма) через порог contamination. score_samples() — непрерывная оценка: более отрицательное = более аномальное; полезен для ранжирования аномалий по тяжести.
- **contamination** — ожидаемая доля аномалий; задаётся вручную (0.01–0.1 в большинстве задач). Завысишь → много ложных тревог; занизишь → пропустишь реальные аномалии.
