# 55. PCA простыми словами

## Зачем это нужно

PCA (Principal Component Analysis) — самый распространённый метод снижения размерности. Он используется в сотнях задач: сжатие изображений, анализ геномных данных, визуализация, борьба с переобучением. Понимание PCA открывает один из ключевых инструментов data science.

## Простая интуиция

Представьте тени от предметов. Если поставить лампу под правильным углом, тень от 3D-объекта сохранит максимум информации о его форме. PCA находит такой «угол освещения» для ваших данных — направление, в котором данные наиболее «разбросаны», и проецирует их туда.

Важный нюанс: PCA не «тень» в прямом смысле — он создаёт **новые признаки** (линейные комбинации старых): `PC1 = 0.3·age + 0.8·salary + 0.1·exp`. Эти признаки — не исходные колонки. Вернуться к исходным значениям точно уже нельзя.

## Что это такое

- **PCA** — метод, находящий линейные комбинации признаков (главные компоненты) с максимальной дисперсией
- **Главные компоненты (PC)** — новые оси, ортогональные друг другу, упорядоченные по убыванию объяснённой дисперсии. **Ортогональные** — значит независимые (угол 90°): знание значения PC1 ничего не говорит о PC2
- **Explained variance ratio** — доля общего разброса данных, захваченная каждой компонентой. Если PC1=0.45, значит 45% всего разнообразия данных объясняется первой компонентой
- **n_components** — сколько компонент оставить
- PCA — это SVD (Singular Value Decomposition) матрицы данных

## Как это работает

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# Обязательно: нормализуем ПЕРЕД PCA (fit только на train!)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

# Вариант 1: задать число компонент вручную
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)
print(f"Объяснённая дисперсия: {pca.explained_variance_ratio_.round(3)}")
print(f"Итого: {pca.explained_variance_ratio_.sum():.1%}")

# Вариант 2: задать долю сохранённой дисперсии
pca_95 = PCA(n_components=0.95)  # автовыбор числа компонент
X_95 = pca_95.fit_transform(X_scaled)
print(f"Компонент для 95%: {pca_95.n_components_}")
```

## Простой пример

Датасет геномных данных: 1 200 пациентов, 1 000 признаков (уровни экспрессии генов). PC1 объясняет 28% дисперсии, PC2 — 15%, PC3 — 9%. График первых двух компонент показывает три чётких кластера — позже оказывается, что они соответствуют трём подтипам заболевания, ещё не известным врачам. Вместо 1 000 генов в модель передают 40 компонент (93% дисперсии), обучение занимает секунды вместо минут.

## Краткий итог

- PCA находит ортогональные направления максимальной дисперсии в данных
- Обязательно нормализуйте данные перед PCA
- Обучайте PCA только на train, затем применяйте трансформацию к train и test
- Выбор n_components через explained_variance_ratio_: обычно 90-95% дисперсии достаточно

## Ключевые термины урока

- **PCA** — создаёт НОВЫЕ признаки (PC1, PC2...) как линейные комбинации исходных; не выбирает исходные признаки, а смешивает их. Следствие: интерпретируемость теряется — «PC1 = 0.4·age + 0.3·income − 0.2·debt» нечитаемо.
- **Главные компоненты (PC)** — ортогональны (независимы под углом 90°); PC1 объясняет наибольшую долю дисперсии, PC2 — наибольшую из оставшихся. Ортогональность = отсутствие корреляции между компонентами.
- **Explained variance ratio** — доля разброса данных, объяснённая каждой компонентой. Кумулятивное 90–95% — практический порог для выбора n_components; ниже = потеря информации, выше = сохраняем шум.
- **fit только на train** — pca.fit(X_train) → параметры компонент вычислены; pca.transform(X_test) применяет те же параметры. Никогда pca.fit(X_test) — test заразит компоненты.
