# 57. Поиск аномалий

## Зачем это нужно

Аномалии встречаются везде: мошеннические транзакции, сбои оборудования, ошибки в данных, кибератаки. Часто у нас нет размеченных примеров аномалий — только нормальные данные. Поиск аномалий без разметки — задача unsupervised learning с огромным практическим значением.

## Простая интуиция

Представьте охранника в толпе. Большинство людей ходят похожими маршрутами. Человек, который бегает зигзагами, стоит подозрительно долго в одном месте или появляется в неожиданное время — аномалия. Охранник выучил «нормальное» поведение и замечает отклонения.

## Что это такое

- **Аномалия (выброс, outlier)** — объект, значительно отличающийся от остальных
- **Типы аномалий**:
  - *Точечные* — отдельная точка далеко от основного облака
  - *Контекстуальные* — нормальное значение в ненормальном контексте (жара зимой)
  - *Коллективные* — группа точек аномальна вместе
- **Статистические методы** — Z-score, IQR: простые пороговые правила
- **ML методы** — Isolation Forest, Local Outlier Factor (LOF), AutoEncoder
- **Задача**: обычно есть только нормальные данные, аномалии — редкие или отсутствуют в обучении

## Как это работает

```python
import numpy as np
from sklearn.ensemble import IsolationForest

# Подход 1: статистика (Z-score) — только для одномерных данных
z_scores = np.abs((X - X.mean()) / X.std())
print(f"Z-score аномалий: {(z_scores > 3).sum()}")

# IQR — ещё один простой одномерный метод
Q1, Q3 = np.percentile(X, [25, 75])
outliers_iqr = ((X < Q1 - 1.5*(Q3-Q1)) | (X > Q3 + 1.5*(Q3-Q1))).sum()
print(f"IQR аномалий: {outliers_iqr}")

# Подход 2: ML (Isolation Forest) — работает в многомерных данных
iso = IsolationForest(contamination=0.02, random_state=42)
labels = iso.fit_predict(X_multi)   # -1 = аномалия, 1 = норма
print(f"Isolation Forest аномалий: {(labels == -1).sum()}")
```

## Простой пример

Банк анализирует 500 000 транзакций по 8 признакам: сумма, время суток, город, тип терминала, частота использования карты и др. Z-score по каждому признаку отдельно пропускает транзакцию на 1 500 рублей в 3:00 ночи в новом городе — каждый признак по отдельности выглядит нормально. Isolation Forest видит всю картину целиком: сочетание «ночь + новый город + нетипичный магазин» — аномалия. Выявлено 4 200 подозрительных транзакций (0.84%), направлено на ручную проверку.

## Краткий итог

- Аномалии — объекты, значительно отличающиеся от остальных данных
- Статистические методы (Z-score, IQR) просты, но работают только в 1D
- ML методы (Isolation Forest, LOF) лучше в многомерных данных
- Ключевой параметр: contamination — ожидаемая доля аномалий

## Ключевые термины урока

- **Аномалия** — объект без метки «аномалия» при обучении; модель учится на «нормальных» данных и отмечает то, что на них не похоже. В отличие от supervised классификации, нет y=1 для аномалий.
- **Z-score** — работает только если признак нормально распределён и только по одному признаку. Порог |z| > 3 ловит ~0.3% нормальных точек как ложные аномалии. При тяжёлых хвостах — ненадёжен.
- **IQR** — устойчив к распределению (не предполагает нормальность), но тоже одномерный. Подходит для feature engineering (удаление выбросов); для detection в production — слаб.
- **Isolation Forest** — многомерный: аномалия изолируется за меньше случайных разрезов. Не требует нормальности. Ключевой параметр: contamination (0.01–0.1 в большинстве задач).
- **LOF (Local Outlier Factor)** — сравнивает плотность точки с плотностью соседей; хорош для кластеров разной плотности, где Isolation Forest ошибается. Медленнее и хуже масштабируется.
