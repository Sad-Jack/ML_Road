# 51. Обучение без учителя

## Зачем это нужно

В реальных задачах разметка данных — это дорого и долго. Иногда меток просто нет: вы хотите понять структуру данных, найти группы клиентов или обнаружить аномалии. Именно здесь применяется обучение без учителя — один из трёх фундаментальных подходов в ML.

## Простая интуиция

Представьте, что вы разбираете шкаф с вещами и раскладываете их по стопкам: зимнее, летнее, спортивное. Никто не говорил вам, как разложить — вы сами нашли похожие вещи и сгруппировали их. Это и есть обучение без учителя: найти структуру там, где нет заранее заданных ответов.

## Что это такое

- **Unsupervised learning** — класс алгоритмов ML, работающих с данными без меток
- Основные задачи:
  - **Кластеризация** — разбить объекты на похожие группы (K-Means, DBSCAN)
  - **Снижение размерности** — сжать данные, сохранив структуру (PCA, t-SNE)
  - **Обнаружение аномалий** — найти объекты, сильно отличающиеся от остальных
  - **Генеративные модели** — научиться генерировать новые похожие объекты (VAE, GAN)
- В отличие от supervised learning, нет «правильного ответа» для проверки

## Как это работает

```python
from sklearn.cluster import KMeans
import numpy as np

# X — данные без меток, просто точки в пространстве
# Задача: найти 3 кластера без знания y

unique, counts = np.unique(labels, return_counts=True)
print("Кластер → размер:", dict(zip(unique, counts)))
# {0: 102, 1: 98, 2: 100} — три примерно равных кластера
```

## Простой пример

Интернет-магазин накопил данные о 50 000 покупателях: частота покупок, средний чек, категории товаров — но без каких-либо меток. Запустили K-Means (K=3) и получили три группы: «редкие покупатели с малым чеком» (32 000 человек), «регулярные покупатели среднего чека» (14 000), «VIP с высокой частотой и большим чеком» (4 000). Метки никто не ставил — алгоритм нашёл структуру сам. Теперь для каждой группы можно запустить разные маркетинговые кампании.

## Краткий итог

- Unsupervised learning работает с данными без меток, ища скрытую структуру
- Основные задачи: кластеризация, снижение размерности, обнаружение аномалий
- Сложно оценивать качество без «правильных ответов»
- Часто используется как вспомогательный инструмент в supervised пайплайне

## Ключевые термины урока

- **Unsupervised learning** — обучение без меток: алгоритм ищет скрытую структуру в данных.
- **Кластеризация** — разбиение данных на группы схожих объектов.
- **Снижение размерности** — уменьшение числа признаков с сохранением структуры данных.
- **Обнаружение аномалий** — поиск объектов, значительно отличающихся от остальных.
- **Генеративные модели** — модели, обученные создавать новые объекты, похожие на обучающие.
