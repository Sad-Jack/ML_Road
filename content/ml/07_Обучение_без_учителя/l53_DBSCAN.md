# 53. DBSCAN

## Зачем это нужно

K-Means требует задавать K заранее и плохо работает с кластерами нестандартных форм. DBSCAN решает обе проблемы: автоматически определяет число кластеров и умеет находить кластеры произвольной формы. Плюс — выявляет выбросы как отдельный класс.

## Простая интуиция

Представьте, что вы идёте по городу и собираете людей в группы. Правило простое: если рядом (в радиусе eps) стоит достаточно людей (min_samples), — это плотная зона, часть кластера. Одиночки в пустынных местах — это выбросы. Вы не знаете заранее, сколько будет групп — они формируются сами.

## Что это такое

- **DBSCAN** (Density-Based Spatial Clustering of Applications with Noise) — алгоритм кластеризации на основе плотности
- **eps** — радиус окрестности: насколько близко должны быть точки
- **min_samples** — минимальное количество точек в радиусе eps для формирования плотной зоны
- **Основные точки** (core points) — точки с достаточным числом соседей
- **Граничные точки** — в eps-окрестности основной точки, но сами не основные
- **Шум (noise)** — точки, не принадлежащие ни одному кластеру (метка -1)

## Как это работает

```python
from sklearn.cluster import DBSCAN
import numpy as np

# DBSCAN находит кластеры произвольной формы без заданного K
dbscan = DBSCAN(eps=0.3, min_samples=5)
labels = dbscan.fit_predict(X)

# -1 = шум (outlier), остальные — номер кластера
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
print(f"Кластеров: {n_clusters}, шумовых точек: {(labels == -1).sum()}")

# eps слишком мало → всё шум; eps слишком велико → один большой кластер
```

## Простой пример

Логистическая компания анализирует GPS-треки 10 000 доставок по городу. K-Means с K=5 делит город на 5 географических зон, но маршруты доставки имеют произвольные формы вдоль улиц. DBSCAN с eps=200м и min_samples=10 находит 12 плотных зон доставки (жилые кварталы, бизнес-центры, промзоны) и помечает 340 точек как шум — это нетипичные адреса в промышленных районах. Именно шумовые точки требуют отдельного планирования маршрута.

## Краткий итог

- DBSCAN находит кластеры произвольной формы на основе плотности точек
- Параметры eps и min_samples нужно подбирать под данные
- Точки с метками -1 — выбросы (шум)
- Не требует задавать K заранее, но плохо масштабируется на большие датасеты

## Ключевые термины урока

- **DBSCAN** — алгоритм кластеризации, находящий плотные группы точек произвольной формы.
- **eps** — максимальное расстояние между точками для попадания в одну окрестность.
- **min_samples** — минимальное число точек для образования плотного кластера.
- **Шум (noise)** — точки с меткой -1: аномалии, не попавшие ни в один кластер.
- **Плотность** — число точек в окрестности заданного радиуса; основа алгоритма.
