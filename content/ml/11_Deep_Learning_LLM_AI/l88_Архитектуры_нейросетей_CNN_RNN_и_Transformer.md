# 88. Архитектуры нейросетей: CNN, RNN и Transformer

## Зачем это нужно

Deep Learning — это не одна универсальная нейросеть. Для разных типов данных существуют разные **архитектуры** — разные способы организации слоёв. Знать их на уровне интуиции нужно, чтобы понимать, какой инструмент для чего подходит.

## Простая интуиция

Разные задачи — разная структура:
- Фотографию лучше анализировать **блоками пикселей** → CNN
- Текст лучше читать **последовательно** → RNN/LSTM
- Смысл слова зависит от **контекста всей фразы** → Transformer

## Что это такое

Архитектура нейросети — это способ организации слоёв и связей между ними. Разные архитектуры возникли как ответ на конкретные типы данных: пространственные (изображения), последовательные (текст, аудио), смешанные. Выбор архитектуры определяет, какие паттерны сеть способна обнаруживать и насколько эффективно использует данные.

Четыре ключевых семейства:
- **CNN** — для пространственных данных с локальными паттернами
- **RNN/LSTM** — для последовательностей с зависимостью от прошлого
- **Transformer** — универсальная архитектура через механизм внимания
- **GAN** — для генерации новых данных через конкуренцию двух сетей

## Как это работает

**CNN — Convolutional Neural Network**

**Для чего:** изображения, видео, любые пространственные данные.

**Идея:** вместо того чтобы смотреть на все пиксели сразу, CNN применяет **маленькие фильтры** (ядра свёртки), которые скользят по изображению и обнаруживают локальные паттерны: края, текстуры, формы. Глубокие слои собирают паттерны из паттернов — от краёв до лиц и объектов.

```
Вход: изображение 224×224
→ Conv слой (края, текстуры)
→ Conv слой (формы, части объектов)
→ Fully Connected
→ Выход: класс (кошка/собака)
```

**Применения:** распознавание объектов, медицинские снимки, автопилот.

**RNN и LSTM — Recurrent Neural Network**

**Для чего:** последовательности — текст, временные ряды, аудио.

**Идея:** у RNN есть **скрытое состояние** — «память» о предыдущих шагах. При обработке каждого слова (или временного шага) сеть учитывает, что было раньше.

**Проблема:** обычный RNN плохо помнит далёкое прошлое. LSTM (Long Short-Term Memory) решает это через специальные «ворота», которые решают: что запомнить, что забыть, что передать дальше.

**Применения:** генерация текста, перевод, распознавание речи (до эпохи Transformer).

**Transformer**

**Для чего:** текст, код, изображения, аудио — сегодня главная архитектура для всего.

**Идея:** вместо последовательной обработки Transformer анализирует весь контекст сразу, учитывая взаимосвязи между всеми элементами последовательности одновременно.

**Применения:** GPT, BERT, Claude, Gemini, DALL-E, Whisper — почти всё современное DL.

**GAN — Generative Adversarial Network**

**Для чего:** генерация изображений, аудио, синтетических данных.

**Идея:** два нейрона играют в игру: **Generator** создаёт фейковые данные, **Discriminator** пытается отличить их от настоящих. Оба учатся друг на друге. Итог — Generator научается создавать очень реалистичные данные.

**Применения:** генерация лиц, deepfake, аугментация данных. Сегодня во многом вытеснены диффузионными моделями.

## Простой пример

Три задачи в одном стартапе: (1) Классификация фото товаров на 50 категорий (одежда, электроника и др.) — выбирают CNN (ResNet-50), потому что важны локальные паттерны: текстура ткани, форма устройства. Accuracy 94%. (2) Анализ тональности отзывов покупателей (позитив/негатив) — выбирают дообученный Transformer (BERT), потому что важен полный контекст предложения. F1 = 0.91. (3) Генерация описаний товаров по фото — выбирают мультимодальный Transformer (CLIP + GPT), потому что задача требует понять изображение и сгенерировать текст.

## Краткий итог

- Архитектуры не конкурируют — у каждой своя ниша: CNN для пространственных данных, RNN/LSTM для последовательностей, Transformer для глобального контекста
- Transformer вытеснил RNN из большинства NLP-задач в 2018+; для новых задач с текстом — начинайте с Transformer
- CNN остаётся непревзойдённой для изображений с локальными паттернами; Transfer Learning (ResNet, EfficientNet) экономит недели обучения
- Выбор архитектуры определяется типом данных — это первый вопрос при запуске DL-проекта
- GAN вытесняется диффузионными моделями для генерации изображений, но полезен для аугментации данных

## Ключевые термины урока

- **CNN** — локальные паттерны: фильтр 3×3 видит только соседние пиксели. Это быстро и эффективно для изображений (соседние пиксели связаны), но слабо для текста (слово и его антоним могут стоять далеко).
- **RNN/LSTM** — последовательная обработка: каждый шаг обновляет скрытое состояние. Проблема длинных зависимостей: LSTM лучше RNN, но Transformer решил её кардинально. В 2024+ RNN вытеснен Transformer из NLP.
- **Transformer** — анализирует всю последовательность сразу, учитывая взаимосвязи между всеми элементами; именно это позволило обучать модели на огромных корпусах. Механика работы подробно рассматривается в следующем уроке.
- **GAN** — генератор создаёт данные, дискриминатор отличает их от реальных; оба обучаются одновременно. Нестабильное обучение — известная проблема; в 2024 диффузионные модели вытесняют GAN для генерации изображений.
