# 91. Что такое LLM

## Зачем это нужно

LLM (Large Language Models) — это технология за ChatGPT, Claude, Gemini. Понимание того, как они работают, их возможностей и ограничений, критично для любого специалиста в области данных и технологий в 2024 году.

## Простая интуиция

Представьте книгочея, прочитавшего весь интернет. Он не знает «правды» — он знает, что обычно пишут после тех или иных слов. Спросите его «Столица Франции» — он ответит «Париж», потому что это слово чаще всего следует после таких слов. Именно так работает LLM: предсказывает наиболее вероятный следующий токен. Важно: LLM не запоминает тексты дословно — он учит распределение вероятностей над токенами. Это принципиально отличается от простого поиска по тексту или n-gram-модели.

## Что это такое

- **LLM** (Large Language Model) — трансформер, предобученный на огромных объёмах текста
- **Pre-training** — самообучение: предсказание следующего слова на триллионах токенов
- **SFT** (Supervised Fine-Tuning) — дообучение базовой модели на размеченных диалогах «вопрос → правильный ответ»; именно SFT превращает «предсказатель токенов» в полезного ассистента, прежде чем применяется RLHF
- **RLHF** (Reinforcement Learning from Human Feedback) — дообучение через человеческие оценки
- **Токен** — единица текста: примерно 1 слово или 4 символа
- **Температура** — параметр случайности генерации: высокая = творческий/непредсказуемый, низкая = детерминированный
- **Контекстное окно** — максимальный объём текста, который модель видит сразу (8K-1M токенов)

## Как это работает

LLM генерирует текст токен за токеном. На каждом шаге модель получает весь предыдущий контекст и предсказывает вероятности для каждого возможного следующего токена. Затем один токен выбирается (с учётом параметра температуры) и добавляется к контексту.

Ключевые параметры при работе с LLM:

| Параметр | Что регулирует | Типичные значения |
|---|---|---|
| temperature | Случайность генерации: 0 = детерминированный, выше = разнообразнее | 0 для фактов/кода, 0.7–1.0 для творчества |
| max_tokens | Максимум генерируемых токенов | 200–4000 |
| top_p | Nucleus sampling: из какой доли вероятностей выбирать | 0.9–0.95 |
| context_window | Максимум токенов на входе | 8K–1M токенов |

Pre-training → SFT → RLHF/DPO: сначала модель обучается предсказывать следующий токен на огромных текстах (pre-training), затем через SFT учится отвечать в нужном формате, и наконец выравнивается через оценки людей (RLHF) или прямую оптимизацию предпочтений (DPO), чтобы отвечать полезно и безопасно.

Два основных архитектурных типа LLM: **decoder-only** (GPT, Claude, Gemini) — генерируют текст токен за токеном, используются для диалога и генерации; **encoder-only** (BERT) — кодируют текст в представление для понимания/классификации, но не генерируют свободный текст.

## Простой пример

Юридическая компания тестирует два LLM для анализа договоров. Запрос: «Есть ли в этом договоре штрафные санкции за досрочное расторжение?». Claude с температурой 0 и контекстным окном 200K токенов: читает весь договор (80 страниц, ~60K токенов), находит пункт 14.3 с штрафом 30% от суммы. GPT-4 с temperature=0.8: отвечает уверенно, но «галлюцинирует» несуществующий пункт 12.1. Вывод: для фактических юридических задач temperature=0 обязательна, а выбор модели с большим контекстным окном позволяет обработать весь документ без разбивки.

## Краткий итог

- Сила LLM — в scale: те же принципы, что у маленьких моделей, но на порядки больше данных и параметров дают качественный скачок
- Ключевые параметры: temperature (случайность), max_tokens, context window
- Галлюцинации — следствие предсказания правдоподобности, а не фактической точности
- RLHF корректирует **alignment** — разрыв между «статистически правдоподобным текстом» (результат pre-training) и «полезным и безопасным ответом» (то, что нужно пользователю)
- Ограничение контекстного окна — практический потолок: длинные документы нужно делить или использовать архитектуры с расширенным контекстом

## Ключевые термины урока

- **LLM (Large Language Model)** — трансформер, обученный на огромных текстах предсказывать следующий токен.
- **Pre-training** — предобучение на большом корпусе текстов для получения базовой модели.
- **SFT (Supervised Fine-Tuning)** — дообучение базовой модели на примерах правильных ответов; ключевой шаг между pre-training и RLHF.
- **RLHF** — обучение с подкреплением из человеческой обратной связи для выравнивания модели.
- **Токен** — минимальная единица текста, с которой работает LLM (слово, часть слова, символ).
- **Температура** — параметр случайности генерации: выше = разнообразнее, ниже = детерминированнее.
- **Контекстное окно** — максимальное число токенов, которые LLM может учитывать за раз.
