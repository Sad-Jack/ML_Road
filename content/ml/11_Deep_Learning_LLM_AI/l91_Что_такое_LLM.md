# 88. Что такое LLM

## Зачем это нужно

LLM (Large Language Models) — это технология за ChatGPT, Claude, Gemini. Понимание того, как они работают, их возможностей и ограничений, критично для любого специалиста в области данных и технологий в 2024 году.

## Простая интуиция

Представьте книгочея, прочитавшего весь интернет. Он не знает «правды» — он знает, что обычно пишут после тех или иных слов. Спросите его «Столица Франции» — он ответит «Париж», потому что это слово чаще всего следует после таких слов. Именно так работает LLM: предсказывает наиболее вероятный следующий токен.

## Что это такое

- **LLM** (Large Language Model) — трансформер, предобученный на огромных объёмах текста
- **Pre-training** — самообучение: предсказание следующего слова на триллионах токенов
- **RLHF** (Reinforcement Learning from Human Feedback) — дообучение через человеческие оценки
- **Токен** — единица текста: примерно 1 слово или 4 символа
- **Температура** — параметр случайности генерации: высокая = творческий/непредсказуемый, низкая = детерминированный
- **Контекстное окно** — максимальный объём текста, который модель видит сразу (8K-1M токенов)

## Как это работает

```python
# Ключевые параметры при работе с LLM
params = {
    "temperature": 0.7,      # 0 = детерминированный, 1+ = творческий
    "max_tokens": 500,       # максимум генерируемых токенов
    "top_p": 0.95,           # nucleus sampling
    "context_window": 200000 # максимум токенов на входе
}
```

## Простой пример

Юридическая компания тестирует два LLM для анализа договоров. Запрос: «Есть ли в этом договоре штрафные санкции за досрочное расторжение?». Claude с температурой 0 и контекстным окном 200K токенов: читает весь договор (80 страниц, ~60K токенов), находит пункт 14.3 с штрафом 30% от суммы. GPT-4 с temperature=0.8: отвечает уверенно, но «галлюцинирует» несуществующий пункт 12.1. Вывод: для фактических юридических задач temperature=0 обязательна, а выбор модели с большим контекстным окном позволяет обработать весь документ без разбивки.

## Краткий итог

- Сила LLM — в scale: те же принципы, что у маленьких моделей, но на порядки больше данных и параметров дают качественный скачок
- Ключевые параметры: temperature (случайность), max_tokens, context window
- Галлюцинации — следствие предсказания правдоподобности, а не фактической точности
- RLHF корректирует **alignment** — разрыв между «статистически правдоподобным текстом» (результат pre-training) и «полезным и безопасным ответом» (то, что нужно пользователю); RAG решает проблему устаревших знаний

## Ключевые термины урока

- **LLM (Large Language Model)** — трансформер, обученный на огромных текстах предсказывать следующий токен.
- **Pre-training** — предобучение на большом корпусе текстов для получения базовой модели.
- **RLHF** — обучение с подкреплением из человеческой обратной связи для выравнивания модели.
- **Токен** — минимальная единица текста, с которой работает LLM (слово, часть слова, символ).
- **Температура** — параметр случайности генерации: выше = разнообразнее, ниже = детерминированнее.
- **Контекстное окно** — максимальное число токенов, которые LLM может учитывать за раз.
