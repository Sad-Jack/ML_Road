# 96. Что такое AI Agent

## Зачем это нужно

LLM, который только отвечает на вопросы, — это чат-бот. AI Agent — это LLM, который может планировать, использовать инструменты и выполнять многошаговые задачи автономно. Это следующий уровень применения LLM: от ответов к действиям.

## Простая интуиция

Обычный LLM — как консультант, который отвечает на вопросы. AI Agent — как ассистент, которому можно сказать «организуй мне командировку» и он сам найдёт рейсы, забронирует отель, отправит письма и внесёт встречи в календарь. Разница — в способности действовать, а не только советовать.

## Что это такое

- **AI Agent** — система: LLM + инструменты + память + планирование
- **Инструменты (tools)** — функции, которые LLM может вызывать (поиск, API, код, база данных)
- **ReAct (Reason + Act)** — паттерн: Рассуждение → Действие → Наблюдение → Рассуждение...
- **Память**: контекстная (в промпте), долгосрочная (vector store), процедурная (память о навыках)
- **Планирование** — разбивка сложной задачи на шаги
- **Human-in-the-loop** — контроль человека на критических шагах

## Как это работает

Агент работает по циклу ReAct (Reason + Act):

```
Задача: "Найди цену акций Apple и сравни с вчерашней"

Шаг 1 — Рассуждение: нужна сегодняшняя цена → вызову search_web
Шаг 1 — Действие:    search_web("Apple stock price today") → $181.5
Шаг 1 — Наблюдение:  получил цену $181.5

Шаг 2 — Рассуждение: нужна вчерашняя цена → вызову search_web
Шаг 2 — Действие:    search_web("Apple stock price yesterday") → $179.2
Шаг 2 — Наблюдение:  получил цену $179.2

Шаг 3 — Рассуждение: нужно посчитать разницу → вызову calculate
Шаг 3 — Действие:    calculate("181.5 - 179.2") → 2.3
Шаг 3 — Наблюдение:  разница $2.3

Шаг 4 — Рассуждение: все данные есть → формулирую ответ
Ответ: "Акции выросли на $2.3 (1.3%)"
```

Каждый инструмент описан для LLM в виде текстового описания функции. LLM выбирает нужный инструмент по этому описанию, не зная кода. Результат каждого вызова добавляется в контекст — агент «помнит» всё, что произошло в текущей сессии.

## Простой пример

Агент для автоматизации аналитического отчёта: задача «Подготовь еженедельный отчёт по продажам». Шаг 1: агент вызывает инструмент `query_database("SELECT * FROM sales WHERE week=current_week")` — получает 15 000 строк. Шаг 2: вызывает `run_python_code("calculate_stats(data)")` — считает метрики (рост 12%, топ-5 товаров). Шаг 3: обнаруживает аномалию — резкий рост продаж категории «электроника» в Сибири — вызывает `search_news("электроника Сибирь промо")` — находит причину. Шаг 4: `create_report(metrics, insights)` — генерирует PDF. Вся цепочка: 4 инструмента, 8 минут, без участия аналитика.

## Краткий итог

- AI Agent = LLM + инструменты + память + планирование
- ReAct паттерн: Рассуждение → Действие → Наблюдение — базовый цикл агента
- Инструменты расширяют возможности: поиск, код, API, базы данных
- Human-in-the-loop обязателен для критических и необратимых действий

## Ключевые термины урока

- **AI Agent vs LLM** — LLM отвечает за один «ход»; агент запускает цикл: рассуждение → инструмент → результат → следующий шаг. Агент может сделать 10+ вызовов инструментов для ответа на один вопрос.
- **ReAct** — Thought (что мне нужно?) → Action (вызов инструмента) → Observation (результат) → Thought (теперь я знаю...). Прозрачность рассуждений — ключевое свойство: можно найти, где агент ошибся.
- **Инструменты (tools)** — любые Python-функции, описанные в JSON-схеме. LLM выбирает инструмент по описанию, не зная его кода. Плохое описание инструмента = агент выбирает не тот инструмент.
- **Human-in-the-loop** — обязателен для необратимых действий: «отправить email», «удалить файл», «провести платёж». Агент без HITL на таких действиях — автоматическая точка отказа с реальными последствиями.
- **Память** — краткосрочная (контекст диалога), долгосрочная (vector store с прошлыми взаимодействиями). Без памяти агент «забывает» между сессиями и не учится на прошлых ошибках.
