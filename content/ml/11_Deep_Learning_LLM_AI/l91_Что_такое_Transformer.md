# 90. Что такое Transformer

## Зачем это нужно

Transformer — архитектура, изменившая AI. GPT, BERT, Claude, Gemini, T5 — всё это трансформеры. Понимание базовой идеи attention позволяет разобраться, почему эти модели так мощны и какие у них ограничения.

## Простая интуиция

Читая предложение «Банк открыли раньше, чем солнце» — слово «банк» значит разное (финансовый или берег реки). Чтобы понять, нужно посмотреть на весь контекст. Attention — механизм, который для каждого слова спрашивает: «на какие другие слова мне стоит обратить внимание, чтобы понять себя?»

## Что это такое

- **Transformer** — архитектура, предложенная в статье "Attention is All You Need" (2017)
- **Токен** — единица обработки текста; обычно слово или часть слова: «безполезный» → «без», «полезный»; трансформер работает с последовательностью токенов
- **Attention** — механизм, позволяющий каждому токену «смотреть» на все другие токены
- **Self-attention** — attention внутри одной последовательности (слово смотрит на другие слова)
- **Multi-head attention** — несколько механизмов attention параллельно; каждый head фокусируется на разных аспектах контекста (синтаксис, семантика, дальние связи между словами)
- **Encoder** — обрабатывает входную последовательность (BERT)
- **Decoder** — генерирует выходную последовательность (GPT)

## Как это работает

В основе Transformer лежит механизм attention. Для каждого токена (слова) модель вычисляет, насколько важен каждый другой токен при его обработке. Это делается через три вектора: Query (что ищем), Key (что доступно), Value (что передать).

Упрощённая схема для предложения «Банк открыли раньше»:

```
Для токена "Банк":
  → сравнить с "открыли": вес 0.15
  → сравнить с "раньше":  вес 0.21
  → сравнить с "реки":    вес 0.64  ← высокий вес → понимаем контекст
Итоговый вектор "Банк" = взвешенная сумма всех токенов
```

Ключевые шаги:
1. Каждый токен преобразуется в три вектора: Q, K, V
2. Для каждого токена вычисляется score с каждым другим через скалярное произведение Q·Kᵀ
3. Scores нормализуются через softmax → получаем веса (attention weights)
4. Итоговый вектор токена = взвешенная сумма V всех токенов

Multi-head attention повторяет это параллельно несколько раз — каждый «head» фокусируется на разных аспектах.

## Простой пример

Задача: определить тональность отзыва «Телефон хорош, но батарея разряжается быстро». Слово «хорош» — позитивный сигнал, «быстро» само по себе нейтральное, но вместе с «разряжается» — негативный. RNN обрабатывает слово за словом и к слову «быстро» может «забыть» контекст «хорош» из начала. Transformer: каждое слово смотрит на все остальные. Слово «быстро» получает высокий вес attention к словам «батарея» и «разряжается». Итог: смешанная тональность — модель корректно определяет её как MIXED, а не просто POSITIVE или NEGATIVE.

## Краткий итог

- Transformer доминирует в современном ML: не знаешь что выбрать для последовательностей — начинай с трансформера
- Self-attention решил главную проблему RNN: дальние зависимости теперь не теряются — каждый токен видит весь контекст
- Все современные большие языковые модели (GPT, Claude, Llama, BERT) — это трансформеры
- Главное ограничение: контекстное окно — фиксированный максимальный размер входа

## Ключевые термины урока

- **Transformer** — архитектура нейросети на основе механизма attention без рекуррентности. Ключевое отличие от RNN: обрабатывает всю последовательность за один проход, а не по одному токену — это сделало возможным обучение на огромных корпусах.
- **Attention** — механизм, взвешивающий значимость каждого элемента для предсказания текущего. Формула: score = softmax(Q·Kᵀ / √d_k)·V; деление на √d_k предотвращает исчезновение градиентов при больших размерностях.
- **Self-attention** — каждый токен учитывает все остальные токены в последовательности. В отличие от RNN, нет «забывания»: токен из начала предложения влияет на токен в конце с той же лёгкостью, что и соседний.
- **Multi-head attention** — несколько параллельных блоков attention для разных аспектов контекста. Один head может отслеживать синтаксические зависимости, другой — семантическое сходство; результаты конкатенируются.
- **Encoder / Decoder** — части Transformer: encoder понимает входную последовательность (применяется в BERT для задач классификации), decoder генерирует выходную (применяется в GPT для генерации текста).
