# 93. Этика и ответственность в AI

## Зачем это нужно

AI-системы влияют на жизни людей: решают, кому дать кредит, кого взять на работу, как распределить ресурсы. Эти системы могут дискриминировать, нарушать приватность, работать непрозрачно. Ответственный AI — не только этически правильно, но и юридически и коммерчески необходимо.

## Простая интуиция

Нанимающий менеджер, который оценивает только кандидатов из элитных вузов — предвзятый. Если обучить ML-модель на исторических данных найма, где так и работали, — модель унаследует эту предвзятость. Мусор на входе → предвзятость на выходе. И модель не будет знать, что она несправедлива.

## Что это такое

- **Bias (предвзятость)** — систематическая ошибка в пользу/против определённых групп
- **Fairness (справедливость)** — равное обращение с группами. Метрики: equal accuracy, demographic parity
- **Explainability/Interpretability** — возможность объяснить решение модели (SHAP, LIME)
- **Privacy** — защита персональных данных. Методы: federated learning, differential privacy
- **Accountability** — кто несёт ответственность за решения AI?
- **EU AI Act** — регуляторная база ЕС для AI, классифицирует риски

## Как это работает

```python
import numpy as np

# Проверка fairness: сравниваем метрики по группам
y_pred = model.predict(X_test)
gender = X_test["gender"].values

for group_name, mask in [("Female", gender == 0), ("Male", gender == 1)]:
    acc = (y_pred[mask] == y_test[mask]).mean()
    hire_rate = y_pred[mask].mean()
    print(f"{group_name}: accuracy={acc:.3f}, hire_rate={hire_rate:.3f}")

# Female: accuracy=0.76, hire_rate=0.31
# Male:   accuracy=0.88, hire_rate=0.62
# Большая разница → сигнал предвзятости в данных или модели
```

## Простой пример

Банк обучил модель одобрения кредитов на данных 2010–2020 годов. В этот период женщины реже получали кредиты (историческая предвзятость). Модель воспроизвела паттерн: approval rate для мужчин 68%, для женщин 41%, хотя дефолт-рейтинг одинаков (4%). Юридический отдел требует объяснения каждого отказа (GDPR). SHAP-анализ показывает: признак «тип занятости» вносит наибольший вклад в предвзятость — исторически женщины чаще работали неполный день. После удаления биографических прокси-признаков и rebalancing: approval rate женщин вырос до 59%, дефолт-рейтинг остался 4%.

## Краткий итог

- AI-модели наследуют предвзятость из исторических данных — это нужно активно отслеживать
- Fairness, explainability, privacy — три столпа ответственного AI
- Регуляции (EU AI Act, GDPR) делают это юридическим обязательством
- Проверяйте метрики по группам, а не только общий accuracy

## Ключевые термины урока

- **Bias (предвзятость)** — систематические ошибки модели, дискриминирующие определённые группы.
- **Fairness (справедливость)** — равное качество модели для разных групп пользователей.
- **Explainability / Interpretability** — возможность объяснить, почему модель приняла то или иное решение.
- **Privacy** — защита персональных данных пользователей при обучении и использовании модели.
- **EU AI Act** — европейское регулирование AI, устанавливающее требования по рискам и прозрачности.
