# 89. Prompt Engineering

## Зачем это нужно

Качество ответа LLM зависит от качества запроса. Правильно составленный промпт — это разница между полезным ответом и мусором. Prompt Engineering — практическая навык, применимый прямо сейчас к любому LLM.

## Простая интуиция

Промпт — это как задание сотруднику. «Напиши что-нибудь про Python» — расплывчато, результат непредсказуем. «Напиши 5 практических советов по оптимизации Python кода для ML-пайплайна, для аудитории junior разработчиков, в формате нумерованного списка» — конкретно, результат предсказуем и полезен.

## Что это такое

- **Zero-shot prompting** — задать вопрос без примеров: «Переведи на английский: ...»
- **Few-shot prompting** — дать 2-3 примера перед задачей: «Пример 1: ... → ... Теперь: ...»
- **Chain-of-thought (CoT)** — попросить рассуждать: «Думай шаг за шагом»
- **System prompt** — инструкции о роли и поведении модели («Ты — эксперт по ML»)
- **Structured output** — просить ответ в определённом формате (JSON, список, таблица)
- **Temperature** — параметр случайности; для кода: 0, для творчества: 0.7-1.0

## Как это работает

```python
# Примеры шаблонов промптов

# 1. Zero-shot
zero_shot = """
Классифицируй тональность отзыва как POSITIVE, NEGATIVE или NEUTRAL.

Отзыв: "Доставка быстрая, но товар не соответствует описанию"
Тональность:
"""

# 2. Few-shot
few_shot = """
Классифицируй тональность:
Отзыв: "Отличный продукт!" -> POSITIVE
Отзыв: "Ужасное качество" -> NEGATIVE
Отзыв: "Обычный товар" -> NEUTRAL

Отзыв: "Доставка быстрая, но товар не соответствует описанию"
Тональность:
"""

# 3. Chain-of-thought
cot_prompt = """
Задача: есть датасет с 1000 строками, 20% пропущенных значений.
Выбери стратегию обработки пропусков.

Думай шаг за шагом:
1. Какой тип данных?
2. Каков процент пропусков?
3. Какая задача ML?
4. Что порекомендуешь?
"""

# 4. Структурированный вывод
structured = """
Проанализируй модель ML и верни ответ строго в JSON формате:
{
  "problem_type": "classification|regression|clustering",
  "recommended_algorithms": ["algo1", "algo2"],
  "key_risks": ["risk1"],
  "next_steps": ["step1"]
}

Задача: предсказать вероятность покупки для 50K клиентов.
"""
print("Шаблоны промптов готовы к использованию")
```

## Простой пример

Команда автоматизирует классификацию обращений в службу поддержки (5 категорий). Zero-shot: «Классифицируй: [текст]» — accuracy 71%, много ошибок между «техническая проблема» и «вопрос по функционалу». Few-shot (2 примера на категорию): accuracy 89% — модель видит паттерн разграничения. Chain-of-Thought для сложных случаев: «Сначала определи суть проблемы, потом выбери категорию» — спорных классификаций стало на 40% меньше. Структурированный вывод в JSON: `{"category": "billing", "priority": "high", "sentiment": "negative"}` — парсинг для CRM-системы без дополнительного кода.

## Краткий итог

- Few-shot и Chain-of-Thought — наиболее эффективные техники без дополнительного обучения: несколько примеров или «думай шаг за шагом» заметно улучшают результат
- System prompt задаёт роль и поведение для всего диалога
- Для точности: низкая temperature + структурированный вывод
- Явный формат ответа (JSON, список) улучшает предсказуемость и упрощает парсинг

## Ключевые термины урока

- **Zero-shot prompting** — запрос без примеров; модель отвечает опираясь на предобученные знания.
- **Few-shot prompting** — запрос с несколькими примерами «вопрос → ответ» для настройки поведения.
- **Chain-of-Thought (CoT)** — явная просьба «думай шаг за шагом» для повышения точности.
- **System prompt** — инструкция, задающая роль и поведение модели для всего диалога.
- **Temperature** — параметр случайности: низкая = точные ответы, высокая = творческие.
