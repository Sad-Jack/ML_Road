# 86. Как нейросеть учится на ошибках

## Зачем это нужно

Ты уже знаешь, что нейросеть — это слои с весами (урок «Что такое нейронная сеть»). Но как она вообще учится? Откуда берутся правильные веса? Ответ — в механизме обучения: loss, градиентный спуск и backpropagation. Без этого понимания нейросети остаются чёрным ящиком.

## Простая интуиция

Представь: ты бросаешь мяч в ворота и промахиваешься. Ты смотришь на ошибку (насколько промахнулся) и чуть корректируешь бросок. Повторяешь. Постепенно попадаешь точнее.

Нейросеть делает то же самое:
1. Делает предсказание
2. Считает, насколько ошиблась (loss)
3. Понимает, как надо изменить веса, чтобы ошибка уменьшилась (градиент)
4. Делает маленький шаг в правильном направлении
5. Повторяет миллионы раз

## Что это такое

**Loss (функция потерь)** — число, которое показывает, насколько плохо нейросеть справилась. Чем больше — тем хуже. Цель обучения: минимизировать loss.

Примеры:
- MSE (Mean Squared Error) — для регрессии
- Cross-Entropy — для классификации

**Веса (weights)** — числа в каждом нейроне, которые нейросеть подстраивает в процессе обучения. Именно в них хранится всё, что модель «знает».

**Градиент** — направление, в котором loss растёт быстрее всего. Чтобы его уменьшить, надо двигаться в **противоположную** сторону.

**Gradient Descent (градиентный спуск)** — алгоритм: обновляй веса маленькими шагами против градиента.

**Learning rate** — размер шага. Слишком большой — перепрыгнешь минимум. Слишком маленький — будешь учиться вечно.

**Backpropagation** — алгоритм для вычисления градиентов через всю сеть. По правилу цепочки (chain rule) он считает, как каждый вес во всех слоях повлиял на итоговую ошибку.

## Как это работает

На каждом шаге обучения нейросеть делает прямой проход (forward pass): данные проходят через слои и получается предсказание. Затем считается loss — насколько предсказание отличается от правильного ответа. Backpropagation считает, как каждый вес повлиял на ошибку, и обновляет веса на маленький шаг против градиента. Этот цикл повторяется для каждого батча данных, а весь датасет проходит несколько раз (эпохи).

Базовый алгоритм такого обучения — SGD (Stochastic Gradient Descent). Это один из оптимизаторов; современные сети часто обучают с Adam или RMSProp — это отдельные алгоритмы, а не варианты SGD.

**Батч и эпоха:**
- **Батч** — небольшая подвыборка данных (например, 32 примера). Считаем градиент на батче, а не на всём датасете — быстрее.
- **Эпоха** — один полный проход через все обучающие данные.
- Обычно обучают 10–100+ эпох.

## Простой пример

Нейросеть учится распознавать рукописные цифры на 60 000 изображениях. Размер батча — 64. За одну эпоху происходит 60 000 / 64 ≈ 937 обновлений весов. На первой эпохе loss = 2.31 (случайные предсказания), accuracy = 0.11. К 10-й эпохе loss = 0.08, accuracy = 0.974. Learning rate = 0.001 — маленький, но стабильный. Попытка с lr = 0.1: loss скачет и не сходится. Попытка с lr = 0.00001: сходится, но нужно 200 эпох вместо 10.

## Краткий итог

- Нейросеть учится итеративно: forward → вычислить loss → backward → обновить веса — и так тысячи раз
- Learning rate — самый критичный гиперпараметр: слишком большой → расходимость, слишком маленький → обучение встанет
- Батч — компромисс: считать градиент на всём датасете точно, но медленно; на одном примере быстро, но зашумлённо
- Эпоха = один полный проход по данным; модели нужно несколько эпох, чтобы уточнить знание
- SGD (Stochastic Gradient Descent) — один из базовых оптимизаторов; Adam и RMSProp — отдельные адаптивные алгоритмы, не варианты SGD, но работающие по той же идее итеративного обновления весов

## Ключевые термины урока

- **Loss (функция потерь)** — число, которое нейросеть стремится минимизировать. MSE для регрессии, Cross-Entropy для классификации. Нельзя использовать accuracy как loss: она не дифференцируема.
- **Градиент** — вектор направления роста loss. Важно: обновление идёт в строго противоположную сторону. Формула: `w_new = w_old − lr × ∂L/∂w`.
- **Gradient Descent** — итеративный спуск по поверхности loss. Не гарантирует нахождение даже локального минимума: поверхность loss нейросети очень сложная, и оптимизатор ищет «достаточно хорошую» точку. На практике это работает, но математических гарантий нет.
- **Learning rate** — самый критичный гиперпараметр: слишком большой → loss скачет и расходится, слишком маленький → обучение занимает сотни эпох. Типичные значения: 1e-3 (Adam) до 1e-5 (при донастройке предобученной модели).
- **Backpropagation** — вычисление градиентов для всех весов сразу через chain rule. Без него пришлось бы перебирать каждый вес по отдельности — невозможно для сетей с миллиардами параметров.
- **Эпоха** — один полный проход по датасету. Смысл нескольких эпох: каждый пример влияет на веса несколько раз, модель «уточняет» знание.
- **Батч** — компромисс между точностью градиента (весь датасет) и скоростью (один пример). Размер 32–256 — стандарт; слишком маленький батч → зашумлённые обновления, слишком большой → хуже обобщение.
