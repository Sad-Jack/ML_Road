# 94. Что учить дальше: Python, SQL, ML-практика

## Зачем это нужно

Вы прошли курс по основам ML. Это фундамент, но не конец пути. Теперь главный вопрос: что изучать дальше, чтобы стать практикующим специалистом? Этот урок — практическая дорожная карта.

## Простая интуиция

Вы выучили правила дорожного движения. Теперь нужно сесть за руль и ехать. Знание теории без практики — это красивая карта, по которой вы не ходили. Настоящее обучение начинается с первого реального проекта.

## Что это такое

Три направления, которые нужно развивать параллельно:

**1. Инструменты (что учить дальше):**
- **Python глубже**: pandas (группировки, merge, pivot), sklearn (всё что не дошли), matplotlib/seaborn
- **SQL**: SELECT, JOIN, GROUP BY, оконные функции — обязательно для работы с данными
- **Git**: ветки, pull requests — работа в команде
- **Jupyter/VS Code**: профессиональная среда разработки

**2. ML практика:**
- **Kaggle**: соревнования учат лучше любого курса
- **Датасеты**: UCI ML Repository, Hugging Face Datasets, Kaggle Datasets
- **Pet-проекты**: свой датасет + своя задача = реальный опыт

**3. Специализация (выбрать направление):**
- Data Analyst: SQL + visualization + statistics
- ML Engineer: Python + deployment + MLOps
- Data Scientist: statistics + experimentation + storytelling
- NLP/CV: специализированные курсы + DL

## Как это работает

```python
# Дорожная карта следующих 6 месяцев

roadmap = {
    "Месяц 1-2: Закрепление основ": [
        "Повторить pandas: groupby, merge, pivot_table",
        "SQL: Basic SELECT, JOINs, GROUP BY (SQLBolt, Mode Analytics)",
        "Закончить 2-3 Kaggle Learn модуля",
    ],
    "Месяц 3-4: Первый проект": [
        "Найти датасет на Kaggle или UCI ML Repository",
        "Поставить задачу (классификация или регрессия)",
        "Полный пайплайн: EDA → Feature Engineering → Модель → Оценка",
        "Опубликовать ноутбук на Kaggle",
    ],
    "Месяц 5-6: Первое соревнование и специализация": [
        "Вступить в Kaggle Getting Started соревнование (Titanic, House Prices)",
        "Изучить XGBoost/LightGBM: они побеждают в 80% табличных задач",
        "Выбрать специализацию и найти курс следующего уровня",
    ]
}

for period, tasks in roadmap.items():
    print(f"\n{period}:")
    for task in tasks:
        print(f"  ✓ {task}")

# Ресурсы
resources = {
    "Практика": ["kaggle.com/learn", "leetcode.com (SQL)", "mode.com/sql-tutorial"],
    "Датасеты": ["kaggle.com/datasets", "huggingface.co/datasets", "archive.ics.uci.edu"],
    "Книги": ["Hands-On ML (Géron)", "Python for Data Analysis (McKinney)"],
    "YouTube": ["3Blue1Brown (математика)", "StatQuest (ML интуиция)", "Andrej Karpathy (DL)"]
}
print("\nКлючевые ресурсы:")
for category, items in resources.items():
    print(f"  {category}: {', '.join(items)}")
```

## Простой пример

Студент закончил базовый курс ML в марте. План на 6 месяцев: апрель-май — освоил pandas groupby и merge на реальном датасете продаж (10 000 строк), прошёл SQLBolt до уровня оконных функций. Июнь-июль — взял датасет House Prices с Kaggle (1 460 домов, 79 признаков), построил полный пайплайн: EDA, feature engineering, Random Forest, GridSearchCV. RMSE вошёл в топ-30% лидерборда. Август-сентябрь — дообучился XGBoost, улучшил до топ-15%, получил 3 upvote на ноутбук. В октябре откликнулся на вакансию Junior Data Scientist с ссылкой на Kaggle-профиль.

## Ключевые термины урока

- **Python для ML** — основной язык: pandas, sklearn, matplotlib, numpy.
- **SQL** — язык для работы с базами данных; необходим для получения данных в большинстве компаний.
- **Git** — система контроля версий; обязательный инструмент в любой команде.
- **Kaggle** — платформа с соревнованиями и датасетами для практики ML.
- **Pet-проект** — небольшой самостоятельный ML-проект для практики и портфолио.
