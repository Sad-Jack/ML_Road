# 86. Что такое Embeddings

## Зачем это нужно

Компьютеры работают с числами, но данные бывают категориальными: слова, пользователи, товары, города. Embeddings — способ превратить любой объект в осмысленный числовой вектор. Это основа NLP, рекомендательных систем и многих других ML-задач.

## Простая интуиция

Представьте карту. Каждый город — точка с координатами (x, y). Похожие города (близкие географически или культурно) — находятся рядом на карте. Embeddings делают то же самое для слов, пользователей или товаров: превращают их в координаты в многомерном пространстве, где «похожие» объекты находятся рядом.

## Что это такое

- **Embedding** — плотный числовой вектор фиксированной размерности, представляющий объект
- **Word2Vec** — классический алгоритм получения эмбеддингов слов: слова с похожим контекстом → похожие векторы
- **Семантическое сходство** — похожие объекты имеют близкие векторы (косинусное расстояние)
- **Embedding layer** — обучаемый слой нейросети, создающий эмбеддинги категорий
- Пример: вектор("король") - вектор("мужчина") + вектор("женщина") ≈ вектор("королева")

## Как это работает

```python
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# Эмбеддинги слов (в реальности 100-300 измерений, здесь 3 для наглядности)
embeddings = {
    "кот":        np.array([0.9, 0.1, 0.2]),
    "кошка":      np.array([0.85, 0.15, 0.25]),
    "машина":     np.array([0.1, 0.9, 0.3]),
    "автомобиль": np.array([0.15, 0.85, 0.35]),
}

def sim(w1, w2):
    return cosine_similarity([embeddings[w1]], [embeddings[w2]])[0][0]

print(f"кот ↔ кошка:       {sim('кот', 'кошка'):.3f}")       # близко
print(f"кот ↔ машина:      {sim('кот', 'машина'):.3f}")      # далеко
print(f"машина ↔ автомобиль: {sim('машина', 'автомобиль'):.3f}")  # близко

# Похожие объекты → близкие векторы — это ключевое свойство embeddings
```

## Простой пример

Маркетплейс с каталогом из 200 000 товаров. Каждый товар описан текстовым описанием. One-Hot Encoding по словарю — вектор размером 50 000 (размер словаря), 99.9% нулей. Embedding (Word2Vec или sentence-transformer): плотный вектор размером 128. Запрос «беспроводные наушники Sony»: система находит эмбеддинг запроса, вычисляет косинусное сходство с 200 000 товарными эмбеддингами за 20 мс, топ-10 похожих — «наушники JBL» (0.94), «наушники Beats» (0.91), «bluetooth-колонка Sony» (0.87). Семантический поиск работает даже если точное слово не совпадает.

## Краткий итог

- Embeddings — универсальный язык для ML: одна идея лежит за текстом, рекомендациями и поиском одновременно
- Похожие объекты имеют близкие векторы (семантическое сходство)
- Используются в NLP, рекомендациях, поиске, любых задачах с категориальными данными
- Косинусное сходство — стандартная метрика близости embeddings

## Ключевые термины урока

- **Embedding** — плотный числовой вектор, представляющий объект (слово, товар, пользователя).
- **Word2Vec** — классическая модель для получения векторных представлений слов.
- **Семантическое сходство** — близость объектов в векторном пространстве отражает смысловую близость.
- **Embedding layer** — обучаемый слой нейросети, превращающий индексы в плотные векторы.
