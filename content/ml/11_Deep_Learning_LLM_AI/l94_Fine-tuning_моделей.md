# 93. Fine-tuning моделей

## Зачем это нужно

Pre-trained модели (BERT, GPT, LLama) — мощные, но общие. Fine-tuning адаптирует их к специфической задаче или домену. Это позволяет получить специализированную модель без обучения с нуля (что стоило бы миллионы долларов).

## Простая интуиция

Врач-терапевт (pre-trained модель) знает медицину в целом. Специализация в кардиологии (fine-tuning) — дополнительная учёба, опираясь на уже имеющуюся базу. Это быстрее и дешевле, чем начинать медицинское образование заново.

## Что это такое

- **Full fine-tuning** — обновление всех весов модели на новых данных. Дорого, требует много GPU памяти
- **Feature extraction** — заморозить все слои, обучить только последний (классификационный) слой
- **LoRA** (Low-Rank Adaptation) — обновить только небольшие матрицы-адаптеры; эффективно по памяти
- **Instruction tuning** — дообучение следовать инструкциям (как у ChatGPT)
- **Domain adaptation** — дообучение на специализированных текстах (медицина, юридические документы)
- Когда нужен: специфический формат вывода, специализированный домен, конкретный стиль

## Как это работает

Три подхода к fine-tuning по степени «глубины» адаптации:

```
Базовая модель (предобученные веса)
        │
        ├── Feature extraction: заморозить все слои, обучить только последний
        │   └── Быстро, мало GPU, но ограниченная гибкость
        │
        ├── LoRA: добавить небольшие матрицы-адаптеры к каждому слою
        │   └── ~1% параметров обновляется, базовые веса заморожены
        │
        └── Full fine-tuning: обновить все параметры
            └── Максимальная гибкость, но требует много GPU и риск catastrophic forgetting
```

Ключевой принцип: learning rate при fine-tuning должен быть очень маленьким (2e-5 вместо стандартных 0.001). Большой learning rate «перезапишет» предобученные веса — модель забудет всё, что знала. Маленький rate позволяет аккуратно адаптироваться к новой задаче, сохранив общие знания.

## Простой пример

Банк хочет LLM для анализа финансовой отчётности клиентов. Prompt engineering (GPT-4): 40% ошибок при интерпретации специфической банковской терминологии («EBITDA с корректировками» трактуется неверно). Fine-tuning BERT на 5 000 банковских отчётов с разметкой: ошибки падают до 8%. LoRA на Llama-3: обновляет 0.3% параметров (12M из 4B), обучение на одном A100 GPU за 3 часа, результат сопоставим с full fine-tuning (разница 1.2% в accuracy), но в 8 раз дешевле и быстрее.

## Краткий итог

- Fine-tuning — не первый шаг: сначала исчерпай prompt engineering — он быстрее и дешевле; fine-tuning когда этого недостаточно
- LoRA сделал fine-tuning доступным: достаточно потребительской GPU для адаптации моделей с миллиардами параметров
- Маленький learning rate (2e-5) защищает знания предобучения от catastrophic forgetting
- Catastrophic forgetting — главный риск full fine-tuning; LoRA минимизирует его, не меняя базовые веса

## Ключевые термины урока

- **Fine-tuning vs Prompt Engineering** — сначала исчерпай промпт-инжиниринг: бесплатно, мгновенно. Fine-tuning оправдан, когда нужен конкретный стиль/поведение, которого нельзя добиться промптом, или нужно убрать длинный контекст примеров.
- **Full fine-tuning** — обновляет все параметры; требует GPU ≥ 2× размера модели в памяти (FP16 + оптимизатор). Для GPT-3 (175B) — нереально на потребительском железе.
- **LoRA** — добавляет две малых матрицы (rank r << d) к каждому слою; обновляет только их (~1% параметров). Качество близко к full fine-tuning, VRAM уменьшается в 3–10 раз. Базовые веса заморожены.
- **Catastrophic forgetting** — при fine-tuning на узкой задаче модель «забывает» общие знания предобучения. Важно: это нельзя «откатить» без переобучения с нуля — поэтому профилактика (маленький lr 2e-5, 1–3 эпохи, LoRA) важнее лечения. Признак catastrophic forgetting: модель хорошо работает на целевой задаче, но провалилась на простых общих вопросах.
- **Instruction tuning** — fine-tuning на парах «инструкция → ответ»; обеспечивает следование инструкциям пользователя. Без instruction tuning base-модель продолжает текст, а не отвечает на вопросы.
