# 77. Как работает поисковая выдача

## Зачем это нужно

Ты каждый день пользуешься Google или Яндексом. Кажется, что поисковик «просто находит» нужные страницы. На самом деле за каждой выдачей стоит ML-модель, которая за доли секунды обрабатывает сотни сигналов и решает, что поставить первым.

## Простая интуиция

Из библиотеки в миллиард книг нужно выбрать 10 самых подходящих для конкретного человека с конкретным вопросом — и сделать это за 0.2 секунды. Это задача поисковой выдачи.

## Что это такое

Поисковая выдача — результат ранжирующей модели, которая для каждого запроса сортирует все возможные страницы по релевантности и показывает лучшие.

Три группы сигналов:
- **Сигналы запроса** — что именно ищет пользователь, какое намерение стоит за словами
- **Сигналы страницы** — насколько контент соответствует запросу, насколько авторитетна страница
- **Сигналы пользователя** — история поиска, местоположение, устройство, время суток

## Как это работает

Для каждой пары «запрос + страница» модель вычисляет score релевантности из десятков признаков:

| Признак | Пример |
|---|---|
| Совпадение слов | «iPhone 14» есть в заголовке страницы? |
| Намерение запроса | «купить» → коммерческий, «что такое» → информационный |
| Авторитетность | Сколько других сайтов ссылаются на эту страницу? |
| Свежесть | Насколько недавно обновлена страница? |
| Местоположение | Поиск «кафе» → показываем ближайшие |
| История пользователя | Ты уже был на этой странице? |

**Intent (намерение запроса)** — одна из важнейших идей. Запрос «iPhone 14» может означать разное:
- «хочу купить» → на первом месте должен быть магазин
- «хочу узнать характеристики» → на первом месте должна быть статья
- «хочу посмотреть фото» → нужна галерея

Модель учится распознавать intent и подбирает тип контента под него.

**Как модель учится**

Основной сигнал — клики пользователей. Если на запрос «ноутбук для работы» пользователи чаще кликают на второй результат, а не на первый — значит второй релевантнее. Это implicit feedback, точно как в рекомендательных системах (урок 61).

## Простой пример

Пользователь в Москве вечером вводит запрос «пицца». Поисковик определяет intent как «заказать еду рядом», а не «рецепт». Три сигнала работают одновременно: сигнал пользователя — геолокация Москва, Замоскворецкий район; сигнал запроса — слово «пицца» без уточнений → коммерческий intent; сигнал страниц — рейтинг и расстояние заведений. Топ-3 результата: три пиццерии в 500м. Тот же запрос «пицца» для пользователя без геолокации и с историей кулинарных сайтов выдаёт рецепты на первых позициях.

## Практика

1. Какие три группы сигналов использует поисковая модель? Приведи по одному примеру из каждой.
2. Почему запрос «ресторан» даст разные результаты для пользователя в Москве и Новосибирске?
3. Что такое intent запроса? Как он влияет на тип результатов на первом месте?
4. Пользователь вводит запрос и не кликает ни на один результат, а уточняет запрос. О чём это говорит системе?
5. Мини-кейс: пользователь ищет «Python». Придумай три разных намерения, которые могут стоять за этим словом, и объясни почему для каждого первый результат должен быть разным.

## Краткий итог

- Поиск давно перестал быть поиском по ключевым словам: сотни сигналов делают из него персонализированную ранжирующую систему
- Три группы сигналов: запрос, страница, пользователь
- Не угадав intent, нельзя дать правильный результат: одно слово означает разные запросы у разных людей в разный момент
- Клики пользователей — основной обучающий сигнал (implicit feedback)
- Выдача персональна: история, местоположение и устройство меняют результаты

## Ключевые термины урока

- **Поисковая выдача** — ранжированный список результатов на поисковый запрос.
- **Сигналы запроса** — характеристики текста запроса, используемые для ранжирования.
- **Сигналы страницы** — характеристики веб-страницы (контент, ссылки, авторитет).
- **Intent (намерение)** — реальная цель пользователя за поисковым запросом.
- **Персонализация** — учёт истории конкретного пользователя при формировании выдачи.
