# 62. Гибридные рекомендательные системы

## Зачем это нужно

Коллаборативная фильтрация плохо работает для новых пользователей и новых объектов. Контентная создаёт «пузырь» и не учитывает скрытые предпочтения. В реальных продуктах — Netflix, Spotify, YouTube — ни один подход не используется в одиночку. Их комбинируют.

## Простая интуиция

Netflix знает: ты смотришь много триллеров (поведение → collaborative), тебе нравятся фильмы с Нолана (характеристики → content-based), сейчас вечер пятницы и ты на телевизоре, а не на телефоне (контекст). Итоговый список рекомендаций — результат всех этих сигналов вместе.

## Что это такое

Гибридная рекомендательная система — комбинация нескольких подходов для получения более точных и разнообразных рекомендаций. Чаще всего объединяют коллаборативную и контентную фильтрацию, иногда добавляют контекстные сигналы.

## Как это работает

Три основных способа комбинировать подходы:

**1. Взвешенный** — каждый подход даёт оценку, итог — взвешенная сумма:

`финальная оценка = 0.6 × CF_оценка + 0.4 × CB_оценка`

**2. Переключающийся** — выбираем подход в зависимости от ситуации:

| Ситуация | Подход |
|---|---|
| Новый пользователь (нет истории) | Content-based |
| Новый объект (нет оценок) | Content-based |
| Есть история у пользователя и объекта | Collaborative filtering |

**3. Каскадный** — один подход фильтрует, второй ранжирует:
- Сначала CB отбирает объекты, похожие по характеристикам
- Потом CF расставляет их по вероятности понравиться конкретному пользователю

**Пример: Spotify**

| Ситуация | Что использует |
|---|---|
| Новый трек без истории прослушиваний | Анализ звука (content-based) |
| Популярный трек с историей | Поведение похожих слушателей (collaborative) |
| Discover Weekly | Смешивает CF + CB + контекст |

## Простой пример

Маркетплейс с 2 миллионами пользователей и 500 000 товаров использует переключающийся гибрид. Новый пользователь при регистрации — показывают топ продаж в выбранных категориях (популярность) и похожие товары на те, что он смотрел (content-based). После 5 покупок переключаются на коллаборативную фильтрацию: находят 300 пользователей с похожей историей покупок. Итоговый список рекомендаций: 60% из CF, 30% из content-based, 10% «новинки в твоих категориях».

## Практика

1. Почему ни коллаборативная, ни контентная фильтрация в одиночку не дают наилучший результат?
2. Какой подход лучше использовать для нового пользователя без истории и почему?
3. Spotify добавил новый трек, который никто ещё не слушал. Как гибридная система может его рекомендовать?
4. В чём разница между взвешенным и переключающимся гибридом? Когда каждый из них предпочтительнее?
5. Мини-кейс: интернет-магазин электроники. Данные: история покупок, описания товаров (категория, бренд, характеристики), текущая страница. Какой гибридный подход ты бы выбрал?

## Краткий итог

- Гибридные системы комбинируют несколько подходов — чаще всего CF + CB
- Три способа комбинировать: взвешенный, переключающийся, каскадный
- Реальные системы (Netflix, Spotify) используют десятки сигналов одновременно
- Переключающийся гибрид — хорошее решение проблемы холодного старта
- Выбор архитектуры зависит от данных, задачи и бизнес-требований

## Ключевые термины урока

- **Гибридная рекомендательная система** — система, комбинирующая несколько подходов (CF + контентный).
- **Взвешенный подход** — смешивание результатов нескольких методов с заданными весами.
- **Переключающийся подход** — выбор метода в зависимости от ситуации (например, холодный старт).
- **Каскадный подход** — один метод фильтрует кандидатов, другой их ранжирует.
