# 03. Модель: input → output

## Зачем это нужно

Любую ML-задачу можно описать простой схемой:

```
input → model → output
```

Это помогает сразу понять:

- что мы даём модели на вход;
- что модель должна вернуть на выходе;
- какую задачу мы вообще решаем.

Без этой схемы легко запутаться ещё до обучения модели.

## Простая интуиция

Модель можно представить как преобразователь. Она получает данные об объекте и возвращает результат. Например:

```
данные о квартире → модель → предсказанная цена
```

Пока не важно, как именно модель работает внутри. На этом этапе важно понять главное: что подаём на вход и что хотим получить на выходе.

## Что это такое

**Модель** — это алгоритм, который учится находить связь между входными данными и результатом. После обучения модель может получать новые данные и делать предсказание.

**Input** — то, что модель получает на вход. Это могут быть:

- данные о квартире;
- текст письма;
- изображение;
- история действий пользователя.

**Output** — то, что модель возвращает. Это может быть:

- цена;
- класс;
- вероятность;
- рекомендация;
- текстовый ответ.

## Как это работает

1. Формулируем задачу.
2. Передаём модели input — данные об объекте.
3. Модель обрабатывает эти данные.
4. На выходе получаем output — предсказание.
5. Сравниваем предсказание с реальным ответом, чтобы понять, насколько модель справилась.

Важно: output модели — это не факт. Это предположение модели на основе данных, которые она увидела.

## Простой пример

Задача: предсказать цену квартиры.

Input:

```
площадь: 55 м²
комнат: 2
этаж: 7
район: центр
```

Output:

```
предсказанная цена: 10.8 млн
```

Модель не гарантирует, что квартира точно стоит `10.8 млн`. Она делает предсказание на основе похожих примеров, которые видела раньше.

Инсайт: изменение одного элемента input меняет output — без единой правки алгоритма. Если поменять `район: центр` на `район: окраина`, модель выдаст другое число. Это значит, что поведение модели определяется не кодом, а данными на входе.

## Практика

1. В задаче «спам / не спам» что будет input, а что output?
2. Один и тот же набор данных о пользователе можно использовать для разных задач. Возьми признаки: `age`, `country`, `orders_count`, `last_login_days`. Опиши два разных output, которые можно предсказывать по этим данным — и как изменится смысл задачи.
3. Почему output модели — это предсказание, а не точный факт?
4. В уроке «Что такое машинное обучение» мы разбирали пример с квартирами. Опиши этот пример в виде схемы input → model → output: что конкретно было на входе и что на выходе?
5. В уроке «Типы обучения» мы видели три типа задач. Для каждого типа (supervised, unsupervised, RL) придумай один пример и опиши его как схему input → output: что подаётся на вход и что должно быть на выходе.

## Краткий итог

- Любую ML-задачу можно описать схемой `input → model → output`.
- Понять, что является input, важнее, чем выбрать алгоритм: неверно определённый вход делает задачу нерешаемой.
- Тип output определяет всё: задача, алгоритм, метрика — всё зависит от того, что нужно предсказать.
- Output модели — это предсказание, а не гарантия.
- Перед обучением нужно понять, что модель получает и что должна предсказать.

## Ключевые термины урока

- **Модель** — функция, принимающая input и возвращающая output. Ключевая ловушка: одна и та же модель даст разный output, если изменится состав input — даже при одном и том же алгоритме.
- **Input (вход)** — данные, которые модель получает для предсказания. Именно от состава input зависит, что вообще может выучить модель: признака нет на входе — модель его не учтёт.
- **Output (выход)** — результат, который модель возвращает. Тип output определяет весь дальнейший pipeline: как интерпретировать результат, какую метрику использовать, какая постобработка нужна (пороговая обрезка для вероятности, округление для числа). Менять формат output после обучения нельзя без переобучения модели.
- **Схема input → model → output** — универсальный фреймворк для описания любой ML-задачи. Прежде чем выбирать алгоритм, нужно ответить на два вопроса: что на входе и что должно быть на выходе.
