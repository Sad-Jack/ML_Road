# 02. Типы обучения: supervised, unsupervised, RL

## Зачем это нужно

Прежде чем строить модель, нужно понять, какой тип задачи ты решаешь. От этого зависит выбор алгоритма, способ подготовки данных и то, как ты будешь оценивать результат. Путаница в типах обучения — одна из самых частых ошибок начинающих.

## Простая интуиция

Представь, что учишь ребёнка различать кошек и собак. Если ты показываешь картинки и говоришь «это кошка, это собака» — это supervised. Если просто даёшь кучу картинок и говоришь «найди похожие» — это unsupervised. А если ребёнок учится на ошибках в игре, получая очки за правильные ходы — это reinforcement learning.

## Что это такое

Три основных парадигмы машинного обучения:

- **Supervised learning (обучение с учителем)** — у каждого примера есть метка (правильный ответ). Модель учится предсказывать метку по признакам. Пример: предсказать цену квартиры, определить спам.
- **Unsupervised learning (обучение без учителя)** — меток нет. Модель ищет структуру в данных сама. Пример: сгруппировать клиентов по поведению.
- **Reinforcement learning (обучение с подкреплением)** — агент взаимодействует со средой, получает награду или штраф и учится максимизировать награду. Пример: игра в шахматы, управление роботом.

**Метка (label)** — это правильный ответ для конкретного примера в supervised learning. То самое `y` в паре `(X, y)`. Для задачи регрессии метка — число (цена, зарплата, температура). Для классификации — категория (spam / not spam, кошка / собака / птица). Без меток supervised learning невозможен: именно из них модель узнаёт, что считать правильным ответом.

## Как это работает

В supervised learning данные выглядят так: есть таблица с описаниями объектов и столбец с правильными ответами. Модель учится на парах (описание, ответ) и потом предсказывает ответ для новых объектов.

Например, для задачи с квартирами:

```
площадь=50, этаж=3, район=центр → цена=12 млн
площадь=35, этаж=1, район=окраина → цена=6 млн
```

В unsupervised — только описания объектов, без ответов. Алгоритм сам находит группы или паттерны.

В RL нет заранее готового датасета — агент сам собирает опыт, действуя в среде.

## Практика

1. Команда собрала 1 миллион чеков из супермаркета. Бизнес хочет понять, какие продукты обычно покупают вместе, чтобы оптимизировать выкладку. Есть ли готовые метки для этой задачи — и какой тип обучения нужен?
2. У тебя есть данные о транзакциях без меток «мошенничество / не мошенничество». Какой тип обучения подойдёт? Почему?
3. Команда хочет предсказывать отток клиентов. Бизнес говорит: «У нас нет разметки — не знаем, кто ушёл специально, а кто случайно». Как это повлияет на выбор типа обучения?
4. В уроке «Что такое машинное обучение» мы говорили, что модель учится на примерах. Какие примеры нужны для supervised learning и почему без них этот тип обучения невозможен?
5. Задача: предсказать, уйдёт ли клиент (churn). Какой тип обучения подходит? Опиши, как именно будут выглядеть «правильные ответы» для обучения — что именно они должны содержать и как их получить из исторических данных.

## Краткий итог

- Supervised — основа индустрии: метки дороги, но дают чёткий контроль над тем, что нужно предсказывать
- Unsupervised полезен для разведки и предобработки: когда разметить нечего или неизвестно, что искать
- RL применяют там, где правильный ответ определяется взаимодействием: игры, робототехника, оптимизация систем
- Большинство промышленных задач — это supervised learning
- Тип обучения определяет выбор алгоритма и способ подготовки данных
- Главный вопрос: есть ли у тебя размеченные данные?

## Ключевые термины урока

- **Supervised learning (обучение с учителем)** — тип обучения, где у каждого примера есть правильный ответ. Подходит для ~90% промышленных ML-задач; главное ограничение — разметка стоит дорого и требует экспертизы.
- **Unsupervised learning (обучение без учителя)** — тип обучения без правильных ответов. Ключевая сложность: нет чёткого критерия «модель сработала хорошо» — результат оценивают косвенно или вручную.
- **Reinforcement learning (обучение с подкреплением)** — агент учится через награды и штрафы в среде. В реальных бизнес-задачах применяется редко: сложно настроить среду и долго обучать. Кстати, дообучение LLM по обратной связи человека (RLHF) — это тоже применение RL: модель получает «награду» за лучшие ответы. У него своя специфика, но принцип наград — тот же.
- **Метка (label)** — правильный ответ для объекта в supervised learning. Качество меток важнее объёма данных: зашумлённые или противоречивые метки ломают модель сильнее, чем их нехватка.
