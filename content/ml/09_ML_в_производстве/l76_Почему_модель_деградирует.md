# 74. Почему модель деградирует

## Зачем это нужно

Модели стареют. Даже идеально обученная модель через год может работать значительно хуже. Понимание механизмов деградации помогает строить устойчивые системы и правильно настраивать стратегии переобучения.

## Простая интуиция

Представьте прогноз погоды, обученный на данных 1990-х годов. Климат изменился, города разрослись, инструменты измерения улучшились. Модель «устарела» не потому что она плохая, а потому что мир изменился. Модели ML — то же самое.

## Что это такое

- **Model decay / Model staleness** — постепенное ухудшение качества модели в production
- **Concept drift** — изменение самой зависимости между X и y (поведение пользователей меняется)
- **Data drift** — изменение распределения входных данных X
- **Data quality issues** — деградация качества входных данных (пустые поля, неправильные типы)
- **Infrastructure drift** — изменения в инфраструктуре, влияющие на preprocessing

## Как это работает

```python
# Симуляция плавной деградации (data drift)
accuracy_by_month = {1: 0.92, 3: 0.90, 6: 0.85, 9: 0.78, 12: 0.70}

alert_threshold = 0.80
for month, acc in accuracy_by_month.items():
    status = "АЛЕРТ" if acc < alert_threshold else "OK"
    print(f"Месяц {month:2d}: accuracy={acc:.2f}  {status}")

# Стратегии реагирования:
# - По расписанию: переобучать раз в месяц/квартал
# - По триггеру: переобучать при превышении порога дрейфа
# - Онлайн-обучение: обновлять модель непрерывно
```

## Простой пример

Финтех-компания запустила модель кредитного скоринга в январе с F1 = 0.89. К июню F1 = 0.82 — постепенная деградация. Анализ: средний доход заявителей вырос на 18% (data drift из-за роста зарплат), но порог «надёжный заёмщик» в модели не изменился. В марте 2020 года аналогичная модель у банка рухнула за 3 недели из-за concept drift — пандемия изменила связь «стабильная работа → низкий риск» overnight. Первый случай: переобучение по расписанию раз в квартал. Второй случай: нужно было онлайн-обучение или экстренное переобучение за дни.

## Краткий итог

- Все модели деградируют со временем — это нормально и неизбежно
- Concept drift (изменение зависимости) опаснее и сложнее, чем data drift (изменение данных)
- Стратегию переобучения выбирают по скорости drift: чем быстрее меняются данные — тем чаще нужно обновлять модель
- Мониторинг качества данных (пропуски, типы) важен наравне с мониторингом метрик

## Ключевые термины урока

- **Model decay (устаревание модели)** — постепенное ухудшение качества модели со временем.
- **Concept drift** — изменение реальной зависимости между признаками и target.
- **Data drift** — изменение распределения входных данных.
- **Переобучение по расписанию** — периодическое переобучение модели на свежих данных.
- **Переобучение по триггеру** — переобучение при детекции деградации метрик.
