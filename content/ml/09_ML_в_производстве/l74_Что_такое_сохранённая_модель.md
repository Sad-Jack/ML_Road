# 73. Что такое сохранённая модель

## Зачем это нужно

Обучение модели занимает время и ресурсы. После обучения нужно сохранить результат, чтобы использовать модель без повторного обучения. Понимание того, что именно сохраняется и как загружается, обязательно для работы с production ML.

## Простая интуиция

Представьте повара, который записал все свои рецепты в книгу. Открыв книгу, любой повар может приготовить те же блюда без переобучения. Сохранённая модель — это «книга рецептов»: все параметры, которые модель «выучила», записаны и могут быть загружены позже.

## Что это такое

- **Сериализация** — процесс сохранения объекта Python (модели) в файл
- **pickle** — стандартная библиотека Python для сериализации; простой, но не самый безопасный
- **joblib** — рекомендуется для sklearn: эффективно сохраняет numpy массивы
- **ONNX** — открытый формат для обмена моделями между фреймворками (sklearn → ONNX → C++/Java)
- Что сохраняется: параметры модели (веса, пороги), гиперпараметры, preprocessing параметры

## Как это работает

Сериализация — сохранение всего объекта Python на диск в виде байтов. При загрузке байты восстанавливаются обратно в объект — со всеми параметрами, которые модель получила при обучении. Для sklearn-моделей рекомендуется joblib: он эффективно сжимает numpy-массивы, из которых состоят деревья, веса и матрицы. При сохранении Pipeline в файл попадает всё: параметры scaler, параметры encoder, параметры финальной модели. Загруженный Pipeline сразу готов к вызову predict() — без повторного fit(). Важный нюанс: файл привязан к версии sklearn. Если версия изменилась — могут возникнуть предупреждения или ошибки несовместимости.

## Простой пример

ML-команда обучает модель предсказания оттока клиентов: Pipeline из imputer, scaler и Random Forest. Обучение на 50 000 клиентов занимает 4 минуты, происходит каждое воскресенье. Результат сохраняется как `churn_model_2024_w42.pkl` (2.3 МБ). Inference-сервер загружает файл при старте (0.8 секунды), затем обрабатывает запросы. Когда sklearn обновили до 1.4, старый файл вызвал предупреждение при загрузке — пришлось переобучить и пересохранить. Теперь в файле `requirements.txt` фиксирована `scikit-learn==1.4.2`.

## Практика

1. Разработчик сохранил только `model.pkl` без scaler и передал файл коллеге. Коллега загрузил модель и запустил predict — получил неправильные результаты без ошибки. Объясните, почему это произошло и что именно нужно было сохранить.
2. В чём преимущество `joblib` перед стандартным `pickle` для sklearn моделей?
3. Вы обучили модель на sklearn 1.0. Коллега загружает её с sklearn 1.4 и видит предупреждения о несовместимости. Как предотвратить подобные проблемы в будущем?
4. В уроке «От данных до предсказания» мы разбирали, что Pipeline объединяет scaler и модель в единый объект. Почему правильно сохранять весь Pipeline, а не только модель — что потеряется, если сохранить только финальный estimator без scaler?
5. В уроке «Training vs Inference» мы разбирали разницу между обучением и предсказанием. Что конкретно содержится в файле после сохранения Pipeline и почему загруженный объект сразу готов к predict() без повторного fit()?

## Краткий итог

- Используйте joblib.dump() для сохранения sklearn моделей
- Сохраняйте Pipeline (включает scaler + модель) как единый объект
- Версионируйте модели и фиксируйте версию sklearn
- Проверяйте корректность загруженной модели на нескольких примерах

## Ключевые термины урока

- **Сериализация** — сохранение обученного объекта Python на диск. В файл попадает всё: параметры модели, гиперпараметры, параметры scaler — всё, что нужно для predict() без повторного fit().
- **joblib** — рекомендуется вместо pickle для sklearn, потому что эффективно сжимает numpy-массивы. Для больших Random Forest разница в размере файла может быть в разы.
- **pickle** — универсальный сериализатор Python, но небезопасен: загрузка файла из недоверенного источника может выполнить произвольный код.
- **ONNX** — промежуточный формат для экспорта модели из Python в C++, Java, мобильные устройства. Используется, когда inference нужен вне Python-окружения.
- **Версионирование моделей** — фиксация версии sklearn вместе с моделью обязательна: файл, сохранённый на sklearn 1.0, может не загрузиться на sklearn 1.4.
