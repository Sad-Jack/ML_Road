# 66. Как выглядит ML-проект

## Зачем это нужно

Большинство учебников по ML учат алгоритмам, но не показывают, как выглядит настоящий ML-проект. В реальности обучение модели — это лишь малая часть работы. Понимание полного жизненного цикла проекта критично для любого специалиста.

## Простая интуиция

Думайте о ML-проекте как о строительстве дома. Архитектор (постановка задачи) важнее, чем выбор гвоздей (алгоритм). Фундамент (данные) определяет всё остальное. И дом нужно не просто построить, но заселить жильцов и обслуживать (деплоймент и мониторинг).

## Что это такое

Типичный ML-проект включает этапы:

1. **Постановка задачи** — перевод бизнес-вопроса в ML-задачу. Что предсказываем? Какая метрика успеха?
2. **Сбор и анализ данных** — откуда берём данные, EDA, качество данных
3. **Feature Engineering** — создание и отбор признаков
4. **Моделирование** — выбор алгоритма, обучение, валидация
5. **Оценка** — метрики, error analysis
6. **Деплоймент** — упаковка модели в API или сервис
7. **Мониторинг** — отслеживание качества в production

## Как это работает

```python
import pandas as pd
from sklearn.dummy import DummyClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

# 1. Загрузка данных
df = pd.read_csv("data.csv")
X = df.drop("target", axis=1)
y = df["target"]

# 2. Baseline — всегда начинайте с простейшей модели
baseline = DummyClassifier(strategy="most_frequent")
print("Baseline:", cross_val_score(baseline, X, y, cv=5).mean().round(3))

# 3. Реальная модель — сравниваем с baseline
model = RandomForestClassifier(random_state=42)
print("RandomForest:", cross_val_score(model, X, y, cv=5).mean().round(3))
```

## Простой пример

Ритейл-компания хочет предсказывать спрос на товары. Постановка задачи: регрессия — предсказать объём продаж на следующие 7 дней по каждой позиции. Данные: 2 года истории продаж по 500 товарам. Feature engineering: день недели, месяц, праздники, тренд, скользящее среднее. Baseline (DummyRegressor со средним значением): MAE 47 единиц. Случайный лес: MAE 18 единиц — в 2.6 раза лучше baseline. Деплоймент: API, обновление предсказаний каждое воскресенье. Мониторинг: еженедельное сравнение предсказанного и реального спроса.

## Краткий итог

- Проект не закончен, пока модель не задеплоена и не мониторится: обучение — лишь середина пути
- 80% времени уходит на данные, 20% — на моделирование
- Всегда начинайте с baseline и простых решений
- Большинство ML-проектов умирают не из-за плохой точности, а из-за плохого деплоя и отсутствия мониторинга

## Ключевые термины урока

- **Постановка задачи** — первый шаг: определить, что именно мы предсказываем и зачем.
- **Деплоймент** — публикация обученной модели для использования в реальном продукте.
- **Мониторинг** — регулярная проверка качества модели после деплоя.
- **ML-цикл** — итеративный процесс от задачи до мониторинга и обратно.
- **80/20 правило** — около 80% времени в ML-проекте уходит на данные, 20% — на модели.
