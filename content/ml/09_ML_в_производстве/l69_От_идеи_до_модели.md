# 67. От идеи до модели

## Зачем это нужно

Между бизнес-идеей («хотим предсказывать отток клиентов») и работающей моделью — много шагов. Пропуск любого из них ведёт к плохому результату. Этот урок — практическая карта пути от постановки задачи до первого рабочего прототипа.

## Простая интуиция

Вы хотите приготовить новое блюдо. Сначала — понять, что это за блюдо (постановка задачи). Потом — купить ингредиенты и проверить их свежесть (данные). Затем — сначала приготовить простую версию по базовому рецепту (baseline), и только потом — улучшать.

## Что это такое

Путь от идеи до модели включает:

1. **Формализация**: какой тип задачи? (классификация/регрессия/кластеризация). Что такое «успех»?
2. **Данные**: какие есть данные? Есть ли разметка? Достаточно ли объём?
3. **Baseline**: самое простое решение. DummyClassifier, линейная регрессия, среднее
4. **Feature Engineering**: создание информативных признаков из сырых данных
5. **Итерации**: улучшаем baseline, смотрим на ошибки, добавляем признаки, пробуем другие алгоритмы
6. **Валидация**: честная оценка через hold-out или кросс-валидацию

## Как это работает

```python
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.dummy import DummyClassifier
from sklearn.linear_model import LogisticRegression

# Шаг 1: Стратифицированный split
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)

# Шаг 2: Baseline
baseline = DummyClassifier(strategy="most_frequent")
print("Baseline:", cross_val_score(baseline, X_train, y_train, cv=5).mean().round(3))

# Шаг 3: Простая модель
model = LogisticRegression()
print("LogReg:", cross_val_score(model, X_train, y_train, cv=5).mean().round(3))

# Если прогресс есть → пробуем более сложные модели
# Если нет → смотрим на данные и признаки, а не на алгоритм
```

## Простой пример

HR-стартап хочет предсказывать, примет ли кандидат оффер. Данные: 3 000 исторических офферов (принял/отказал), 15 признаков. Формализация: бинарная классификация, метрика — F1-score. Baseline (DummyClassifier, всегда «примет»): accuracy 68%, F1 = 0.81 для класса «примет», F1 = 0 для «откажет» — бесполезная модель. Логистическая регрессия с 3 ключевыми признаками: F1 для «откажет» уже 0.42. Random Forest (50 признаков после feature engineering): F1 = 0.61. Итог: каждый шаг измерен, понятно куда двигаться.

## Краткий итог

- Путь от идеи до модели: формализация → данные → baseline → итерации → валидация
- Всегда начинайте с простого решения и постепенно усложняйте
- Feature engineering часто важнее выбора алгоритма
- Оценивайте прогресс на каждом шаге через кросс-валидацию

## Ключевые термины урока

- **Формализация** — перевод бизнес-задачи в ML-постановку с чёткими входами и выходами. Включает выбор типа задачи (классификация/регрессия), метрики и критерия остановки.
- **Baseline** — точка отсчёта: без неё невозможно оценить, приносит ли сложная модель реальную пользу. DummyClassifier или среднее значение как baseline — норма, не признак слабости.
- **Feature engineering** — создание и отбор признаков из сырых данных. Добавление одного информативного признака часто даёт больший прирост, чем смена алгоритма.
- **Валидация** — защита от переобучения: оцениваем качество на данных, которые модель не видела при обучении. Hold-out или кросс-валидация зависит от размера выборки.
