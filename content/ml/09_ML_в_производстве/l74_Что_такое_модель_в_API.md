# 72. Что такое модель в API

## Зачем это нужно

Jupyter notebook с моделью — это не продукт. Чтобы другие системы и пользователи могли использовать модель, её нужно завернуть в API. REST API — стандартный способ предоставить доступ к ML-модели через HTTP-запросы.

## Простая интуиция

Ресторан — это API. Вы (клиент) делаете заказ (запрос), кухня (модель) готовит блюдо (предсказание), официант приносит результат (ответ). Вам не нужно знать, как работает кухня — вы просто делаете заказ в стандартном формате (меню = API-документация).

## Что это такое

- **HTTP** — протокол передачи данных в интернете: клиент отправляет запрос, сервер возвращает ответ. Стандарт для общения между любыми сервисами
- **JSON** — текстовый формат данных, понятный любому языку программирования: как Python-словарь, но универсальный (`{"age": 35, "income": 80000}`)
- **REST API** — стандарт организации HTTP-интерфейса для взаимодействия между системами
- **Endpoint** — URL, по которому доступен сервис (например, /predict)
- **Request** — входные данные в формате JSON
- **Response** — предсказание модели в формате JSON
- **FastAPI** — современный Python-фреймворк для создания API; быстрый, с автодокументацией
- **Flask** — более простой Python-фреймворк для API

## Как это работает

```python
# Пример API с FastAPI
from fastapi import FastAPI
from pydantic import BaseModel
import joblib
import numpy as np

app = FastAPI()

# Загружаем обученную модель при старте сервера
model = joblib.load("model_pipeline.pkl")

# Схема входных данных
class PredictionRequest(BaseModel):
    age: float
    income: float
    credit_score: float

# Схема ответа
class PredictionResponse(BaseModel):
    prediction: int
    probability: float

@app.post("/predict", response_model=PredictionResponse)
def predict(request: PredictionRequest):
    # Создаём массив признаков из запроса
    features = np.array([[request.age, request.income, request.credit_score]])

    # Предсказание
    prediction = int(model.predict(features)[0])
    probability = float(model.predict_proba(features)[0][1])

    return PredictionResponse(prediction=prediction, probability=probability)

# Запуск: uvicorn app:app --host 0.0.0.0 --port 8000
# Документация автоматически: http://localhost:8000/docs
```

## Простой пример

Страховая компания создаёт API для оценки рисков: endpoint `/predict` принимает JSON с 8 полями (возраст, стаж вождения, марка авто, город, история ДТП и др.), возвращает страховую премию и уровень риска. Мобильное приложение, веб-сайт и CRM брокеров — все три системы делают запросы к одному и тому же endpoint. Модель обновляется раз в квартал: новый `model.pkl` загружается на сервер, перезапускать три клиентских системы не нужно. Среднее время ответа API — 23 мс.

## Краткий итог

- Обернуть модель в REST API — минимум для production: так её могут вызвать любые сервисы независимо от языка
- Загружайте модель один раз при старте сервера
- Автодокументация FastAPI упрощает интеграцию: другие команды сразу видят, как работает API, без отдельной документации
- API принимает JSON запрос, возвращает JSON с предсказанием

## Ключевые термины урока

- **REST API** — стандартный «контракт» между ML-моделью и любым сервисом: независимо от языка клиента (Python, Java, iOS), HTTP + JSON всегда понятен. Альтернатива: gRPC — быстрее, но сложнее; выбирается при очень строгих latency-требованиях.
- **Stateless** — каждый запрос самодостаточен; сервер не помнит предыдущих. Это упрощает горизонтальное масштабирование (запустить 10 копий сервера) и перезапуск без потери состояния.
- **Загрузка модели при старте** — model = pickle.load() один раз в startup-событии, не в каждом запросе. Загрузка большой модели при каждом запросе = latency 5–30 с вместо 5–30 мс.
- **Валидация входа (Pydantic)** — FastAPI + Pydantic проверяют типы и диапазоны до вызова модели. Без валидации: None/NaN попадает в модель → исключение или тихое неправильное предсказание в production.
