# 73. Почему модель в проде ≠ модель на обучении

## Зачем это нужно

Модель показала отличные результаты на тесте, но в production работает хуже. Это одна из самых частых и болезненных проблем в ML. Понимание причин этого разрыва позволяет предотвращать его или быстро диагностировать.

## Простая интуиция

Представьте повара, который учился готовить по летним рецептам с сезонными овощами. Когда наступила зима, доступные ингредиенты изменились — блюда получаются не такими, как в учебе. Модель «учила» на одних данных, но мир вокруг неё изменился.

## Что это такое

- **Training-serving skew** — несоответствие между данными при обучении и данными в production
- **Distribution shift** — изменение статистических свойств данных со временем
- **Concept drift** — изменение самой зависимости между X и y (что было верно год назад, теперь не так)
- **Data drift** — изменение распределения входных признаков X без изменения зависимости
- **Pipeline skew** — разная логика preprocessing при обучении и inference

## Как это работает

Training-serving skew возникает, когда preprocessing при обучении отличается от preprocessing при inference: например, scaler обучался на одном наборе данных, а в production применяется к другому. Решение — один Pipeline для обоих этапов. Data drift — когда само распределение входных данных меняется со временем: пользователи становятся старше, меняется поведение, приходит новая аудитория. Concept drift опаснее: меняется сама зависимость между признаками и целевой переменной. Data drift можно обнаружить, сравнивая статистику входных признаков (среднее, медиана, доля категорий) с обучающими данными — без наличия меток. Concept drift виден только по падению метрик качества.

## Простой пример

Банк обучает модель кредитного скоринга в январе на данных 2023 года. В марте запускает её в production. К декабрю F1-score падает с 0.84 до 0.71. Анализ показывает: средний возраст заявителей вырос с 34 до 41 года (data drift — переехали в онлайн другие группы клиентов), а связь «доход → платёжеспособность» изменилась из-за инфляции (concept drift). Мониторинг входных признаков поймал data drift в октябре, но concept drift обнаружили только по падению метрик в ноябре — на него нет прямых сигналов без меток.

## Практика

1. Модель кредитного скоринга обучена в 2022 году. В 2024 году распределение доходов заявителей сильно изменилось, но связь «доход → платёжеспособность» осталась прежней. Какой тип изменения это? Что нужно сделать: переобучить модель или исправить preprocessing?

## Краткий итог

- Модель в production деградирует из-за drift: данные и зависимости меняются со временем
- Training-serving skew — незаметная ловушка: модель хороша на тестах, но ломается в production из-за разного preprocessing
- Используйте один Pipeline объект для обучения и предсказания
- Мониторьте распределения входных данных для раннего обнаружения drift

## Ключевые термины урока

- **Training-serving skew** — расхождение между preprocessing при обучении и при inference. Лечится единственным способом: один Pipeline объект на оба этапа.
- **Distribution shift** — общий термин для любого изменения данных со временем. Включает два подвида: data drift и concept drift, которые требуют разных реакций.
- **Concept drift** — изменилось P(y|X): та же квартира с теми же признаками теперь стоит иначе. Нельзя обнаружить без реальных меток — только по падению метрик.
- **Data drift** — изменилось P(X): признаки сдвинулись, но зависимость та же. Можно обнаружить мониторингом входных данных без меток.
