# 68. От данных до предсказания

## Зачем это нужно

Модель — это не только .fit() и .predict(). Между сырыми данными и финальным предсказанием — цепочка трансформаций. Если нарушить её порядок, получим неправильные предсказания или data leakage. Pipeline в sklearn решает эту проблему автоматически.

## Простая интуиция

Думайте о конвейере на заводе. Сырой материал проходит через последовательность станков: очистка → нарезка → сборка → проверка качества. Каждый станок делает своё, порядок важен. ML-пайплайн — тот же конвейер: данные → imputer → scaler → модель.

## Что это такое

- **Pipeline** — последовательность трансформаций + финальная модель
- **Трансформеры** — шаги, которые делают fit_transform (scaler, imputer, encoder)
- **Estimator** — финальный шаг (модель), делающий fit и predict
- Ключевое правило: трансформеры должны обучаться ТОЛЬКО на train данных
- **ColumnTransformer** — применяет разные трансформации к разным столбцам

## Как это работает

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.ensemble import RandomForestClassifier

# Pipeline: последовательность шагов, fit_transform → fit_transform → fit
pipeline = Pipeline([
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", StandardScaler()),
    ("model", RandomForestClassifier())
])

# fit: все трансформеры обучаются ТОЛЬКО на train
pipeline.fit(X_train, y_train)
# predict: к test применяется только transform (параметры уже зафиксированы)
pipeline.predict(X_test)

# Для разных типов признаков — ColumnTransformer:
# num_cols → imputer + scaler
# cat_cols → imputer + OneHotEncoder
# Оба передаются в Pipeline как единый preprocessor
```

## Простой пример

Команда строит модель оценки кредитного риска: 10 числовых признаков и 3 категориальных (профессия, регион, тип кредита). Без Pipeline: scaler обучен на всём датасете (data leakage), OneHotEncoder применён отдельно, при деплое забыли передать imputer. Результат: модель в production выдаёт ошибки при NaN. С Pipeline: один объект (imputer → ColumnTransformer → LogisticRegression) обучается только на train, сохраняется в файл, загружается в API. Новый клиент с пропущенным полем обрабатывается корректно — imputer внутри pipeline заполняет пропуск медианой.

## Краткий итог

- Pipeline соединяет preprocessing и модель в единый объект
- Трансформеры обучаются только на train, к test применяется только transform
- ColumnTransformer внутри Pipeline гарантирует: preprocessing обучается только на train и автоматически применяется к test
- Обученный Pipeline можно сохранить и использовать в production

## Ключевые термины урока

- **Pipeline** — объект, объединяющий preprocessing и модель в единую цепочку.
- **Трансформер** — шаг Pipeline, преобразующий данные (scaler, encoder, imputer).
- **Estimator** — финальный шаг Pipeline с методами fit и predict.
- **ColumnTransformer** — трансформер для применения разных шагов к разным колонкам.
