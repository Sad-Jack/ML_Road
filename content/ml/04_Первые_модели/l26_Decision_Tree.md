# 26. Decision Tree

## Зачем это нужно

Decision Tree — первая модель, которая «думает по-человечески»: задаёт вопросы и ветвится. Она важна не только сама по себе, но и как строительный блок для Random Forest и Gradient Boosting — самых популярных алгоритмов для табличных данных.

## Простая интуиция

Дерево решений — как игра «Двадцать вопросов»: возраст > 35? Да → один путь. Нет → другой. Город = «Berlin»? Да → один лист, нет → другой. Так шаг за шагом доходим до ответа. Каждый вопрос — это разбиение данных на две группы.

## Что это такое

**Decision Tree** — модель, которая строит цепочку условий (если-то) для предсказания. На каждом узле выбирается признак и порог, по которому данные делятся так, чтобы группы стали «чище» (метрика: gini, entropy для классификации; MSE для регрессии).

Работает и для классификации (`DecisionTreeClassifier`), и для регрессии (`DecisionTreeRegressor`). Ловит нелинейные зависимости и взаимодействия признаков «из коробки» — в отличие от линейной регрессии.

## Как это работает

```python
from sklearn.tree import DecisionTreeClassifier, plot_tree
from sklearn.metrics import accuracy_score
import matplotlib.pyplot as plt

model = DecisionTreeClassifier(
    max_depth=4,            # максимальная глубина
    min_samples_leaf=20,    # минимум объектов в листе — против переобучения
    random_state=42,
)
model.fit(X_train, y_train)
print(accuracy_score(y_val, model.predict(X_val)))

# Визуализация
plt.figure(figsize=(14, 8))
plot_tree(model, feature_names=X_train.columns, filled=True, fontsize=8)
plt.show()

# Важность признаков
for name, imp in zip(X_train.columns, model.feature_importances_):
    print(f"{name}: {imp:.3f}")
```

Ключевые гиперпараметры:
- `max_depth` — максимальная глубина (главный регулятор сложности);
- `min_samples_leaf` — минимум объектов в листе;
- `min_samples_split` — минимум для разбиения узла;
- `criterion` — `gini` или `entropy`.

## Простой пример

Без ограничений дерево вырастает до тех пор, пока не запомнит все примеры train: train accuracy 1.0, val accuracy 0.70 — классический overfitting. Лекарство — ограничить `max_depth` (обычно 3–10) или увеличить `min_samples_leaf`.

Деревьям не нужно масштабирование признаков — они делят по порогам, а не считают расстояния.

## Краткий итог

- Decision Tree ценен как самостоятельный интерпретируемый инструмент и как строительный блок для ансамблей.
- Ловит нелинейности и взаимодействия признаков без feature engineering.
- Главные параметры: `max_depth`, `min_samples_leaf`.
- Не нужен StandardScaler — работает с порогами.
- Сам по себе нестабилен, зато основа для RF и Gradient Boosting.

## Ключевые термины урока

- **Decision Tree (дерево решений)** — модель, строящая цепочку условий «если-то» для предсказания.
- **max_depth** — максимальная глубина дерева; ограничивает сложность и борется с overfitting.
- **Лист** — конечный узел дерева с итоговым предсказанием.
- **Сплит** — условие, по которому дерево делит данные на группы.
