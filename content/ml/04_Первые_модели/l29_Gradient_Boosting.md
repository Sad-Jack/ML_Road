# 28. Gradient Boosting

## Зачем это нужно

Random Forest строит деревья независимо и усредняет. Gradient Boosting делает иначе: деревья строятся последовательно, каждое новое **исправляет ошибки предыдущих**. Этот подход почти всегда даёт лучшее качество на табличных данных — и поэтому именно бустинг лежит в основе XGBoost, LightGBM, CatBoost (урок 29).

## Простая интуиция

Представь команду экспертов, которые работают по очереди. Первый делает грубый прогноз. Второй смотрит, где первый ошибся, и пытается исправить именно эти ошибки. Третий — то, что пропустили первые двое. Каждый шаг — маленькое уточнение. В сумме получается очень точный ответ.

## Что это такое

**Gradient Boosting** — последовательный ансамбль слабых моделей (обычно неглубоких деревьев). На каждом шаге новое дерево обучается предсказывать **остатки** (ошибки) текущего ансамбля. Финальное предсказание — сумма всех деревьев, помноженных на `learning_rate`.

| | Random Forest | Gradient Boosting |
|---|---|---|
| Как строятся деревья | независимо, параллельно | последовательно |
| Цель каждого дерева | предсказывать y | предсказывать остаток предыдущих |
| Итог | усреднение / голосование | сумма с весом `learning_rate` |
| Глубина деревьев | глубокие | мелкие (3–8) |
| Риск overfitting | низкий | выше — нужна аккуратная настройка |

## Как это работает

```python
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.metrics import accuracy_score

model = GradientBoostingClassifier(
    n_estimators=300,
    learning_rate=0.05,
    max_depth=3,
    subsample=0.8,         # стохастический бустинг — против overfitting
    random_state=42,
)
model.fit(X_train, y_train)
print(accuracy_score(y_val, model.predict(X_val)))
```

Главное правило настройки: **`n_estimators` и `learning_rate` связаны**. Меньший lr → нужно больше деревьев, но качество стабильнее. Стандарт: `lr=0.05–0.1`, `n_estimators=200–1000`, `max_depth=3–6`.

Если `n_estimators` слишком большое и нет early stopping — модель переобучится. Поэтому в проде используют XGBoost / LightGBM / CatBoost: у них есть `early_stopping_rounds` по val-метрике.

## Простой пример

Регрессия. Первое дерево предсказало 100, реальный y = 130 → остаток 30. Второе дерево обучается предсказывать остатки, выдаёт 20. Сумма: 100 + 0.1 * 20 = 102 (с `lr=0.1`). Следующее дерево — предсказывает остаток 28. И так далее. Каждое дерево «доводит» ансамбль ближе к цели.

3. Почему деревья в бустинге обычно мелкие (`max_depth=3–6`), тогда как в Random Forest деревья глубокие?

## Краткий итог

- Gradient Boosting: деревья строятся последовательно, каждое исправляет ошибки предыдущих.
- Слабые модели (мелкие деревья) + маленький `learning_rate` + много итераций.
- Чаще выигрывает у Random Forest на табличных данных.
- Требует настройки и early stopping — иначе overfitting.
- Основа XGBoost / LightGBM / CatBoost (следующий урок).

## Ключевые термины урока

- **Gradient Boosting** — ансамбль, где деревья строятся последовательно, каждое исправляет ошибки предыдущих.
- **Остатки (residuals)** — ошибки текущей модели, которые следующее дерево пытается исправить.
- **learning_rate** — шаг обновления; маленький шаг + много деревьев = точнее.
- **n_estimators** — число деревьев в ансамбле.
