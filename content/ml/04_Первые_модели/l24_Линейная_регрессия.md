# 23. Линейная регрессия

## Зачем это нужно

Линейная регрессия — базовая модель для предсказания чисел и одна из самых интерпретируемых. Через неё удобно понять главные концепции ML: что такое веса, что значит «обучить модель», как `model.fit()` устроен внутри. Хороший первый baseline для любой регрессионной задачи.

## Простая интуиция

Модель ищет прямую (или плоскость), которая лучше всего проходит через точки данных. Если опыт 1 год → 70 000, 2 года → 90 000, 3 года → 110 000 — модель замечает паттерн «+20 000 за каждый год» и строит линейную зависимость.

## Что это такое

**Линейная регрессия** предсказывает число как линейную комбинацию признаков:

```
y = w1*x1 + w2*x2 + ... + wn*xn + b
```

где `w` — веса (насколько каждый признак влияет на ответ), `b` — смещение (intercept, базовое значение). Обучение — это подбор `w` и `b` так, чтобы минимизировать MSE на train.

## Как это работает

```python
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_absolute_error

X_train = df_train[["experience_years", "english_level"]]
y_train = df_train["salary"]

model = LinearRegression()
model.fit(X_train, y_train)           # подбирает w и b методом наименьших квадратов
y_pred = model.predict(X_val)

print("coef_:    ", model.coef_)       # например, [20000, 15000]
print("intercept:", model.intercept_)  # например, 30000
print("MAE:      ", mean_absolute_error(y_val, y_pred))
```

После обучения формула: `salary ≈ 20000 * experience + 15000 * english + 30000`. Для человека с 3 годами опыта и уровнем 4: `20000*3 + 15000*4 + 30000 = 150 000`.

## Простой пример

Когда линейная регрессия не подходит — нелинейные зависимости. Например, цена дома vs площадь обычно растёт сублинейно (250 м² не в 5 раз дороже 50 м²). Тогда либо добавить нелинейные признаки (`area**2`, `log(area)`), либо взять более гибкую модель (дерево, бустинг). Также чувствительна к выбросам и масштабу — для регуляризованных вариантов (Ridge, Lasso) обязательно `StandardScaler`.

## Краткий итог

- Линейная регрессия предсказывает число как `w·x + b`.
- Веса (`coef_`) показывают влияние каждого признака.
- `fit()` и `predict()` — единый интерфейс всего sklearn: выучи однажды — работает для любого алгоритма.
- Только для регрессии. Для классификации — Logistic Regression.
- Простая, быстрая, интерпретируемая — отличный baseline.

## Ключевые термины урока

- **Линейная регрессия** — единственная модель из всех изученных, где каждый `coef_` читается как «на сколько изменится предсказание при +1 к этому признаку»; именно это делает её незаменимой для интерпретации.
- **Веса (coef_)** — не просто числа, а бизнес-объяснение: `coef_[experience]=20000` значит «один год опыта → +20 000 ₽ к зарплате».
- **fit()** — единственный момент, когда модель «видит» target; после него веса зафиксированы и не меняются при predict().
- **predict()** — простое перемножение: `w·x + b`; никакого обучения, только арифметика по уже найденным весам.
