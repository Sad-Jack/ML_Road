# 25. Support Vector Machine простыми словами

## Зачем это нужно

SVM — одна из классических моделей ML, которую важно понимать: она отлично работает на небольших и средних датасетах, устойчива к переобучению при правильной настройке и даёт чёткую геометрическую интерпретацию. Сегодня её часто заменяют деревьями и бустингом, но понять SVM — значит глубже понять, что значит «хорошая граница классификации».

## Простая интуиция

Представь: на листе бумаги разбросаны красные и синие точки. Ты хочешь провести линию, которая их разделяет. Таких линий бесконечно много — но какая **лучшая**?

SVM отвечает: та, которая проходит **как можно дальше от всех точек**. Это называется **максимизацией margin** — зазора между линией и ближайшими точками каждого класса.

```
  Синие  |       | Красные
    ●    |  gap  |   ●
    ●    |       |   ●
    ●    ←margin→   ●
```

Точки, которые лежат прямо на границе зазора — это **support vectors** (опорные векторы). Только они определяют положение линии. Всё остальное — неважно.

## Что это такое

SVM (Support Vector Machine) — алгоритм обучения с учителем для классификации (и регрессии). Он находит **оптимальную гиперплоскость** — разделяющую поверхность в пространстве признаков, которая максимизирует зазор (margin) между классами.

Если данные не разделяются линейно — SVM использует **kernel trick**: преобразует данные в пространство более высокой размерности, где линейное разделение возможно.

Популярные ядра (kernels):
- **Linear** — линейное разделение
- **RBF (Radial Basis Function)** — для нелинейных границ, самый частый
- **Polynomial** — полиномиальная граница

## Как это работает

1. Строим гиперплоскость, которая делит классы
2. Находим точки, ближайшие к этой гиперплоскости (support vectors)
3. Максимизируем зазор между гиперплоскостью и этими точками
4. Параметр **C** регулирует баланс: маленький C — мягкие границы (больше ошибок, меньше переобучение), большой C — жёсткие границы

```python
from sklearn.svm import SVC

model = SVC(kernel='rbf', C=1.0)
model.fit(X_train, y_train)
```

## Простой пример

Медицинская лаборатория классифицирует опухоли по 30 признакам (размеры клеток, текстура ткани) на 569 образцах. Датасет небольшой — SVM с RBF-ядром подходит идеально. После StandardScaler: SVC(kernel='rbf', C=1.0) даёт accuracy 97.4% против Random Forest 96.1%. Support vectors — всего 89 из 455 обучающих примеров: модель опирается только на «пограничные» случаи, игнорируя очевидно доброкачественные/злокачественные опухоли. Именно это делает SVM устойчивым к переобучению при малом датасете.

## Краткий итог

SVM находит разделяющую гиперплоскость с максимальным зазором между классами. Support vectors — точки, которые её определяют. Kernel trick позволяет работать с нелинейными границами. Чувствителен к масштабу, хорошо работает на малых датасетах с многими признаками, но сегодня в большинстве задач уступает бустингу.

## Ключевые термины урока

- **SVM (Support Vector Machine)** — модель, ищущая разделяющую гиперплоскость с максимальным зазором.
- **Margin (зазор)** — расстояние от разделяющей гиперплоскости до ближайших точек каждого класса.
- **Support vectors** — опорные точки, которые определяют положение разделяющей плоскости.
- **Kernel trick** — приём, позволяющий SVM работать с нелинейными границами.
