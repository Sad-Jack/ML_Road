# 26. Support Vector Machine простыми словами

## Зачем это нужно

SVM — одна из классических моделей ML, которую важно понимать: она отлично работает на небольших и средних датасетах с большим числом признаков, устойчива к переобучению при правильной настройке и даёт чёткую геометрическую интерпретацию. Понять SVM — значит глубже понять, что значит «хорошая граница классификации».

## Простая интуиция

Представь: на листе бумаги разбросаны красные и синие точки. Ты хочешь провести линию, которая их разделяет. Таких линий бесконечно много — но какая **лучшая**?

SVM отвечает: та, которая проходит **как можно дальше от всех точек**. Чем шире коридор между линией и ближайшими точками с обеих сторон, тем меньше шансов ошибиться на новых данных.

```
  Синие  |       | Красные
    ●    |  gap  |   ●
    ●    |       |   ●
    ●    ←коридор→  ●
```

Интересно другое: положение линии определяют только точки, стоящие прямо у края коридора. Остальные — даже если их тысячи — вообще не влияют на результат.

## Что это такое

SVM (Support Vector Machine) — алгоритм обучения с учителем для классификации (и регрессии). Он находит **оптимальную гиперплоскость** — разделяющую поверхность в пространстве признаков, которая максимизирует зазор (margin) между классами.

Если данные не разделяются линейно — SVM использует **kernel trick**: преобразует данные в пространство более высокой размерности, где линейное разделение возможно.

Популярные ядра (kernels):
- **Linear** — линейное разделение
- **RBF (Radial Basis Function)** — для нелинейных границ, самый частый
- **Polynomial** — полиномиальная граница

## Как это работает

1. Строим гиперплоскость, которая делит классы
2. Находим точки, ближайшие к этой гиперплоскости (support vectors)
3. Максимизируем зазор между гиперплоскостью и этими точками
4. Параметр **C** регулирует баланс: маленький C — мягкие границы (больше ошибок, меньше переобучение), большой C — жёсткие границы

## Простой пример

Медицинская лаборатория классифицирует опухоли по 30 признакам (размеры клеток, текстура ткани) на 569 образцах. Датасет небольшой — SVM с RBF-ядром дал accuracy 97.4%.

Но интереснее другое: support vectors — всего 89 из 455 обучающих примеров. Модель полностью игнорирует очевидно доброкачественные и злокачественные опухоли и строит границу только по «пограничным» случаям. Именно это делает SVM устойчивым к переобучению на малом датасете: меньше опорных точек — меньше шансов зазубрить конкретные примеры.

## Практика

1. Логистическая регрессия и SVM оба нашли разделяющие границы для одного датасета. Логрег прошёл через середину данных, SVM — через зону максимального зазора. Оба дали val score 0.88. Почему через месяц на новых клиентах SVM скорее всего ошибётся реже — хотя обе модели «правы» на текущих данных?
2. Из урока «Overfitting и Underfitting» мы знаем, что модель может запомнить обучающие данные и плохо работать на новых. Как параметр C в SVM влияет на этот баланс — и что произойдёт, если выставить очень большое C?
3. Два датасета: первый — 10 000 примеров с 5 признаками (покупки, возраст, доход); второй — 300 примеров с 50 медицинскими показателями. Для какого из них SVM предпочтительнее и почему?
4. SVM с linear kernel на задаче распознавания рукописных цифр дал val accuracy = 0.72. После переключения на RBF — 0.94. Что это говорит о структуре данных и почему linear kernel не справился, а RBF — справился?
5. Из урока «Overfitting и Underfitting» мы знаем, как диагностировать проблему по разрыву train/val. SVM с RBF-ядром: train accuracy = 0.99, val accuracy = 0.61. Назови два вероятных объяснения такого разрыва и предложи конкретное изменение параметров для каждого.

## Краткий итог

- Нужна модель для задачи с небольшим датасетом и большим числом признаков — SVM хорошо подходит.
- Данные линейно неразделимы — используйте RBF-ядро; данные разделяются прямой — linear.
- Признаки с разным масштабом ухудшат качество — нормируйте данные перед SVM.
- Параметр C регулирует жёсткость: маленький C = допускает ошибки на train, но лучше обобщает на новых данных.

## Ключевые термины урока

- **SVM (Support Vector Machine)** — в отличие от логистической регрессии, ищет не любую разделяющую плоскость, а ту, которая максимально далека от обоих классов; именно это делает её устойчивой на малых данных.
- **Margin (зазор)** — чем шире зазор, тем устойчивее модель к новым примерам: широкий margin = меньше шансов ошибиться на данных чуть отличных от обучающих.
- **Support vectors** — парадокс SVM: из всех обучающих примеров только 10–20% (пограничные) определяют модель; остальные можно удалить — ничего не изменится.
- **Kernel trick** — математический трюк: вместо явного перевода данных в высокую размерность SVM вычисляет скалярные произведения в ней через функцию ядра — дёшево и эффективно.
