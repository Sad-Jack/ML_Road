# 27. Random Forest

## Зачем это нужно

Одно дерево решений нестабильно: на чуть других данных получится совсем другое дерево. Random Forest решает эту проблему — строит много разных деревьев и усредняет их предсказания. Получается устойчивая, сильная модель, которую часто берут как первый «серьёзный» бейзлайн на табличных данных.

## Простая интуиция

Спросить одного эксперта — рискованно: он может ошибиться или быть предвзят. Спросить 100 независимых экспертов и взять большинство голосов — намного надёжнее. Random Forest — это «толпа» деревьев, каждое из которых учится на случайной подвыборке данных и случайном подмножестве признаков.

## Что это такое

**Random Forest** — ансамбль из множества деревьев (обычно 100–500), построенный по принципу **bagging**:
- каждое дерево обучается на bootstrap-выборке (со случайным повторением);
- в каждом разбиении рассматривается только случайное подмножество признаков (`max_features`).

Для классификации финальный ответ — голосование большинством. Для регрессии — среднее по деревьям. «Random» именно поэтому: и данные, и признаки берутся со случайностью.

## Как это работает

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

model = RandomForestClassifier(
    n_estimators=300,
    max_depth=None,        # деревья глубокие, шум усредняется ансамблем
    min_samples_leaf=2,
    max_features="sqrt",   # стандарт для классификации
    n_jobs=-1,             # параллелим по ядрам
    random_state=42,
)
model.fit(X_train, y_train)
print(accuracy_score(y_val, model.predict(X_val)))

# Какие признаки важнее всего
for name, imp in sorted(zip(X_train.columns, model.feature_importances_),
                       key=lambda p: -p[1]):
    print(f"{name}: {imp:.3f}")
```

Ключевые параметры:
- `n_estimators` — сколько деревьев (больше = стабильнее, но дольше);
- `max_depth` / `min_samples_leaf` — сложность каждого дерева;
- `max_features` — сколько признаков рассматривать в каждом разбиении.

## Простой пример

Decision Tree без ограничений: train accuracy 1.00, val 0.70 — переобучение. Random Forest из 300 деревьев на тех же данных: train 0.95, val 0.88. Разрыв намного меньше, потому что случайные ошибки одного дерева компенсируются другими.

## Краткий итог

- Random Forest = много деревьев на разных подвыборках данных и признаков.
- Классификация: голосование. Регрессия: среднее.
- Стабильнее и точнее одного дерева, не требует масштабирования.
- Главные параметры: `n_estimators`, `max_depth`, `max_features`.
- Отличный стартовый «серьёзный» бейзлайн для табличных задач.

## Ключевые термины урока

- **Random Forest** — устойчивость достигается двойной рандомизацией: баггинг (разные подвыборки строк) + случайное подмножество признаков на каждом сплите (√p из p). Без рандомизации признаков это просто Bagging, а не RF — деревья будут коррелированы.
- **Bagging (Bootstrap Aggregating)** — каждое дерево видит ~63% уникальных объектов; оставшиеся ~37% — Out-of-Bag (OOB) выборка, которая даёт бесплатную оценку качества без отдельного val.
- **n_estimators** — 100–500 деревьев обычно достаточно; после ~200 прирост качества стремится к нулю. В отличие от Gradient Boosting, увеличение n_estimators не вызывает overfitting.
- **feature_importances_** — средняя суммарная редукция gini/MSE по всем деревьям; работает «из коробки», но переоценивает высококардинальные признаки (у признака с тысячей значений больше шансов случайно оказаться хорошим сплитом).
