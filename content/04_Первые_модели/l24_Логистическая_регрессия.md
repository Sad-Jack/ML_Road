# 24. Логистическая регрессия

## Зачем это нужно

Логистическая регрессия — базовая модель для классификации, аналог линейной регрессии в мире классов. Несмотря на слово «регрессия» в названии, она предсказывает классы, а не числа. Понять её — значит понять, как модели работают с вероятностями и порогами.

## Простая интуиция

Вместо того чтобы сразу сказать «fraud» или «not fraud», модель сначала считает: «с какой вероятностью это fraud?». Если вероятность > 0.5 — говорит «fraud», иначе — «not fraud». Порог можно подвинуть, если цена ошибки в одну сторону выше.

## Что это такое

**Logistic Regression** — модель для классификации, которая:
1. строит линейную комбинацию признаков `z = w·x + b`;
2. прогоняет её через сигмоиду `σ(z) = 1 / (1 + e^(-z))` → вероятность от 0 до 1;
3. сравнивает с порогом (по умолчанию 0.5) и выдаёт класс.

`LinearRegression` — для чисел, `LogisticRegression` — для классов. Работает и для мультикласса (через one-vs-rest или softmax).

## Как это работает

```python
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.metrics import accuracy_score, classification_report

pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("clf",    LogisticRegression(max_iter=1000, C=1.0)),
])
pipe.fit(X_train, y_train)

y_pred  = pipe.predict(X_test)               # классы: [0, 1, 0, 1, ...]
y_proba = pipe.predict_proba(X_test)[:, 1]   # вероятности класса 1

print(accuracy_score(y_test, y_pred))
print(classification_report(y_test, y_pred))
```

- `predict()` — возвращает класс (после порога 0.5).
- `predict_proba()` — возвращает вероятности по всем классам.
- `C` — параметр регуляризации (меньше → сильнее регуляризация).

## Простой пример

Модель выдаёт `p(fraud) = 0.82` → класс «fraud». При `p(fraud) = 0.23` → «not fraud». Если пропускать fraud дороже, чем ложно блокировать клиента — снижаем порог до 0.3:

```python
y_pred_strict = (y_proba >= 0.3).astype(int)
```

Recall вырастет, precision упадёт. Какой порог брать — бизнес-решение, не математическое.

## Краткий итог

- Logistic Regression — классификация, несмотря на название.
- Сначала вероятность через сигмоиду, потом порог → класс.
- `predict()` — класс, `predict_proba()` — вероятности.
- Простая, быстрая, интерпретируемая, даёт калиброванные вероятности.
- Нужен StandardScaler; `C` — сила регуляризации.

## Ключевые термины урока

- **Логистическая регрессия** — модель классификации, несмотря на название «регрессия».
- **Сигмоида** — функция, преобразующая линейный выход в вероятность от 0 до 1.
- **predict_proba()** — метод, возвращающий вероятности классов.
- **Граница решения** — линия в пространстве признаков, разделяющая классы.
