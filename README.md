# ML Road

`ML Road` — офлайн-курс в формате одного HTML-файла: интерактивная страница с уроками по Machine Learning (и отдельным Python-треком), поиском, навигацией по модулям и встроенной “базой знаний” (правая панель).

Для пользователя по сути нужен только **`ml_road.html`**. Всё остальное в репозитории — исходники уроков и сборщик, который генерирует/обновляет этот файл.

## Что внутри курса

Курс собран из трёх “источников”:

- **ML-трек**: `content/ml/**`
- **Python-трек**: `content/python/**`
- **Правая панель (KB/Reference)**: `content/right_panel/**`

В итоговом `ml_road.html` контент уроков хранится в блоках вида `<script type="text/plain" id="cdata-...">`, а структура курса (модули/уроки) — в JS-объектах `COURSE` и `PYTHON_COURSE`.

## Как устроены исходники

### Модули

Каждый модуль — это папка внутри трека, например:

- `content/ml/01_База_ML_и_данные/`
- `content/python/02_Pandas/`

Внутри модуля лежит `_module.json` — метаданные модуля (id, номер, заголовок, цвет и т.п.).

### Уроки

Уроки — это `.md` файлы внутри папки модуля. В начале файла используется YAML frontmatter:

```yaml
---
id: l05
order: 5
num: "05"
title: "05. Как выглядят данные в ML"
short: "Как выглядят данные в ML"
---
```

Тело Markdown после frontmatter становится содержимым урока в `ml_road.html`.

### Правая панель (KB)

Статьи правой панели лежат в `content/right_panel/*.md` и тоже используют frontmatter (например: `id`, `order`, `title`, `category`, `type`, `summary`, `tags`, `priority`). Их содержимое попадает в `RIGHT_PANEL_DATA` внутри HTML.

## Сборка: что за файлы в корне

- `ml_road_template.html` — шаблон, куда подставляются структура курса и контент.
- `build.py` — сборщик: читает `content/**`, обновляет `COURSE`/`PYTHON_COURSE`/`RIGHT_PANEL_DATA`, генерирует cdata-блоки и пишет `ml_road.html`.
- `setup_metadata.py` — вспомогательный скрипт для первичной инициализации метаданных (создание `_module.json` и frontmatter по данным из уже готового `ml_road.html`).
- `ml_road_origin.html` — исходная/референсная версия HTML (как правило используется как “донор” или бэкап).

## Как работать с проектом

### Если вы пользователь (нужен только курс)

1. Возьмите файл **`ml_road.html`** (его можно прислать/выложить отдельно от репозитория).
2. Откройте `ml_road.html` в браузере (двойной клик или перетаскивание в окно браузера).

### Если вы автор (правите курс)

Требования: `python3` (желательно Python 3.9+).

1. Редактируйте уроки и статьи в `content/`.
2. Проверка без записи (опционально):

```bash
python3 build.py --check
```

3. Полная пересборка:

```bash
python3 build.py
```

Результат: обновится/создастся `ml_road.html`.

4. Быстро обновить конкретные уроки в уже существующем `ml_road.html` (патч-режим):

```bash
python3 build.py l05 l20
```

Примечание: патч-режим читает текущий `ml_road.html` и заменяет в нём только выбранные `cdata-...` блоки.
