# ML Road

`ML Road` — офлайн-курс в формате одного HTML-файла: интерактивная страница с уроками по Machine Learning (и отдельным Python-треком), поиском, навигацией по модулям и встроенной «базой знаний» (правая панель).

Пользователю нужен только **`ml_road.html`**. Всё остальное в репозитории — исходники уроков и сборщик, который генерирует и обновляет этот файл.

## Как устроены исходники

### Модули

Каждый модуль — папка внутри трека, например `content/ml/01_База_ML_и_данные/`. Внутри лежит `_module.json` с метаданными модуля (id, номер, заголовок, цвет):

```json
{ "id": "m1", "num": "1", "title": "1. База ML и данные", "color": "#4f46e5" }
```

### Уроки

Уроки — `.md` файлы с YAML frontmatter в начале:

```yaml
---
id: l15
order: 15
num: "15"
title: "15. Временные ряды"
short: "Временные ряды"
---
```

- `id` — уникальный идентификатор; становится именем `cdata`-блока в HTML
- `order` — порядок сортировки внутри модуля (целое число)
- `num` — отображаемый номер урока
- `title` — полный заголовок
- `short` — короткое название для сайдбара

### Правая панель (KB)

Статьи лежат в `content/right_panel/*.md`, используют frontmatter с полями `id`, `order`, `title`, `category`, `type`, `summary`, `tags`, `priority`. Попадают в `RIGHT_PANEL_DATA` внутри HTML.

### Устройство HTML

В итоговом `ml_road.html`:
- структура курса хранится в JS-объектах `COURSE` и `PYTHON_COURSE`
- содержимое уроков — в блоках `<script type="text/plain" id="cdata-lXX">`

## Сборка

### Файлы в корне

| Файл | Назначение |
|------|-----------|
| `ml_road_template.html` | Шаблон HTML без контента уроков |
| `build.py` | Основной сборщик: `content/**` → `ml_road.html` |
| `build_ml_road.command` | Двойной клик на macOS — запускает сборку |
| `renumber_lessons.py` | Переименование уроков при вставке новых |
| `strip_refs.py` | Удаление перекрёстных ссылок «Из урока N» |
| `setup_metadata.py` | Первичная инициализация `_module.json` и frontmatter |

### Команды

**Полная пересборка** (template → html):
```bash
python3 build.py
```

**Патч-режим** — обновить только конкретные уроки в существующем HTML:
```bash
python3 build.py l15 l52
```

**Проверка** — убедиться что content/ совпадает с ml_road.html (без записи):
```bash
python3 build.py --check
```

**Принудительная полная пересборка**:
```bash
python3 build.py --full
```

## Как работать с проектом

### Пользователь

1. Скачайте **`ml_road.html`**.
2. Откройте в браузере (двойной клик или перетаскивание).

### Автор

Требования: Python 3.9+.

1. Отредактируйте или создайте `.md` файл в `content/ml/<модуль>/`.
2. Запустите пересборку:
```bash
python3 build.py
```

### Добавление нового урока

1. Создайте файл `lXX_Название.md` в нужной папке модуля.
2. Добавьте frontmatter (id, order, num, title, short).
3. Если урок вставляется в середину курса — запустите `renumber_lessons.py` для сдвига нумерации.
4. Пересоберите: `python3 build.py --full`.

---

## Технические детали

- `build.py` поддерживает дробные значения `order` (float) для временного позиционирования уроков до перенумерации
- `renumber_lessons.py` использует двухфазное переименование через временные файлы — безопасно при конфликтах имён
- Патч-режим заменяет `cdata`-блоки в уже существующем HTML без полной пересборки — быстрее при редактировании одного урока
