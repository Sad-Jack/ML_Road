## 🔴 Критические проблемы

### 1. l29 — Gradient Boosting: математическая ошибка в примере

**Файл:** `content/ml/04_Первые_модели/l29_Gradient-Boosting.md`

Текущий текст перескакивает от «дерево 2 выдаёт 20» к «следующее дерево предсказывает остаток 28» без явного вычисления нового остатка. Студент не понимает, откуда взялось число 28.

**Исправление:** заменить проблемный участок:
```
# Было (упускает шаг пересчёта):
Сумма: 100 + 0.1 * 20 = 102 (с lr=0.1). Следующее дерево — предсказывает остаток 28.

# Стало (явный пересчёт):
Новое предсказание: 100 + 0.1 * 20 = 102.
Новый остаток: 130 − 102 = 28 (не 30 — остаток уменьшился!).
Третье дерево учится предсказывать новый остаток 28.
Сумма: 102 + 0.1 * дерево3_предсказание → и т.д.
```

### 2. l90 — Transformer: пропущено масштабирование в коде attention

**Файл:** `content/ml/11_Deep_Learning_LLM_AI/l90_Transformer.md`

Пример кода вычисляет `scores = np.dot(query, keys.T)` без деления на `sqrt(d_k)`. Без этого масштабирования при большом `d_k` softmax вырождается в one-hot, градиенты обнуляются — это баг, который ломает понимание Transformer.

**Исправление:**
```python
d_k = keys.shape[-1]  # размерность ключей
scores = np.dot(query, keys.T) / np.sqrt(d_k)  # ВАЖНО: масштабируем
weights = softmax(scores)
output = np.dot(weights, values)
```

**Добавить пояснение:** «Без деления на √d_k при большой размерности скалярные произведения становятся огромными → softmax даёт почти one-hot вектор → модель игнорирует все токены кроме одного → обучение ломается.»

### 4. l78/l79 — p-value без кода вычисления, l79 на него опирается

**Файлы:** `content/ml/09_ML_в_производстве/l78_p-value_и_статистика.md`, `l79_AB-тестирование.md`

l78 объясняет p-value концептуально, но не показывает, как его вычислить. l79 при этом использует результат «p-value < 0.05» как данность. Студент не может воспроизвести расчёт.

**Исправление:** добавить в l78 рабочий пример:
```python
from scipy.stats import proportions_ztest

# A/B тест CTR: 105 кликов из 5000 vs 115 из 5000
count = [105, 115]
nobs  = [5000, 5000]
stat, pval = proportions_ztest(count, nobs)
print(f"p-value: {pval:.4f}")
# p-value > 0.05 → отличие незначимо, не деплоим B
```
