**E: accuracy (доля правильных ответов) без введения**

### l14 — Ранжирование в ML
**B: Q1 — пересказ/путаница**  
«Почему нельзя просто предсказывать score числом и сортировать?» — вторая часть создаёт путаницу: модель как раз и делает score + сортировку.  
**E: CTR без введения**

### l15 — Временные ряды
**B: Q2 — вопрос без опоры в тексте**  
«Какую долю взять для теста?» — конкретных рекомендаций по пропорции (80/20, 90/10) урок не даёт.  
**E: train_test_split без введения**  
Цитата: *«Обычный `train_test_split` перемешивает строки случайно»* — функция sklearn не вводилась в l01–l14.

### l25 — Логистическая регрессия
Мультикласс описан как «голосование между классами» — термин из Random Forest/ансамблей, создаёт путаницу. Правильно: OvR/OvO или multinomial/softmax.

### l46 — Bias и Variance
Формула `Error = Bias² + Variance + Irreducible Noise` подана как универсальная. Строго верна **только для MSE**. Для cross-entropy аналогичного разложения нет.

### l48 — Несбалансированные классы: что делать
«Ключевые термины»: *«правильный порог подбирают по PR-кривой»* — PR-кривая нигде в курсе не вводится.

### l54 — K-Means
Инициализация описана как «случайная», тогда как sklearn по умолчанию использует **K-Means++**. Студент не поймёт, почему `n_init=10` нужен.

### l55 — DBSCAN
`min_samples ≥ числу признаков + 1` — эмпирическое правило 1996 года, не работает для данных с >10 признаками.

### l57 — PCA простыми словами
«Ортогональные компоненты = независимые» — неточно. Ортогональность = линейная некоррелированность, но не статистическая независимость.

### l75 — Мониторинг модели
«Ключевые термины»: feature drift и prediction drift повторяют «Что это такое» без добавления нюансов.

### l84 — AI, ML, DL и GenAI
«Нейронные сети» упоминаются концептуально до формального введения в l85 (веса, слои).

### l86 — Как нейросеть учится
*«Это и есть SGD — основа обучения всех нейросетей»* — противоречит тексту урока, где Adam описан как «адаптивный вариант». Adam формально не является SGD.

### l88 — Архитектуры: CNN, RNN, Transformer
GPT, BERT, Claude, Gemini упомянуты как примеры Transformer до введения LLM в l91.

### l91 — Что такое LLM
Аналогия «книгочей знает, что обычно пишут» подталкивает к модели LLM как «большого n-gram». Нет разграничения decoder-only (GPT) vs encoder-only (BERT). Нет упоминания SFT.
